## Pending Tasks & Open Features (Gaps to Close)

While the core functionality is perfectly aligned with the original SOP-006 goals, the cross-AI reviews and implementation run revealed the following open features needed for the next iteration:

1. **Smart Partial Template Overwrites (High Priority)**
   *Current state:* `write-mir.cjs` overwrites the entire template file on publish.
   *Requirement:* As noted by Claude's evaluation, if a user re-runs the onboarding step *after* manually editing the `PROFILE.md` file, their manual edits will be destroyed. We need to upgrade `write-mir.cjs` to target and replace specific `[FILL...]` tags rather than clobbering the whole file.

2. **UI CSS Refactoring (Medium Priority)**
   *Current state:* `index.html` relies heavily on inline styles for its layout.
   *Requirement:* Extract the extensive inline CSS out of `index.html` into a dedicated `onboarding.css` stylesheet to improve maintainability and resolve persistent linter warnings.

3. **ChromaDB Cloud Failover / Native Client (Medium Priority)**
   *Current state:* The installer relies on an orphaned background `python -m chromadb.cli.cli run` process.
   *Requirement:* Provide a configuration toggle to allow users to plug in a cloud vector database (like Supabase pgvector or hosted Chroma) to bypass the local python requirement entirely.

4. **Integration with `mgsd-neuro-auditor` (Low Priority)**
   *Current state:* Standard basic content generation populates the drafts.
   *Requirement:* Route the generated drafts through the `mgsd-neuro-auditor` gate before presenting them to the user in Step 7 to guarantee archetype alignment instantly on install.
