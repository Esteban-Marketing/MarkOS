### Workstream B: Tenant quota and billing-state proof

**Requirements:** TEN-04, BIL-04

Execution artifacts:
- `.planning/phases/55-tenant-quota-and-billing-failure-closure/55-01-PLAN.md`
- `.planning/phases/55-tenant-quota-and-billing-failure-closure/55-02-PLAN.md`
- `.planning/phases/55-tenant-quota-and-billing-failure-closure/55-03-PLAN.md`

**Why this exists:** Runtime entitlement enforcement is present, but the package does not yet prove tenant quota/rate-limit behavior or carry explicit dunning-workflow closure.

**Deliverables**
- Requirement-specific quota model describing plan-tier limits, exhaustion behavior, and rate-limit semantics.
- Automated tests proving tenant quota or plan-limit exhaustion blocks protected actions deterministically.
- Explicit dunning workflow artifact showing degraded state, operator evidence, and recovery path after payment resolution.
- Verification updates connecting the evidence directly to TEN-04 and BIL-04.

**Execution steps**
1. Define canonical plan-tier limits for the missing quota/rate-limit dimensions.
2. Add tests for over-limit behavior at the request, plugin, and agent execution boundaries.
3. Document and, if needed, implement the dunning state machine from billing failure to restoration.
4. Update closure artifacts with direct references to the new tests and runtime behavior.

**Validation**
- New automated coverage exists for tenant quota and rate-limit enforcement.
- Dunning and restoration behavior is demonstrably encoded in runtime or operator evidence surfaces.
- TEN-04 and BIL-04 can be promoted from `Partial` when proof is explicit.
