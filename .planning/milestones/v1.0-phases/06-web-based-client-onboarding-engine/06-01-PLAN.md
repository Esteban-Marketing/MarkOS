# Plan 06-01: Build Web Onboarding App (Multi-Step Form UI)

## Objective
Build a premium, white-labeled, multi-step web onboarding form that collects all seed data needed to populate the 6 RESEARCH/ files. The form is a self-contained HTML/JS/CSS app served locally by a Node.js server during `markos-new-project`. On submission, it generates and downloads `onboarding-seed.json`.

<task id="06-01-T3">
  <title>Build onboarding.html and onboarding.js — Full form logic</title>
  <read_first>
    - `onboarding/onboarding.css`
    - `onboarding/onboarding-config.json`
    - `onboarding/onboarding-seed.schema.json`
    - `.planning/phases/06-web-based-client-onboarding-engine/06-CONTEXT.md` — 6 form steps
  </read_first>
  <action>
    Create `onboarding/index.html` — the complete multi-step form HTML. Include all 6 step sections (shown/hidden via JS). Structure:
    - `<head>`: meta charset/viewport, title from config, link to onboarding.css, Google Fonts preconnect
    - Header with logo (configurable), form title, step indicator
    - Progress bar div
    - 6 step `<section>` divs (one per step, initially hidden except step 1)
    - Navigation buttons (Back / Next / Submit)
    - Completion screen div (hidden initially)

    Form fields per step (based on JSON schema):
    - Step 1 (Company): company name (text), industry (text), country (text), mission (textarea), brand values × 3 (text inputs), tone of voice (select: Formal/Balanced/Casual/Playful), language (text)
    - Step 2 (Product): product name (text), category (text), primary benefit (textarea), feature × 3 (text), price range (text), main objection (textarea)
    - Step 3 (Audience): segment name (text), age range (text), job title (text), pain point × 3 (text), online hangouts (text), vocabulary/phrases (textarea)
    - Step 4 (Competition): competitor name + URL + differentiator + gap × 3 (repeating group)
    - Step 5 (Market): maturity (select), biggest trend (textarea), seasonal patterns (text), regulatory concern (text)
    - Step 6 (Content): best piece URL (url), active channels (checkboxes: LinkedIn/Instagram/Twitter/Facebook/YouTube/TikTok/Email/Blog/Podcast), monthly output (select), best format (select)

    Create `onboarding/onboarding.js` with:
    - Load `onboarding-config.json` and apply primary_color to CSS var
    - Step navigation (currentStep, showStep, validateStep)
    - Progress bar update on each step
    - Validation: required fields must be non-empty before advancing
    - localStorage save/restore on every field change (`markos-onboarding-draft`)
    - `buildSeed()` function that assembles final JSON matching the schema
    - `handleSubmit()`: calls buildSeed(), sets `metadata.generated` = ISO timestamp, `metadata.completeness_score` = number of non-empty sections, downloads JSON as `onboarding-seed.json`, shows completion screen
    - Keyboard: Enter advances to next step (not submit)
  </action>
  <acceptance_criteria>

    - HTML contains a progress bar div

    - JS contains `buildSeed()` function
    - JS contains `localStorage.setItem` calls for draft persistence
    - JS contains `metadata.generated = new Date().toISOString()`
    - JS contains a JSON download trigger (Blob + URL.createObjectURL pattern)
    - All step field IDs match the JSON schema property names
  </acceptance_criteria>
</task>
