## Check 1: ATT-01 attribution drill-down review

**Requirement coverage:** ATT-01
**Goal:** Confirm contact, deal, and campaign attribution remain visible in the CRM shell with deterministic fixed-weight evidence.

### Commands

```powershell
Invoke-WebRequest -Uri "http://localhost:3000/api/crm/reporting/attribution" -Headers @{ Authorization = "Bearer <reporting-token>" }
```

### Pass criteria

- Attribution payload returns CRM-linked rows.
- The reporting cockpit exposes attribution drill-down inside the CRM shell.
- Evidence references remain visible for campaign touch, web activity, or outbound history.

## Check 2: REP-01 reporting cockpit review

**Requirement coverage:** REP-01
**Goal:** Confirm pipeline health, conversion, attribution, SLA risk, and agent productivity remain in one CRM-native cockpit, with the governed central-rollup review captured as a required sub-step of the same cockpit check rather than as a fifth workflow.

### Commands

```powershell
Invoke-WebRequest -Uri "http://localhost:3000/api/crm/reporting/dashboard" -Headers @{ Authorization = "Bearer <reporting-token>" }
```

### Required hosted sub-steps

- Review the tenant reporting cockpit in the shared CRM shell.
- Confirm the executive summary remains attached to the same truth layer as the main cockpit.
- If a central-operator account is available, perform the governed central-rollup review and record the role context in the same run record section.

### Pass criteria

- Dashboard payload returns unified cockpit metrics.
- The product route shows one shell for pipeline health, attribution, risk, productivity, readiness, and verification.
- Executive summary remains connected to the same truth layer.
- Governed central-rollup access, when available, is recorded as part of this same cockpit review and does not require a separate checklist track.
