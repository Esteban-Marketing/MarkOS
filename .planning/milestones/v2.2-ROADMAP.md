# Roadmap: MarkOS v2.2 - Platform Engineering

## Milestone Theme

MarkOS platform engineering: infrastructure fixes, protocol-grade onboarding reliability, operational workflow enablement, and cloud-canonical migration readiness.

## Phase Sequence

1. **Phase 28 - Runtime Integrity (P0)**
   - Approve write-path hardening.
   - Node runtime contract (`>=20.16.0`).
   - Install-time private-data gitignore protection.

2. **Phase 29 - Operational Enablement (P1)**
   - Linear sync endpoint and client.
   - Campaign result endpoint and winners write path.
   - Interview cap and bounded UX transition.

3. **Phase 30 - MarkOSDB Migration (P1.5/P2)**
   - Supabase/Upstash contracts.
   - Local-to-cloud ingestion pipeline.
   - Next.js + Supabase auth boundary integration.

4. **Phase 31 - Rollout Hardening (P2)**
   - Reliability SLOs and telemetry.
   - Migration mode/rollback controls.
   - Security/compliance guardrails.
   - Compatibility deprecation progression.

## Delivery Gate

- No Phase 29/30/31 execution starts before Phase 28 completion.
- Phase 30 requires Phase 29 outcome tagging behavior in place.
- Phase 31 closes v2.2 only after migration safety and policy checks are verifiable.
