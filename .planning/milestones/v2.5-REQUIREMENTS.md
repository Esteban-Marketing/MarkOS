# Requirements: v2.5 Enhancement & Optimization Process

## Installer Experience

- [ ] **DX-01** Bare `npx markos` installs MarkOS through a single obvious happy path for a typical local setup.
- [ ] **DX-02** Installer minimizes prompts by applying safe defaults first and exposing explicit override flags when customization is needed.
- [ ] **DX-03** Installer-owned artifacts converge cleanly on rerun without duplicate `.gitignore` blocks, manifest drift, or ambiguous existing-install behavior.

## Operational Readiness

- [ ] **OPS-READY-01** Install classifies final state as `ready`, `degraded`, or `blocked` using shared env/vector/onboarding readiness checks and actionable next steps.

## Automation Compatibility

- [ ] **CI-01** Install/update flows behave deterministically in CI or other non-interactive environments and never hang waiting for prompts.

## Phase Mapping

- DX -> Phase 35
- OPS-READY -> Phase 35
- CI -> Phase 35
