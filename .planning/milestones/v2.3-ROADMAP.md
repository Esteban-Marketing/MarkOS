# Roadmap: MarkOS v2.3 - Codebase Documentation Intelligence

## Milestone Theme

Make the repository self-navigable through the Get Shit Done framework by producing a canonical codebase map that documents runtime surfaces, folder topology, file responsibilities, and maintenance rules.

## Phase Sequence

1. **Phase 33 - Codebase Documentation Mapping**
   - Bootstrap canonical `.planning/codebase/` documents used by GSD planning and execution.
   - Produce route-by-route documentation for onboarding HTTP endpoints, API wrappers, and CLI entrypoints.
   - Produce folder-by-folder and file-by-file inventories for maintained repository surfaces.
   - Synchronize `README.md`, `TECH-MAP.md`, and `.protocol-lore/CODEBASE-MAP.md` to the canonical map and add freshness checks.

## Delivery Gate

- `.planning/codebase/` exists with both standard GSD mapping docs and MarkOS-specific route/folder/file extensions.
- Every production route and operator entrypoint is traceable to a concrete implementation file.
- Protocol-facing docs link back to the canonical map and do not define conflicting topology truths.
- Verification steps exist for future route additions, file moves, and directory growth.
