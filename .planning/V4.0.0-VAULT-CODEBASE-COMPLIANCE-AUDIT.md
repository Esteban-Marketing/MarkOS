# v4.0.0 Vault-to-Codebase Compliance Audit

**Date:** 2026-04-22  
**Purpose:** Compare the current built MarkOS app against the ideal requirements now canonical in the Obsidian vault, including the v2 blueprint intake, Pricing Engine intake, and SaaS Suite intake.  
**Conclusion:** MarkOS has strong SaaS, tenancy, MCP, webhooks, CRM, billing-ledger, and partial AgentRun foundations, but it is not yet fully compliant with the v2 Marketing Operating System doctrine or the new SaaS Suite doctrine. The missing foundation work is routed into Phases 204-213; SaaS Suite work is reserved for Phases 214-217.

## Phase 204 Review

Phase 204 is still valid and important, but it is not implemented in the current app. Evidence:

- `bin/cli-runtime.cjs` does not yet expose the planned `init`, `plan`, `run`, `eval`, `login`, `keys`, `whoami`, `env`, `status`, and `doctor` commands.
- `bin/commands/` does not exist.
- Phase 204 had a deferred note that could route winget/apt into Phase 205. That is now invalid because Phase 205 is Pricing Engine-owned.

Updates needed:

- Add `204-13-PLAN.md` for v2 compliance guardrails.
- Keep winget/apt out of Phase 205; route them to a later distribution hardening phase or 204.x gap closure.
- Ensure `markos run`, `markos status`, and `markos doctor` understand AgentRun v2, approval, cost, connector, and Pricing Engine placeholder rules.

## Phase 205 Review

Phase 205 has correctly been re-scoped to Pricing Engine Foundation + Billing Readiness. Current code does not implement the new engine. The billing foundation is useful, but it uses static allowances and pricing snapshots rather than engine-owned recommendations.

Updates needed:

- Add plan files `205-01` through `205-08`.
- Reserve fresh contract and migration IDs.
- Treat Stripe/self-serve billing as a handoff after PricingRecommendation, TenantCostModel, and pricing floor objects exist.
- Add explicit compatibility gates for existing hard-coded billing UI/API defaults.

## Phase 206 Review

Phase 206 was still written as generic SOC2 work from 2026-04-17. It now needs to cover the actual v2 risk profile:

- AgentRun/approval evidence.
- Pricing Engine evidence and price-change approval.
- Connector privacy, retention, and failure recovery.
- Research/evidence/source quality governance.
- Cross-tenant learning anonymization.
- Tenant 0 public proof controls.

Updates needed:

- Rewrite Phase 206 discussion context.
- Add SOC2 plan slices for AI governance, data inventory, policy ratification, evidence automation, vendor/crawler review, incident/BCP, and pen-test readiness.

## Close Criteria for Vault Compliance

The application should not be called 100% compliant with the vault until:

- Phase 204 ships CLI GA with v2 guardrails.
- Phase 205 ships Pricing Engine foundation.
- Phase 206 has SOC2 controls that include AI, pricing, connector, and learning risks.
- Phases 207-213 have been researched, planned, executed, and verified.
- Phases 214-217 have been researched and planned before any SaaS Suite implementation claim.
- Active Obsidian doctrine, `.planning/ROADMAP.md`, `.planning/REQUIREMENTS.md`, and implementation contracts agree on the same requirement IDs.
- PageIndex is regenerated after every vault doctrine update.
