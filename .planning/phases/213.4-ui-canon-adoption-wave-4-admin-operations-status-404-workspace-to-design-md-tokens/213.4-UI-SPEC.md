# Phase 213.4 тАФ UI Design Contract

> Visual and interaction contract for redesigning the final wave of MarkOS operator + admin + system surfaces тАФ admin/billing, admin/governance, operations (root), operations/tasks composite, status/webhooks (public), 404-workspace (error hero), and settings/theme тАФ from the legacy "Sora + Space Grotesk + `#0f766e`/`#0d9488` teal fills + `#ffffff`/`#f5f7fa` light bg + 28px radii + `0 18px 44px rgba()` drop-shadows + `linear-gradient` fills" implementation into the canonical MarkOS visual contract defined in `/DESIGN.md` v1.1.0 (dark Kernel Black `#0A0E14` surface, Protocol Mint `#00D9A3` signal, JetBrains Mono + Inter, strict 8px grid, borders over shadows, flat). Also folds in three priority polish fixes flagged by Phase 213.3 ui-review (commit `34e4bee`). Generated by gsd-ui-researcher.

**This is a redesign, not a token-swap port.** All 182 inline-hex occurrences across the 6 module.css files are replaced with token citations or canonical alpha-tint formulas. Every banner-shaped surface composes `.c-notice c-notice--{state}`. The redesign posture mirrors Phases 213.1тАУ213.3 exactly. This phase closes the 213.x DESIGN.md adoption initiative across the full MarkOS chrome surface.

## Spacing Scale

Authoring rule: every `padding`, `margin`, `gap`, `inset`, `top`, `right`, `bottom`, `left` in scope MUST cite a `--space-*` token. No arbitrary px. Off-grid values are auto-FAIL.

| Token | Value | DESIGN.md citation | Usage in this phase |
|-------|-------|--------------------|---------------------|
| `--space-none` | 0 | `spacing.none` | Reset margins on `<h1>`, `<p>`, `<table>` |
| `--space-xxs` | 2px | `spacing.xxs` | Badge inner padding, status-dot hit-area offset, step-index label inner padding |
| `--space-xs` | 8px | `spacing.xs` | Table row `gap`, form field gap (labelтЖФinputтЖФhelp), step-card meta-chip group gap, inline icon-text adjacency |
| `--space-sm` | 16px | `spacing.sm` | Card vertical rhythm, notice padding-block, table `th`/`td` padding-block, status-line `padding`, mobile horizontal page padding, step-card padding |
| `--space-md` | 24px | `spacing.md` | Card padding (default via `.c-card`), gap between major sections, modal padding, page horizontal padding at smтАУmd viewport, operations 3-region column gap |
| `--space-lg` | 32px | `spacing.lg` | Modal padding (via `.c-modal`), page horizontal padding тЙе md, evidence-rail padding, admin-billing hero padding, inter-section gap within large surfaces (operations/tasks) |
| `--space-xl` | 48px | `spacing.xl` | Page vertical padding тЙе lg, status/webhooks page top padding (centered card), major content-region vertical breathing |
| `--space-xxl` | 96px | `spacing.xxl` | Reserved тАФ not used in this wave |

**Allowed exceptions (DESIGN.md documented):**
1. `1px` for hairline borders (`var(--color-border)`) тАФ explicitly preferred over shadows on dark surfaces.
2. `2px` for focus ring width and offset (`var(--focus-ring-width)`, `var(--focus-ring-offset)`).
3. `4px` for `.c-notice` `border-inline-start` accent тАФ composed via `.c-notice` primitive; modules never declare this directly.
4. `8px` for cost-meter / seat-bar / delivery-success track height тАФ Claude's Discretion (visual affordance for progress bars; sizing value, not spacing; cited with rationale comment).
5. `max-width: 1280px` for admin surfaces (`--w-container`) тАФ already a canonical token.
6. `max-width: 720px` for status/webhooks card (centered standalone card; `--w-prose` token).
7. `width: min(540px, 100%)` for 404-workspace card тАФ size-capped hero (composable with `var(--w-modal)` 560px as approximate; cite as `width: min(var(--w-modal), 100%)` in redesign).
8. Three-column layout fractions (`0.9fr 1.4fr 0.9fr`, `30% 45% 25%`) тАФ layout fractions are not spacing values.
