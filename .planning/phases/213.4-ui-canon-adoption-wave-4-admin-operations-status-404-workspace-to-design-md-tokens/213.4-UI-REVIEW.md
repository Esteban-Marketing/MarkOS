## Top 3 Priority Fixes

1. **Runtime reduced-motion and forced-colors not browser-verified** — Users with OS-level motion sensitivity or Windows High Contrast mode cannot be confirmed working purely from static grep; @media blocks exist but runtime behavior is unproven. Fix: complete UAT items 2 and 3 from VERIFICATION.md (Edge forced-colors simulation + OS reduce-motion on operations/tasks step cards). No code change expected; this is a sign-off gate only.

2. **"Cancel" in modal CTAs is generic** — `admin/billing/page.tsx:189` and `admin/governance/page.tsx:106` use bare "Cancel" as the dismiss label on destructive-action confirm modals. DESIGN.md voice principle favors unambiguous labels. Fix: change to "Keep write-off" / "Keep rejection" or equivalent action-specific labels that make the dismissal consequence clear without ambiguity — one-char change per file.

3. **ARIA density is low for complex composites** — operations/tasks composite (step-runner, task-graph, approval-gate, evidence-panel across 1384+ TSX LOC) registers only ~16 total aria-label/role attributes across all 13 TSX files combined. The task-graph DAG and step-runner live-execution panel are interaction-dense and would benefit from explicit `role="status"` on live-update regions and `aria-live="polite"` on step-state changes. Fix: add `role="status"` + `aria-live="polite"` to the step execution status region in step-runner.tsx; add `aria-label` to icon-only buttons in task-graph.tsx.

### Pillar 4: Accessibility (3/4)

MEETS WITH FLAGS.

**Verified by static analysis:**
- `@media (prefers-reduced-motion: reduce)` blocks present: `task-ui.module.css:208` + `settings/members/page.module.css:132`
- `@media (forced-colors: active)` cascade in `app/tokens.css:219` (global — all 213.4 surfaces inherit)
- `@media (pointer: coarse)` NOT redeclared in any of the 6 new module.css files (X-5 test confirms) — inherited from `styles/components.css`
- 2px solid focus ring on `--color-primary` defined globally; no module-level suppression found (zero `outline: none` or `outline: 0` in any 213.4 module.css)
- 404-workspace `.c-card--feature` uses `aria-labelledby="heading"` — wired

**FLAGS requiring human UAT (cannot be resolved by static audit):**
- UAT-2: Forced-colors actual rendering (Edge + Windows High Contrast Aquatic) — structural presence of `@media (forced-colors: active)` confirmed; runtime pixel rendering requires browser
- UAT-3: OS-level reduced-motion on operations/tasks `.stepCard` transitions — `@media` block confirmed at line 208; runtime suppression requires OS setting
- UAT-4: `.c-button` `min-height: 44px` on coarse pointer — global rule confirmed in components.css (X-5 test passes); runtime device emulation required

**ARIA density concern (FLAG):** 16 total `aria-*` attributes across all 13 TSX files is sparse for a composite that includes a live-executing DAG graph (task-graph.tsx), a step-runner with live state transitions, and approval-gate forms. Specifically:
- No `aria-live` or `role="status"` found on step execution status regions in step-runner.tsx
- task-graph.tsx node state changes (queued/executing/completed) lack `aria-label` on state badges
- These are not AC violations (UI-SPEC does not enumerate per-element ARIA requirements) but represent a gap for the final 213.x milestone closure

**Score rationale:** 3/4 not 4/4 because 5 UAT items remain unresolved (VERIFICATION.md human_needed), including two accessibility-critical checks (forced-colors + reduced-motion runtime), and ARIA density on the operations/tasks composite is below what the interaction complexity warrants.

### Pillar 5: Brand Voice (3/4)

MEETS WITH MINOR FLAGS.

**PASS:** Zero items from the banned lexicon (`synergy`, `leverage`, `empower`, `unlock`, `transform`, `revolutionize`, `supercharge`, `holistic`, `seamless`, `cutting-edge`, `innovative`, `game-changer`, `next-generation`, `world-class`, `best-in-class`, `reimagine`, `disrupt`) in any 213.4 TSX file post X-4 fix. No exclamation marks. No "Please" softeners. No emoji.

**Notice copy quality (PASS):** Admin/billing copy is precise and action-oriented — "Billing evidence reconciled. No operator intervention required." / "Billing hold active. Restricted actions paused until billing health is restored." / "Provider sync failure unresolved. Review evidence to reconcile or place a hold." — meets DESIGN.md "imperative or neutral" register.

**MINOR FLAGS:**
- `status/webhooks/page.tsx:48`: `return 'just now'` — this is a date utility function fallback, not product copy displayed as a primary label. It appears in a `formatRelative()` helper returning a display string. "just now" is technically a softener per DESIGN.md voice tone but the context is a timestamp formatter fallback, not a CTA or state label. Low severity — recommend changing to a literal time string ("< 1 minute ago") for strict compliance.
- `admin/billing/page.tsx:189` / `admin/governance/page.tsx:106`: "Cancel" as modal dismiss CTA — generic but contextually unambiguous in a modal with a clearly-labeled destructive action button. DESIGN.md bans the listed lexicon but does not explicitly name "Cancel." Flagged as a voice improvement opportunity, not a violation.

**Score rationale:** 3/4 because "just now" in a user-visible string output is a softener adjacent to the banned register, and the "Cancel" labels in high-stakes destructive modals miss an opportunity for action-specific copy.
