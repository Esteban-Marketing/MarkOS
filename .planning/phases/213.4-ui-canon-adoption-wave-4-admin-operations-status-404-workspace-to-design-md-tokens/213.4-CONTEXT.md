## Specific Ideas

- **Brand voice anchor (carry-forward from 213.1+213.2+213.3):** Strip all "Please" softeners. Imperative or neutral copy. No exclamation marks. No emoji.
- **Polish fix priority:** F-1/F-2/F-3 land in plan 213.4-01 BEFORE any new surface work. This ensures Phase 213.3 visual debt clears before more surfaces are added — also makes verification cleaner (213.3 surfaces would fail F-1/F-2/F-3 ui-review on re-audit otherwise).
- **404-workspace hero panel:** Compose `.c-card--feature` (32px radius, 32px padding) for the error hero. Body uses bracketed `[err]` glyph + `<h2>` "Workspace not found." + body text + `.c-button--primary` "Back to dashboard" + `.c-button--tertiary` "Switch workspace". NO red splash. NO "Oops!" copy. Tone matches DESIGN.md "Don't shout" principle.
- **Status page register (status/webhooks):** Public-facing — copy must be customer-grade. "All systems operational" / "Investigating" / "Identified" / "Monitoring" / "Resolved" — match Phase 203 status-page substrate verbatim (do NOT introduce new states). State badges via `.c-badge--{success,warning,error}` with bracketed-glyph prefix.
- **Operations dashboard:** Operator role — internal copy, density-priority. Approval queue + run-status overview + connector-health summary. `.c-status-dot--live` + `[ok] live` for active runs; `.c-status-dot--error` + `[err] failed` for failed runs.
- **Operations/tasks composite:** Largest surface. Task graph (DAG visualization), step-runner (live execution), evidence-panel (artifact display), approval-gate (gating UI), task-store (state management — likely no UI in store itself). Each sub-component composes primitives independently. `task-graph.tsx` may have inline SVG nodes — preserve SVG; only update wrapper className. `step-runner.tsx` is the largest tsx (344 LOC) — sub-component extraction NOT recommended (already a sub-component).
- **Admin/governance approval-gate forms:** Use `.c-input` + `.c-field` for any approval comment textarea. Action buttons: `.c-button--primary` Approve + `.c-button--destructive` Reject. Confirm-modal on Reject (Phase 207 audit-log emit).
- **Admin/billing reconciliation:** Discrepancy rows compose `.c-badge--warning` for "Mismatch" / `.c-badge--info` for "Pending review" / `.c-badge--success` for "Reconciled". Write-off action: `.c-button--destructive` with `.c-modal` confirm.
- **Theme-toggle surface (`settings/theme`):** Special case — the canonical visual diagnostic for the design system. Stories should demonstrate dark mode + light mode + forced-colors mode token swatches. The `page.tsx` at 15 LOC is just a switch wrapper — preserve. Stories are where the value lives.
