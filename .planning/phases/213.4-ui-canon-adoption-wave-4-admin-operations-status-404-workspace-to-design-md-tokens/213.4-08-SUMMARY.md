## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] OT-8 test redirected to correct source file**
- **Found during:** Task 2
- **Issue:** Plan's OT-8 test asserted `c-modal`/`c-backdrop` in `step-runner.tsx`, but those live in `approval-gate.tsx` (ApprovalGate is rendered by StepRunner). step-runner.tsx only has the `.c-button` row.
- **Fix:** Split OT-8 into two tests: step-runner.tsx asserted for button primitives; approval-gate.tsx asserted for modal/backdrop.
- **Files modified:** `test/ui-a11y/213-4-admin-ops-a11y.test.js`

**2. [Rule 1 - Bug] X-1 regex false-positive on border-color**
- **Found during:** Task 2 (first suite run)
- **Issue:** Regex `/color:\s+var\(--color-primary\)\s*;/` matched `border-color: var(--color-primary);` in `admin/billing/page.module.css` (evidence rail active stripe — an intentional border, not a text color).
- **Fix:** Line-anchored regex `/^\s+color:\s+var\(--color-primary\)\s*;/` via split-line check.
- **Files modified:** `test/ui-a11y/213-4-admin-ops-a11y.test.js`

**3. [Rule 1 - Bug] X-4 banned lexicon hit in task-graph.tsx**
- **Found during:** Task 3 (X-4 verification)
- **Issue:** `task-graph.tsx` line 128: `"Complete all prior steps to unlock this step for execution."` — "unlock" is in DESIGN.md banned lexicon.
- **Fix:** Changed to `"Complete all prior steps to enable this step for execution."`
- **Files modified:** `app/(markos)/operations/tasks/task-graph.tsx`
- **Commit:** `572347d`
