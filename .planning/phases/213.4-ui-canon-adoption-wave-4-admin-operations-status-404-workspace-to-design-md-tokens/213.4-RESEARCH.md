## Risks + Pitfalls

- **`task-ui.module.css` scope creep (LARGEST risk):** 525 LOC, 79 hex, 5 sub-components. Executor must track which sub-component consumes which class. Build the class→component usage map before rewriting.
- **`page.tsx` Tailwind inline hex in operations/tasks (OT-9):** `bg-[#f5f7fa]`-style Tailwind classes mixed with CSS Modules. Must convert ALL inline hex utilities to CSS Module classes. Easy to miss during grep scan.
- **Phase 203 wiring breakage risk (status/webhooks):** Public-facing. Copy register is locked. Do NOT rename `classifyStatus()` output values or `statusCopy()` keys. Wiring check is mandatory per plan (SW-7).
- **`.c-notice` on `.statusLine` in status/webhooks:** The `data-status` attribute drove bespoke CSS variants. After redesign, the variant is driven by `className` prop in TSX (conditional `c-notice c-notice--{state}`), not CSS attribute selectors. Executor must update TSX to pass class names, not just data attributes.
- **`prefers-reduced-motion` wildcard in status/webhooks:** The `* { … !important }` block is a WCAG trap — it overrides inline styles from third-party embeds. Replace with targeted rules. Grep for `!important` in redesigned module must return 0.
- **Transform-jiggle on `.stepCard:hover` (OT-1):** `transform: translateY(-1px)` is banned per DESIGN.md. This is in the HOVER selector — easy to miss. Grep for `translateY` must return 0.
- **`.stepCardDimmed` opacity upgrade:** 0.8 → 0.5 (clearer inactive signal per UI-SPEC). Document rationale in comment.
- **`admin/layout.tsx` role gate:** Verify file exists before touching any `admin/*` file. If it has role guards, preserve verbatim — DO NOT modify as part of className updates.
- **`task-store.tsx` (minimal UI):** Lowest risk sub-component. State management module with minimal render surface. If it has no classNames to update, confirm explicitly rather than skipping without checking.
- **Off-grid `20px` spacing in both admin modules:** `.sectionTitle margin-bottom: 16px`, `.panel gap: 16px` → `var(--space-sm)`. Common trap: 16px IS on-grid (maps to `--space-sm`), but 20px is NOT. Check each value.
- **`evidenceItem` left-stripe decision (admin/billing):** A 4px left stripe on evidence items is NOT a `.c-notice` context (it's a visual affordance, not a banner). Document the distinction in a comment so executor doesn't reflexively apply the primitive.
- **Stories with `useEffect`/`fetch` (status/webhooks):** `page.tsx` uses live data. Stories must use mock `snapshot` prop or story-level args — NOT live fetch. Mirror the Storybook pattern from `213.3-settings-a11y.test.js` mocking approach.

- **Q: Can `(pointer: coarse)` block be omitted from all 6 new modules?** — YES per T-5. The global rule in `styles/components.css` covers all `.c-button` elements. Modules MUST NOT redeclare — the absence IS the correct behavior.
