## Setup runbook

Generate the apt signing key interactively:

```bash
gpg --full-generate-key
# Choose RSA 4096, 2-year expiry.
# Name: MarkOS Apt Signing
# Email: apt@markos.dev
```

Export the private key for CI:

```bash
gpg --armor --export-secret-keys apt@markos.dev | base64 > markos-apt-key.b64
```

Export the public key for users:

```bash
gpg --armor --export apt@markos.dev > key.gpg
gpg --fingerprint apt@markos.dev
```

Publish `key.gpg` at:

```text
https://apt.markos.dev/key.gpg
```

Set these GitHub Actions secrets:

- `WINGET_PKGS_PAT`: PAT for the winget-pkgs fork/PR path.
- `MARKOS_APT_GPG_KEY`: contents of `markos-apt-key.b64`.
- `MARKOS_APT_GPG_PASSPHRASE`: signing key passphrase, or empty if the key is unprotected.
- `MARKOS_APT_REPO_HOST`: scp host, for example `deploy@apt.markos.dev`.
- `MARKOS_APT_REPO_KEY`: private SSH key for the apt repo upload host.

Provision apt hosting:

- Point `apt.markos.dev` at the static hosting/CDN bucket.
- Ensure the upload user can write the repo root containing `dists/` and `pool/`.
- Publish the exported public key as `/key.gpg`.

Submit winget:

- Fork `https://github.com/microsoft/winget-pkgs`.
- Copy the rendered version directory from `winget-pkgs/manifests/m/markos/markos/<version>/`.
- Open a PR to Microsoft winget-pkgs.
- Wait for maintainer review, usually 1-7 days.

## Smoke tests

Debian or Ubuntu after apt secrets and DNS are live:

```bash
sudo install -d -m 0755 /etc/apt/keyrings
curl -fsSL https://apt.markos.dev/key.gpg | sudo gpg --dearmor -o /etc/apt/keyrings/markos.gpg
echo "deb [signed-by=/etc/apt/keyrings/markos.gpg] https://apt.markos.dev stable main" | sudo tee /etc/apt/sources.list.d/markos.list
sudo apt update
sudo apt install markos
markos --version
markos doctor --check-only
```

Windows after Microsoft merges the winget PR:

```powershell
winget install markos.markos
markos --version
markos doctor --check-only
```

Release CI preflight:

```bash
node --test --test-isolation=none test/cli/204.1/winget-manifest-shape.test.js test/cli/204.1/apt-bump-shape.test.js
node --check scripts/distribution/bump-winget-manifest.cjs

```

## Rollback

- apt key compromise: revoke the signing key, remove the CI secret, publish a new `key.gpg`, regenerate and resign the repo with the replacement key, then update the documented fingerprint.
- apt bad package: remove the affected `.deb` from `pool/`, regenerate `Packages`, `Packages.gz`, `Release`, `InRelease`, and `Release.gpg`, then upload the corrected repo tree.
- apt host incident: remove or empty `dists/` and `pool/` at the host, then publish a maintenance notice until a clean signed repo is available.
- winget bad manifest: open a follow-up PR to `microsoft/winget-pkgs` removing or correcting the affected version directory.
