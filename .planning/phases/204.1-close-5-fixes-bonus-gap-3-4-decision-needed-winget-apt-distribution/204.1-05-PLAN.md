Winget manifest schema reference (https://github.com/microsoft/winget-pkgs/blob/master/doc/manifest):
- ManifestVersion 1.6.0 (current as of 2026-04)
- PackageIdentifier MUST be `<Publisher>.<PackageName>` — this plan uses `markos.markos`
- 3-file split is mandatory for manifest schema >= 1.0: version + installer + locale
- `winget validate --manifest <dir>` runs as the CI gate

Debian repo format (https://wiki.debian.org/DebianRepository/Format):
- Repo structure: `dists/stable/main/binary-amd64/Packages` (gzipped) + `dists/stable/Release` (signed)
- Pool layout: `pool/main/m/markos/markos_<version>_amd64.deb`
- GPG signing: `gpg --clearsign --output InRelease Release` + `gpg --abifile Release.gpg --detach-sign Release`

Existing P204 distribution flow (from RESEARCH.md):
- npm tarball is the source of truth for hash/URL
- All distribution channels reference the same npm-published tarball (single artifact, multiple front doors)
- bumps trigger on `npm publish` succeeded → CI dispatches version bumps in parallel
