<decisions>
## Implementation Decisions

### D-01: Audit trail (P0 — UI-REVIEW Top Fix #1)
- New file: `bin/lib/cli/audit.cjs` exporting `recordCommandAudit({command, args_redacted, profile, exit_code})`
- POSTs minimal envelope to tenant audit endpoint after each command finishes
- Wrapper applied via `bin/markos.cjs` dispatch (not per-command) — single integration point, all 11 commands gain audit automatically
- Args redaction: scrub `--password`, `--token`, `--key`, env values; whitelist remaining flags
- Failure mode: audit POST failure logs to stderr `(verbose mode only)` and DOES NOT change command exit code
- Acceptance: `grep -c "recordCommandAudit" bin/markos.cjs` returns 1; `grep -rc "recordCommandAudit" bin/lib/cli/audit.cjs` shows the export; integration test verifies audit row created on `markos status` invocation

### D-02: Spinner primitive (P1 — UI-REVIEW Top Fix #2)
- New file: `bin/lib/cli/spinner.cjs` exporting `createSpinner({label, stream='stderr'})` → returns `{tick, stop}`
- UTF-8 frames `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏` at 80ms; ASCII fallback `\|/-\\` at 100ms (per UI-SPEC §Token Layer 3)
- Gates: `shouldUseColor + isTTY + !shouldUseJson` (per UI-SPEC §Suppression Matrix)
- Wire into: `keys list`, `whoami`, `env list`, `env pull`, `env push`, `status`, `run` submit, `login` poll
- Acceptance: `ls bin/lib/cli/spinner.cjs` exists; spinner test asserts UTF-8 frames in TTY+UTF8 env, ASCII frames in TTY+`LANG=C` env, no output in non-TTY env

### D-03: stdout/stderr stream separation (P1 — UI-REVIEW Top Fix #3)
- Move success-message writes from stdout → stderr in:
  - `bin/commands/keys.cjs:246` (`Key … revoked.`)
  - `bin/commands/env.cjs:242,306,378` (Pulled/Pushed/Deleted)
  - `bin/commands/login.cjs:49` (`✓ Logged in …`)
- JSON-mode output stays on stdout (per UI-SPEC §Streams)
- Acceptance: `grep -c "process.stdout.write" bin/commands/{keys,env,login}.cjs` for those specific success lines returns 0; redirected-output integration test verifies success messages NOT in stdout when human mode + redirected

### D-04: ASCII fallback for box chars (P2 — UI-REVIEW Top Fix #4 + UI-SPEC Audit Gap #1)
- New helper in `bin/lib/cli/output.cjs`: `pickGlyphs()` returning `{topLeft, topRight, bottomLeft, bottomRight, horiz, vert, junction}` based on UTF-8 detection (`process.env.LANG`, `LC_ALL`, `process.platform`)
- UTF-8 tier: `┌`, `┐`, `└`, `┘`, `─`, `│`, `┼`
- ASCII tier (LANG=C, cmd.exe legacy chcp 437): `+`, `+`, `+`, `+`, `-`, `|`, `+`
- Replace hardcoded U+2500 chars in: `errors.cjs:56-58`, `bin/commands/whoami.cjs:80,86,89`, `bin/commands/status.cjs:178,181,183`, `bin/commands/doctor.cjs:52,70`
- Acceptance: `grep -c "[┌│└─]" bin/lib/cli/errors.cjs bin/commands/{whoami,status,doctor}.cjs` returns 0 (all glyphs come through pickGlyphs); test asserts ASCII output when `LANG=C`

### D-05: <60-col width breakpoint in renderTable (P2 — UI-REVIEW Top Fix #5 + UI-SPEC Audit Gap #5)
- Update `bin/lib/cli/output.cjs::renderTable` to detect `process.stdout.columns < 60`
- Below threshold: switch from horizontal table to vertical key:value layout (one entity per record, blank line between)
- Above threshold: existing horizontal table renderer
- Use `process.stdout.columns ?? 80` (default 80 if not TTY)
- Acceptance: `renderTable` test asserts vertical output when columns=50; horizontal output when columns=120; `markos keys list` integration test on COLUMNS=50 shell renders vertical

### D-06: env mask rule reconciliation (Bonus Gap #4 — DECISION REQUIRED)
- **Conflict:** UI-SPEC §Mask rules says "last 4 chars visible" for env list. Migration 76 (`83_markos_unverified_signups.sql` ↔ check actual migration) ships first-4-chars + ellipsis.
- **Decision (LOCKED):** Use **last-4** per UI-SPEC. Last-4 better protects against secret-prefix patterns (e.g., `sk_live_...` AWS-style prefixes leak the secret family).
- Update migration 76 OR client-side mask logic (whichever is the actual implementation site) to match: `mask(secret) = '*'.repeat(secret.length - 4) + secret.slice(-4)` for secrets ≥ 8 chars; full mask for < 8 chars
- Acceptance: `node -e "const m = require('./bin/lib/cli/env'); if(m.maskSecret('sk_live_abc123def') !== '*'.repeat(13) + '23de') process.exit(1);"` ← exact pattern test (write the assertion to match the implementation site once located)

### D-07: status pricing placeholder enforcement (Bonus Gap #3 — DECISION REQUIRED)
- **Conflict:** UI-SPEC §Pricing-sensitive output mandates `{{MARKOS_PRICING_ENGINE_PENDING}}` on plan-tier copy in `markos status`. `status.cjs:189` plan-tier line currently has no placeholder enforcement.
- **Decision (LOCKED):** Surface `{{MARKOS_PRICING_ENGINE_PENDING}}` in plan-tier output until approved PricingRecommendation exists.
- Logic: if `pricing_recommendation_id` from API response is null → render `Plan: {{MARKOS_PRICING_ENGINE_PENDING}}`. Else render approved tier name.
- Acceptance: `grep -c "MARKOS_PRICING_ENGINE_PENDING" bin/commands/status.cjs` returns >= 1; integration test mocks null pricing_recommendation_id and asserts placeholder in output

### D-08: winget distribution (deferred from P204 Wave 1)

- Manifest files: `markos.markos.installer.yaml`, `markos.markos.locale.en-US.yaml`, `markos.markos.yaml`

- Bump script: `scripts/distro/winget-bump.cjs` mirrors Scoop bump pattern
- Acceptance: manifest YAML lints clean via `winget validate --manifest <path>` (in CI matrix); install docs `docs/install/winget.md` exists

### D-09: apt distribution (deferred from P204 Wave 1)
