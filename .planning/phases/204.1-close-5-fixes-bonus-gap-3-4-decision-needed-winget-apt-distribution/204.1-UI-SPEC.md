## Per-Fix Acceptance Contracts

### D-01: Audit Trail Wiring

| Property | Contract |
|----------|---------|
| Integration point | `bin/markos.cjs` dispatch wrapper — one call site, all 11 commands gain audit |
| New file | `bin/lib/cli/audit.cjs` exports `recordCommandAudit({command, args_redacted, profile, exit_code})` |
| Args redaction | Strip `--password`, `--token`, `--key`, env values from logged args; whitelist remaining flags |
| Failure mode | Audit POST failure -> `stderr` `(verbose only)`: `[info] audit: POST failed (<status>)` — DOES NOT change exit code |
| Stream | Audit failure message: stderr only, DIM, `[info]` prefix (per DESIGN.md `[ok]/[err]/[warn]/[info]` rule) |
| Acceptance criteria | `grep -c "recordCommandAudit" bin/markos.cjs` = 1; `markos status` integration test verifies audit row created |

**Copywriting (audit failure, verbose mode only):**
```
[info] audit: POST failed (503). run continuing.
```
No newline prefix/suffix. DIM styling. Never shown in default or quiet mode.

### D-02: Spinner Primitive

| Property | Contract |
|----------|---------|
| New file | `bin/lib/cli/spinner.cjs` |
| Export | `createSpinner({label, stream='stderr'})` -> `{tick, stop}` |
| UTF-8 frames | `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏` at 80ms interval — 10 frames, linear cycle |
| ASCII frames | `\|/-\\` at 100ms interval — 4 frames, linear cycle |
| Format | `<frame> <label>` to stderr. Frame: DIM. Label: plain. No trailing punctuation. |
| Motion register | kernel-pulse only: `ease-linear`, 2s effective period (80ms * 25 ticks = 2s), no spring, no overshoot |
| Suppression gates | Suppressed when: `shouldUseJson` OR `!process.stdout.isTTY` OR `process.env.NO_COLOR` is set OR `--quiet` flag active |
| ASCII gate | Falls to ASCII frames when `shouldUseUnicode()` returns false (see D-04 pickGlyphs gate) |
| Wire-in targets | `keys list`, `whoami`, `env list`, `env pull`, `env push`, `status`, `run` (pre-SSE-attach), `login` (poll) |
| Stop behavior | `stop(label?)` clears the spinner line; if label provided, prints `[ok] <label>` (GREEN) to stderr |
| Forbidden | Spring/ease-out/bounce easing. Decorative spinners on fast (<200ms) operations. Emitting to stdout. |

**Exemplar output (UTF-8 TTY):**
```
⠋ fetching keys...
```
**Exemplar output (ASCII TTY, LANG=C):**
```
| fetching keys...
```
**Exemplar — suppressed (non-TTY / --json):** no output.

### D-03: stdout / stderr Stream Split

| Property | Contract |
|----------|---------|
| Rule | stdout = data (table rows, JSON). stderr = humans (status, errors, spinner, progress, confirmations). |
| Fix sites | `bin/commands/keys.cjs:246` (`Key … revoked.`) -> stderr |
| | `bin/commands/env.cjs:242` (Pulled) -> stderr |
| | `bin/commands/env.cjs:306` (Pushed) -> stderr |
| | `bin/commands/env.cjs:378` (Deleted) -> stderr |
| | `bin/commands/login.cjs:49` (`Logged in …`) -> stderr |
| JSON mode | JSON output stays on stdout regardless. Success confirmation in JSON mode: included in the JSON envelope, not emitted separately. |
| Acceptance | `grep -c "process.stdout.write"` for the specific success lines in those files = 0; redirected integration test verifies success messages NOT in stdout in human mode |

**Success line format (all 5 sites, after move to stderr):**
```
[ok] <past-tense verb> <object>
```
Examples:
- `[ok] revoked key_abc123`
- `[ok] pulled 4 env vars -> .markos-local/.env`
- `[ok] pushed 4 env vars`
- `[ok] deleted DATABASE_URL`
- `[ok] logged in as ten_abc123`

GREEN color via ANSI when color enabled. Plain `[ok]` prefix when NO_COLOR.

### D-04: ASCII Fallback for Box Characters

| Property | Contract |
|----------|---------|
| New helper | `pickGlyphs()` in `bin/lib/cli/output.cjs` |
| Returns | `{topLeft, topRight, bottomLeft, bottomRight, horiz, vert, junction}` |
| UTF-8 tier glyphs | `┌` `┐` `└` `┘` `─` `│` `┼` |
| ASCII tier glyphs | `+` `+` `+` `+` `-` `\|` `+` |
| Detection logic | UTF-8 tier when: `(process.env.LANG?.includes('UTF-8') OR process.env.LC_ALL?.includes('UTF-8')) OR (process.platform !== 'win32' AND process.stdout.isTTY)`. Else: ASCII tier. |
| Fix sites | `bin/lib/cli/errors.cjs:56-58` (error box border) |
| | `bin/commands/whoami.cjs:80,86,89` (status box) |
| | `bin/commands/status.cjs:178,181,183` (section dividers) |
| | `bin/commands/doctor.cjs:52,70` (check-list border) |
| Acceptance | `grep -c "[┌│└─]" bin/lib/cli/errors.cjs bin/commands/{whoami,status,doctor}.cjs` = 0 after fix; test asserts ASCII output when `LANG=C` |

**Box exemplar — UTF-8:**
```
┌── UNAUTHORIZED
│ API key expired. Set a new key.
└── hint: run 'markos login'
```

**Box exemplar — ASCII (LANG=C or non-TTY):**
```
+-- UNAUTHORIZED
| API key expired. Set a new key.
+-- hint: run 'markos login'
```

### D-05: <60-Column Width Breakpoint

| Property | Contract |
|----------|---------|
| Detection | `process.stdout.columns ?? 80` |
| Threshold | `< 60` columns -> degraded layout |
| Degraded layout | Vertical key:value stream — one property per line, blank line between records, no box/table borders |
| Standard layout | `>= 60` columns -> existing `renderTable` horizontal layout (unchanged) |
| Transition | No animation. Instant layout switch based on column count at render time. |
| Fix site | `bin/lib/cli/output.cjs::renderTable` |
| Acceptance | `renderTable` unit test: columns=50 -> vertical output; columns=120 -> horizontal; `markos keys list` integration: `COLUMNS=50` shell renders vertical |

**Degraded layout exemplar (COLUMNS=50, `markos keys list`):**
```
name:        prod-deploy
fingerprint: sha256:4a7bc123
created:     2026-04-01T09:00:00Z
status:      active

name:        ci-runner
fingerprint: sha256:9f2de456
created:     2026-04-10T12:00:00Z
status:      active
```

**Standard layout exemplar (COLUMNS=120):**
```
NAME         FINGERPRINT        CREATED               STATUS

prod-deploy  sha256:4a7bc123    2026-04-01T09:00:00Z  active
ci-runner    sha256:9f2de456    2026-04-10T12:00:00Z  active
```

### D-06: env Mask Rule (last-4, locked)

| Property | Contract |
|----------|---------|
| Rule | Show last 4 chars only for secrets >= 8 chars. Full mask for < 8 chars. |
| Formula | `'*'.repeat(secret.length - 4) + secret.slice(-4)` |
| Rationale | Last-4 protects against secret-family-prefix leakage (`sk_live_`, `AKIA`, `ghp_` prefixes expose the secret family if first-4 shown) |
| Display format | `VAR_NAME=*************ef4a` (stars + last 4) |
| Fix sites | Client-side mask logic at env list render path; migration 76 if it contains mask logic |
| Acceptance | `node -e "const m = require('./bin/lib/cli/env'); ..."` last-4 assertion; `markos env list` shows last-4 not first-4 |

**Visual exemplar (`markos env list`, masked column):**
```
KEY           VALUE
------------  --------------------
DATABASE_URL  *****************5432
LINEAR_KEY    ***********abc4
OPENAI_KEY    ***********ef9a
```

## Copywriting Contract (Delta from 204-UI-SPEC)

All 204-UI-SPEC §Copywriting rules apply unchanged (caveman mode, no emoji, lowercase hints,
SCREAMING_SNAKE_CASE codes, sentence-case messages, banned lexicon enforced). The following
are additions specific to 204.1:

| Element | Copy |
|---------|------|
| Audit failure (verbose) | `[info] audit: POST failed (<status>). run continuing.` |
| Spinner label format | `<verb-ing> <object>...` (e.g. `fetching keys...`, `polling for auth...`, `submitting run...`) |
| Stream-corrected success (all 5 sites) | `[ok] <past-tense verb> <object>` — GREEN when color enabled |
| Pricing placeholder (null tier) | `• plan: {{MARKOS_PRICING_ENGINE_PENDING}}` |
| Pricing placeholder (approved tier) | `✓ plan: <tier-name>` |
| Narrow-layout label separator | `<KEY>:` followed by two spaces then value (consistent with `pad-cell` = 2 spaces) |

**Banned in this phase (same as 204-UI-SPEC):** emoji, `!`, marketing adjectives, `just` as softener.
