# Phase 204.1 Plan 02 Summary

Plan `204.1-02` closes the P1 CLI feedback and stream-splitting gaps by adding a shared spinner primitive and moving human-mode status prose off stdout. The result is that long-lived fetches now show progress in interactive terminals, while pipe consumers keep receiving only structured result output on stdout.

## What changed

- Added `bin/lib/cli/spinner.cjs`.
  - Exports `createSpinner`, `isUtf8Locale`, `FRAMES_UTF8`, `FRAMES_ASCII`, and `CLEAR_LINE`.
  - Uses the shared `shouldUseJson()` / `shouldUseColor()` suppression rules and always renders to stderr.
  - Chooses UTF-8 braille frames in UTF-8 terminals and ASCII `\|/-` frames in non-UTF-8 locales.
- Updated CLI commands.
  - `bin/commands/keys.cjs` now shows `fetching keys` during `keys list`, and revoke success/abort prose now writes to stderr.
  - `bin/commands/whoami.cjs` now shows `fetching identity` during the whoami fetch.
  - `bin/commands/env.cjs` now shows `fetching env`, `pulling env`, and `pushing env`, and the pull/push/delete human-mode confirmations now write to stderr.
  - `bin/commands/login.cjs` now shows `polling for approval`, and the device-flow `Go to:`, `Code:`, success, and abort lines now write to stderr.
  - `bin/commands/run.cjs` now shows `submitting run`, and the fire-and-forget submit hints now write to stderr.
- Added `test/cli/204.1/spinner.test.js` and `test/cli/204.1/stream-separation.test.js`.
  - Covers locale tiers, suppression cases, stop-line clearing, stderr/stdout separation for keys/env/login/run, and human-mode prompt-decline paths.

## Repo-reality notes

- The original plan packet mentioned wiring `status` in this slice, but `status.cjs` also needed the pricing-placeholder change from `204.1-04`. I kept the status spinner deferred until Plan 04 so the file only needed one follow-up pass.
- `login.cjs` was easier to stabilize as a clean file rewrite than as a string-by-string patch because the existing source contained mojibake-heavy literals that made targeted patching brittle.
