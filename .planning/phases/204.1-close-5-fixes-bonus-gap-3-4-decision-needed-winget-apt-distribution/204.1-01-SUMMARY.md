# Phase 204.1 Plan 01 Summary

Plan `204.1-01` closes the P0 CLI audit-trail gap from the Phase 204 review by adding a shared CLI-side audit primitive and wiring it once at the real operator dispatch boundary in `bin/install.cjs`. The wrapper now records a redacted audit envelope after every operator command completes, including commands that terminate via `process.exit()`, while preserving the original exit behavior and swallowing all audit-side failures.

## What changed

- Added `bin/lib/cli/audit.cjs`.
  - Exports `redactArgs`, `redactPositional`, and `recordCommandAudit`.
  - Redacts `token`, `password`, `key`, explicit `--token|--password|--key` positional pairs, and `markos env put|set KEY VALUE` secret payloads before serialization.
  - Posts the exact envelope `{ command, args_redacted, profile, exit_code, duration_ms, cli_version }` to `POST /api/tenant/cli-audit`.
  - Treats missing auth, transport errors, 401/403s, and the planned `404` endpoint-not-yet-present case as best-effort no-ops, with stderr output only in `--debug` mode.
- Updated `bin/install.cjs`.
  - Added `dispatchWithAudit(...)` plus a small `CommandExitSignal` shim so the dispatcher can intercept command-local `process.exit(...)`, write audit data, then resume the original exit path.
  - Replaced the 10 operator dispatch sites (`doctor`, `env`, `eval`, `init`, `keys`, `login`, `plan`, `run`, `status`, `whoami`) with the shared audit wrapper.
  - Updated the top-level `run().catch(...)` handler to honor `error.exitCode` when an audited command throws a real error instead of calling `process.exit(...)`.
- Added `test/cli/204.1/audit-trail.test.js`.
  - Covers the two required redaction paths, non-secret flag preservation, 202 success behavior, 404 fallback, transport-failure tolerance, exact envelope keys, and two dispatcher integration cases proving audit still runs for intercepted `process.exit(...)` and thrown command failures.

## Repo-reality notes

- The plan packet referenced `bin/markos.cjs` and a `request(...)` helper, but this repository's real dispatcher is `bin/install.cjs` and the current HTTP primitive is `bin/lib/cli/http.cjs`. The implementation integrates at the actual dispatch site instead of inventing a second entrypoint.
- Several operator commands call `process.exit(...)` directly. A plain `try/finally` wrapper would have missed those paths in real usage, so the dispatcher now temporarily intercepts `process.exit(...)` long enough to record the audit row first.
- The current phase only ships the CLI-side emitter. If the server endpoint is not present yet, the agreed `404` fallback now degrades cleanly with the debug-only `[audit] endpoint not yet available` notice.

## Outcome

Phase 204's audit contract is now observable at the CLI boundary instead of living only in review notes. Every operator command flows through one audited dispatcher, secrets are scrubbed before the envelope is built, and audit transport problems cannot change the user-visible exit code or leak onto stdout.
