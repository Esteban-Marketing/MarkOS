## RESEARCH.md Risk Mitigations

### R2 (Storybook decorator clobbering DESIGN.md tokens) — MITIGATED

**Before:** `.storybook/preview.tsx` ran `ThemeProvider` decorator on every story by default, wrapping `<Story />` in `<div style={{ ... backgroundColor: tokens["color.bg.canvas"], ... }}>` which hard-coded the legacy white-paper brand-pack canvas at the wrapper level — clobbering any DESIGN.md token cascade from `app/globals.css` because the wrapper's inline `backgroundColor` had higher specificity than `body { background: var(--color-surface) }`.

**After:**
1. `import "../app/globals.css"` at the top of preview.tsx so `:root` of the storybook iframe inherits `--color-surface`, `--color-primary`, `--font-mono`, `--w-container`, `--w-sidebar`, `.c-*` primitives, and base typography from DESIGN.md.
2. ThemeProvider early-returns `<Story />` directly when `themeVariant !== "legacy" && themeVariant !== "white-label"`. Only opt-in legacy and white-label stories now render inside the brand-pack wrapper. All 18 existing stories that previously got the wrapper by default will now render against DESIGN.md tokens — that is the regression we are correcting.
3. The 5 new `RotationGraceBanner` stories explicitly do NOT set `parameters.theme: "legacy"`, so they consume DESIGN.md tokens (Kernel Black surface, Protocol Mint links, `var(--color-warning)` text on `rgb(255 184 0 / 0.12)` background per Plan-02's banner module).
