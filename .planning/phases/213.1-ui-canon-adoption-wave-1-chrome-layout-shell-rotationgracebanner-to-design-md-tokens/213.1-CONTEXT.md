<specifics>
## Specific Ideas

- **Brand voice anchor:** UI-SPEC.md AC#5 strips "Please" softener from the denied-page body and revises to "Unable to establish tenant context. Sign in again." Match `obsidian/brain/Brand Stance.md` voice — developer-native, AI-first, quietly confident, no exclamation marks.
- **Bracketed-glyph state coding:** UI-SPEC.md adds `[warn]` / `[err]` prefix to `<strong>` text in `RotationGraceBanner.tsx`. This is the canonical pattern from DESIGN.md "Color blindness" rule (state color is never the only signal). Phase 203 banner test updates in lockstep — same plan, same commit if possible.
- **`tenantPill` ≈ `c-chip--mint` + tenant ID inside `c-chip-protocol`:** UI-SPEC pairs the mint chip with a squared protocol chip, so the tenant marker reads `[live] [tenant_id_xyz]` visually — signature MarkOS pattern from DESIGN.md.
- **Status dot in RotationGraceBanner:** the kernel-pulse `.c-status-dot--live` (2s opacity pulse) is the only signature animation; UI-SPEC reserves it for "genuinely live" state. Banner uses the static `.c-status-dot--{idle,error}` variants (no pulse) for severity dots — pulse only on the live tenant indicator in the shell sidebar.
- **DESIGN.md amendments (non-blocking, future PR before 213.2):** UI-SPEC filed 5 proposals. None block this phase. Recommendation D from gsd-ui-checker: schedule the amendment PR before 213.2 plan-phase so Wave 2 can compose `.c-notice--warning` directly instead of inlining the formula.
