- **Found during:** Task 3 verification (`npm run build-storybook`)

Bisect confirmed the storybook build was failing for this reason regardless of plan files (legacy + new module.css both triggered it once Vite moved to a deeper PostCSS-load code path; the legacy config in early bisect attempts only "worked" because of stale `storybook-static/` cache).
- **Fix:** Replaced `app/**/*.css` with safe phrasing `any app/(...)/.css` in the JSDoc comment. Verified `node -e "import('./postcss.config.mjs').then(m => console.log(m.default))"` now loads cleanly.
- **Files modified:** `postcss.config.mjs`
- **Commit:** `4f82b52`
- **Side effect:** The file was untracked in git prior to this commit (CONTEXT.md had noted it was "just added in this milestone" but never staged); it is now committed so the storybook build is reproducible from a clean checkout.

**3. [Rule 3 – Blocking issue] Missing `tailwindcss` + `@tailwindcss/postcss` packages in `node_modules`**
