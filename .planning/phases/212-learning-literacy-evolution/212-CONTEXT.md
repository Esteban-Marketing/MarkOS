# Phase 212 Context - Learning and Literacy Evolution

**Status:** Reframed for executable planning on 2026-04-27.

## Why this phase exists

Phase 211 closes the first operating loop. Phase 212 decides whether MarkOS can learn from that loop in a governed way: tenant-scoped where it must stay local, anonymized where it is aggregated, reviewed before it becomes central literacy, and action-producing instead of passive archive drift.

This phase is not a second owner of evidence, compliance, task routing, or loop measurement. It consumes those systems and turns their outputs into learning objects.

## Required phase shape

1. Run a Wave 0.5 upstream preflight and architecture lock for evidence, measurement, privacy, and task-handoff readiness.
2. Define `ArtifactPerformanceLog` with expected outcome, actual outcome, evidence, attribution, degraded states, and lesson outputs.
3. Define `TenantOverlay` with confidence, provenance, expiry, suppression, and review cadence.
4. Define `LiteracyUpdateCandidate` and an admin-reviewed promotion path for central literacy updates.
5. Define anonymization, aggregation, consent, sample-size, and confidence thresholds for cross-tenant learning.
6. Define `LearningRecommendation` records that create tasks or strategy refresh work instead of silent prompt drift.
7. Preserve doc 17 future learning categories for PLG, ABM, community, PR, partnerships, events, and developer marketing as compatibility metadata only.

## Non-negotiables

- No tenant identifiers or PII in cross-tenant learning.
- No central literacy promotion without anonymization, sample-size, confidence, and admin review gates.
- No tenant overlay without provenance, evidence linkage, expiry, and review posture.
- No experiment or recommendation learning that bypasses EvidenceMap, approval, or task visibility.
- No "AI learned this" claim without measurable outcome and a source trail.

## Done means

GSD can point to a named performance-log contract, tenant-overlay policy, promotion-review gate, anonymization policy, and learning-recommendation handoff that all stay privacy-safe, evidence-aware, and operationally reviewable.
