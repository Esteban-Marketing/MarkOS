# Phase 64.2: GSD Surface and Instruction Alignment - Context

**Gathered:** 2026-04-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Align the instruction-target policy, generators, templates, workflows, and agent wording across the refreshed `.github` GSD framework and the MarkOS-localized `.claude` surface so shared GSD workflows and local Claude runtime behavior stay aligned. This phase preserves the active localized `.claude` runtime and dispatch layer, adds managed root `copilot-instructions.md` support alongside root `CLAUDE.md`, and stops before manifest refresh, end-to-end verification, or broader customization-boundary closure that belongs in 64.3.

</domain>

<decisions>
## Implementation Decisions

Instruction policy decisions:
- **D-01:** 64.2 should implement dual-root instruction support.
- **D-02:** Root `CLAUDE.md` remains the live MarkOS-local Claude contract.
- **D-03:** Root `copilot-instructions.md` should be added and managed as the project instruction contract required by the refreshed `.github` shared GSD surfaces.

Ownership boundary decisions:
- **D-04:** Treat `.github/**` as the canonical shared GSD framework surface for agents, skills, workflows, templates, and tooling content.
- **D-05:** Keep `.claude/**` as the localized Claude runtime and dispatch layer.
- **D-06:** 64.2 must align behavior, targets, and wording across those surfaces without collapsing `.claude` path locality into `.github`.

Alignment depth decisions:
- **D-07:** 64.2 should align policy surfaces only: root instruction targets, generator behavior, templates, workflows, and agent wording.
- **D-08:** 64.2 should not pull manifest refresh, end-to-end verification, or final customization-boundary documentation forward from 64.3.
- **D-09:** The missing `GSD-ALIGN-04` and `GSD-ALIGN-05` entries in `.planning/REQUIREMENTS.md` are a documentation gap, not a planning blocker.

Generator and template decisions:
- **D-10:** Use one shared managed policy to emit two deliberate root outputs: `CLAUDE.md` and `copilot-instructions.md`.
- **D-11:** Do not rely on manual parallel editing to keep the two root instruction files aligned.
- **D-12:** Do not treat `.github/copilot-instructions.md` as a substitute for root `./copilot-instructions.md`.

Localized runtime decisions:
- **D-13:** `.claude/skills/**` should stay pointed at localized `.claude/get-shit-done/**` workflow paths in 64.2.
- **D-14:** 64.2 is a policy-alignment phase, not a runtime migration phase.

the agent's Discretion:

- Which workflow, template, and agent wording changes are necessary to encode the dual-root policy cleanly while preserving localized `.claude` behavior.
- Whether to patch `.planning/REQUIREMENTS.md` during 64.2 if it can be done as a narrow documentation cleanup without expanding phase scope.

</decisions>
