# Plan 16-01: Documentation Enrichment

This plan outlines the strategy to optimize the MGSD documentation for both human maintainers and autonomous LLM agents.

## Proposed Changes

### 1. Codebase Inline Enrichment
- **Critical Path**: `onboarding/backend/agents/orchestrator.cjs`, `onboarding/backend/agents/llm-adapter.cjs`.
- **Action**: Implement "LLM Context Blocks" (`/** @llm_context ... */`) that explicitly describe the *intent* and *failure boundaries* of each function to help future agents debug them more effectively.

### 2. Prompt Instruction Precision
- **Target**: All `.agent/prompts/*.md` specialized agents.
- **Action**: 
  - Add `## FAILURE MODE AWARENESS`: Describe what "bad" looks like for this role.
  - Add `## CONTEXT RELAY`: Link to specific `.protocol-lore/` and `MIR/` files that are authoritative for this agent's logic.

### 3. Protocol Deep-Linking & Lore Hardening
- **File**: `.protocol-lore/INDEX.md` and `.protocol-lore/CODEBASE-MAP.md`.
- **Action**: Cross-link the high-level lore directly to the implementing source files using `(file:///...)` URIs.

### 4. Gold-Standard Behavioral Examples
- **NEW**: `.agent/prompts/examples/GOLD-STANDARD.md`.
- **Action**: Create a single, rich reference file containing 3-4 "perfect" interaction logs between a user and various MGSD agents.

## Verification Plan
- **Automated**: Run `gsd-map-codebase` (if available) to verify all components are registered and documented.
- **Manual**: Trigger a simulated "Documentation Audit" session with an LLM agent to verify context retrieval efficiency.
