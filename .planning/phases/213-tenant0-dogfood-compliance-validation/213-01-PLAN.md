# Plan 213-01 - Tenant 0 Workspace, Brand Pack, Connector List, and Data Policy

**Phase:** 213 - Tenant 0 Dogfood and Compliance Validation  
**Status:** Planned  
**Requirements:** T0-01, T0-02, CONN-01..06, QA-01..15

## Objective

Create the internal MarkOS tenant setup needed to run MarkOS on itself.

## Scope

- Define Tenant 0 org, tenant, owner roles, brand pack, approval policy, connector list, and data-source policy.
- Identify allowed public proof sources and private-only sources.
- Link setup to Pricing Engine and evidence policy.

## Acceptance Criteria

- Tenant 0 has real owner, brand, connector, approval, and evidence configuration.
- Private data cannot leak into public proof.
- Missing connectors create recovery tasks instead of fake proof.

## Verification

- Setup checklist maps every Tenant 0 field to a source and owner.
