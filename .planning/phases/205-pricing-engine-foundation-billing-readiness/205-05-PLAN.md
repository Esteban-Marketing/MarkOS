# Plan 205-05 — Pricing API + 6 MCP tools + content guardrail

## Objective

Ship the 4 read-only HTTP routes and 6 MCP tools that expose pricing intelligence (competitive matrix, pricing position, competitor pricing, pricing floor, alerts, recommendations) to agents and surfaces. Ship the content guardrail any pricing-sensitive copy generator (Phase 211) MUST call before emitting dollar figures. This is the **read-before-write gate**: Plan 205-07 (Stripe mutations) MUST NOT merge until 205-05 tests are green.

## Current code path this migrates

- `lib/markos/mcp/cost-table.cjs` — retains its static caps as fallback; gains `resolveTenantDailyCap(tenantId, supabase)` that reads approved `packaging_change` recommendations.
- `lib/markos/mcp/cost-meter.cjs` — consumes `resolveTenantDailyCap` when available; no schema change.
- `lib/markos/mcp/pipeline.cjs` (assumed existing) — MCP tool registry point; 6 new tools registered here.
- `api/billing/*` — unchanged by this plan (205-06 rewrites tenant-summary; 205-07 adds linkage to recommendations).

## must_haves

- 4 read-only API routes live under `api/tenant/pricing/`: matrix, position, floor, alerts/subscribe (SSE).
- 6 MCP tools registered with bound names, input/output schemas, budget admission via `cost-meter.cjs`: `pricing.matrix.read`, `pricing.position.read`, `pricing.competitor.read`, `pricing.floor.read`, `pricing.alerts.read`, `pricing.recommendation.read`.
- ALL MCP tools read-only in this plan. Mutating tools land in 205-07.
- `resolveTenantDailyCap(tenantId)` in `cost-table.cjs` reads approved `packaging_change` recommendations via Plan 205-04 store; falls back to `FREE_TIER_CAP_CENTS`/`PAID_TIER_CAP_CENTS` if none.
- `lib/markos/content/pricing-guardrail.ts` exports `assertPricingMcpTappedOrPlaceholder(ctx)`: must have queried at least one pricing MCP tool in the current AgentRun, OR emit `{{MARKOS_PRICING_ENGINE_PENDING}}`. Throws `PRICING_CONTENT_GUARDRAIL_VIOLATED` otherwise.
- Placeholder-residue test (`test/vitest/pricing/placeholder-residue.test.ts`) fails if any file under `app/(markos)/settings/billing/**` or `api/billing/**` contains hard-coded tier name or dollar amount outside the approved exclusions.
- F-116 YAML populated with 4 HTTP operations + 6 MCP tool schemas.
- No `stripe` import (read-only plan).
