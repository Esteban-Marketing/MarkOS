# Plan 205-05 - Pricing API and MCP Surface

**Phase:** 205 - Pricing Engine Foundation + Billing Readiness  
**Status:** Planned

## Objective

Expose Pricing Engine context through tenant-scoped APIs and MCP tools so agents can stop guessing about pricing.

## Scope

- API families for pricing intelligence, competitive matrix, cost model, pricing floor, recommendations, tests, watch list, and alerts.
- MCP tools: `get_competitive_pricing_matrix`, `get_pricing_position`, `get_competitor_pricing`, `get_pricing_floor`, `get_recent_pricing_alerts`, `get_pricing_recommendation`.
- Audit and cost visibility for pricing tool calls.
- Guardrail: pricing-sensitive content must query pricing MCP tools or use `{{MARKOS_PRICING_ENGINE_PENDING}}`.

## Acceptance Criteria

- Pricing MCP tools are registered, schema-validated, tenant-scoped, and tested.
- API contracts regenerate OpenAPI without collisions.
- Pricing-sensitive agent tests fail when neither Pricing Engine context nor placeholder is present.
