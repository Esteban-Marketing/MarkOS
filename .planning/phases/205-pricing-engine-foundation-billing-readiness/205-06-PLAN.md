# Plan 205-06 - Pricing Engine Operator UI

**Phase:** 205 - Pricing Engine Foundation + Billing Readiness  
**Status:** Planned

## Objective

Give operators a decision-first Pricing Engine UI that explains cost, competition, recommendation quality, and approval state.

## Scope

- Pricing dashboard.
- Competitive matrix with evidence.
- Cost model wizard.
- Recommendation cards with options, projected impact, risk, reversibility, supporting data, frameworks, assumptions, and decision controls.
- Price tests, watch list, and alerts.
- Billing/settings UI compatibility with engine-owned placeholders and approved recommendations.

## Acceptance Criteria

- No active UI copy presents hard-coded MarkOS public prices as canonical.
- Every recommendation card shows cost context, evidence, assumptions, and approval state.
- Price-test activation is visibly gated.
- Existing billing settings fixtures are either replaced by engine-backed values or labeled as placeholder/demo state.
