# Plan 205-06 — Pricing Engine Operator UI + Billing Placeholder Rewrite

## Objective

Ship 8 operator pages (dashboard, matrix, cost-model wizard, recommendations index + detail, price tests, watch list, alerts) + 4 reusable components, rewrite `app/(markos)/settings/billing/page-shell.tsx` and `api/billing/tenant-summary.js` to be placeholder-safe (no hard-coded `Growth Monthly` / `$150.00`), and add the pricing-decision lineage panel on admin billing. Enforce grep-residue gate: `app/(markos)/settings/billing/**` and `api/billing/**` contain zero hard-coded tier names or dollar amounts after this plan lands.

**This plan is the SOLE UI plan in Phase 205.** All deliverables author to DESIGN.md v1.1.0 and `styles/components.css` v1.1.0 from day-1 (no retrofit). Every visual decision MUST cite a `var(--*)` token from `app/tokens.css` and compose `.c-*` primitives from `styles/components.css`. The full visual contract — 40 ACs across PR-*, PB-*, AB-EXT-*, CM-* — is enumerated in `.planning/phases/205-pricing-engine-foundation-billing-readiness/205-UI-SPEC.md`. Each task below cross-references the relevant ACs in its `<acceptance_criteria>` block.

## Current code path this migrates

- `app/(markos)/settings/billing/page-shell.tsx` — currently renders `"Growth Monthly"` title + `usageRows` + `invoices` + `$150.00` total. Rewritten to prop-drive from `tenant-summary.js`. Already adopted DESIGN.md v1.1.0 in Phase 213.3 — this plan extends it placeholder-safe.
- `app/(markos)/admin/billing/page.tsx` — operator view; gains a collapsible "Pricing decision lineage" panel fed by linked `PricingRecommendation.recommendation_id`. Already adopted DESIGN.md v1.1.0 in Phase 213.4 — additive extension only.
- `api/billing/tenant-summary.js` — currently returns static sample. Rewritten to call `readTenantRecommendations` (Plan 205-05) + `readTenantPricingFloor`; returns placeholder when no approved recommendation.
- Storybook fixtures — reshaped to `placeholder_state | approved_state | hold_state` variants instead of a baked `Growth Monthly` variant.

## must_haves

- 8 pages live under `app/(markos)/pricing/`.
- 4 reusable components under `components/markos/pricing/`.
- `page-shell.tsx` and `tenant-summary.js` emit `{{MARKOS_PRICING_ENGINE_PENDING}}` when no approved recommendation; visibly labeled as placeholder state (uses `PlaceholderBanner` composing `.c-notice c-notice--info`).
- Pricing decision lineage panel visible on admin billing page; chain IDs render via `.c-chip-protocol`.
- F-117 YAML populated with UI-facing read contracts for dashboard, matrix, cards.
- Grep-residue gate: `grep -rE '(Growth Monthly|\\bStarter\\b|\\bProfessional\\b|\\$[0-9]+)' app/\\(markos\\)/settings/billing/ api/billing/ lib/markos/billing/ app/\\(markos\\)/pricing/ components/markos/pricing/` returns 0 matches outside allowed exclusions.
- DESIGN.md v1.1.0 token-only authoring: zero inline hex in any `.module.css` or TSX inline style across all 205-06 deliverables.
- Banned-lexicon = 0 across all copy (per CLAUDE.md banned list + UI-SPEC Copywriting Contract).
- Bracketed-glyph state coding (`[ok]`/`[warn]`/`[err]`/`[info]`) on every state signal — color is never the only signal.
- `.c-card--feature` is PROHIBITED in this phase (reserved for 404-workspace hero + 213.5 marketing).
- Tables use vanilla `<table>` semantics — `.c-table` primitive deferred to Phase 214+.
- Mint-as-text via `var(--color-primary-text)` (PlaceholderBanner CTA, `.c-button--tertiary` text, `.c-chip-protocol` text).
- 5 Playwright specs green.
- Storybook variants per `RecommendationState` value.
- Accessibility: `axe-playwright` run green on all 8 pages (QA-01).
- No `stripe` import in any file in this plan.
