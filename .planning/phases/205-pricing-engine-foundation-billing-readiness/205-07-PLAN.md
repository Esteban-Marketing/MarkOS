# Plan 205-07 — Billing/Stripe Handoff from Approved Pricing Recommendations

## Objective

Ship `createStripePriceFromApprovedRecommendation` + `reconcileApprovedRecommendationToStripe` as the SOLE path from Pricing Engine → Stripe. Refuse to mutate Stripe for any recommendation where `state !== 'approved'` OR `decision.actor_id === null`. Preserve active-subscriber grace period when `affects_existing_subscribers !== true`. Import `stripe@^17` here and ONLY here. Retarget `pricing-catalog.ts` as a read-through adapter onto approved recommendation snapshots. Register 3 mutating MCP tools (approve + activate + sync) wired to the Plan 205-04 approval bridge.

This plan is gated behind Plan 205-05 green tests per RESEARCH §Common Pitfalls #10.

## Current code path this migrates

- `lib/markos/billing/stripe-sync.ts` — currently preview-only stubs. Gains real Stripe SDK calls + 2 new exports.
- `lib/markos/billing/pricing-catalog.ts` — retargeted from strict static dict to a read-through adapter querying `markos_pricing_recommendations` (state=approved) with `{{MARKOS_PRICING_ENGINE_PENDING}}` fallback. Existing callers survive.
- `lib/markos/billing/invoice-line-items.ts` — gains nullable `pricing_recommendation_id` + `pricing_engine_context_ref` on line-item shape; operator invoice renders them, tenant-facing invoice does NOT.
- `lib/markos/billing/usage-normalizer.ts` — gains `normalizeAgentRunCost(run, pricingContext?)` that accepts optional `PricingEngineContext`.
- `api/billing/operator-reconciliation.js`, `api/billing/holds.js` — add optional `linked_pricing_recommendation_id` field in payload.
- `package.json` — adds `"stripe": "^17.0.0"` dependency.

## must_haves

- `createStripePriceFromApprovedRecommendation(stripe, rec)` creates `stripe.billing.meters` (metered) and/or `stripe.prices` (licensed) with correct metadata; uses 2024 GA API (NO legacy `UsageRecord`).
- Refuses with `PRICING_RECOMMENDATION_NOT_APPROVED` if `rec.state !== 'approved'`.
- Refuses with `PRICING_RECOMMENDATION_NO_DECISION_ACTOR` if `decision.actor_id === null`.
- Refuses with `PRICING_RECOMMENDATION_NO_CHOSEN_OPTION` if `chosen_option_id` null or not found.
- Active-subscriber grace: if `affects_existing_subscribers !== true`, Stripe price applies to new subscriptions only; existing subscribers stay on previous price (snapshot via `superseded_pricing_snapshot_id`).
- Secondary approval required when `affects_existing_subscribers === true` (inherited from 205-04 state machine).
- `reconcileApprovedRecommendationToStripe` idempotent via `AgentSideEffect` with `effect_type='price.change'` and 207-01 unique key.
- `pricing-catalog.ts` retains public API (`buildPricingSnapshot`, `resolvePricingCatalogUnitPrice`) but internally reads approved recommendations.
- 3 mutating MCP tools registered: `pricing.recommendation.approve`, `pricing.test.activate`, `pricing.billing.sync`.
- Invoice UI rejects fields sourced from `pricing_subject='competitor'` (per RESEARCH §Common Pitfalls #2).
- F-118 YAML populated.
- Stripe SDK imported ONLY in this plan's files: `lib/markos/billing/stripe-sync.ts`, `api/tenant/pricing/billing/sync.js`, `api/tenant/pricing/billing/reconcile.js`, `api/tenant/pricing/recommendations/[recommendation_id]/sync.js`, and `test/vitest/pricing/stripe-*.test.ts`.
- `stripe@^17` added to `package.json`.
