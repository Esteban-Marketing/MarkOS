# Plan 205-08 ã Tenant 0 Pricing Dogfood

## Objective

Seed MarkOS Tenant 0 (SaaS business_type) with a real cost model, real competitor watch list, and a first approved `PricingRecommendation` ã then verify the full approval ladder, Stripe-internal price creation, `pricing_engine_context` write-through to Phase 207 AgentRun, and evidence-pack population. This is the **end-to-end dogfood gate**.

Hard constraint per RESEARCH Approved Decision #6: this plan creates a `PricingRecommendation` for Tenant 0 SaaS but MUST NOT unresolve `{{MARKOS_PRICING_ENGINE_PENDING}}` on public-facing surfaces (`docs/pricing/**`, `llms.txt`, landing page copy). Public unresolution is deferred to Phase 213 after SOC2-I evidence clears.

## Current code path this migrates

- No production code migration here ã this plan ships seed data + verification scripts + an explicit test that fails if any plan in 205 attempts to unresolve the public placeholder.
- Tenant 0 fixtures land under `.planning/tenant-zero/pricing/` (planning-tree only, not shipped to tenants).
- Seed scripts under `scripts/tenant-zero/` use the APIs shipped by 205-02..07 ã no new library code.

## must_haves

- Tenant 0 (SaaS) cost model seeded with real-ish SaaS values; `status='ready'`; pricing floor computed.
- Tenant 0 competitor watch list seeded with 3-5 entries (allowlisted, robots_ok, `fetch_cadence='manual_only'`).
- First `PricingRecommendation` for Tenant 0 created in `state='proposed'`, then moved through `proposed ã approved ã activated` via the approval bridge with a real approval token.
- Stripe price + meter created via `createStripePriceFromApprovedRecommendation` with `markos_public_tier_locked: 'true'` metadata.
- `pricing_engine_context` written on the AgentRun that wrapped the seed execution.
- Evidence-pack for Tenant 0 returns the approved recommendation in its `pricing` slot.
- Public docs + `docs/pricing/public-tier-placeholder.md` stay on `{{MARKOS_PRICING_ENGINE_PENDING}}`.
- Test `no-public-unresolution-in-205.test.ts` fails CI if any file under `docs/pricing/`, `public/llms.txt`, or the landing page template has the placeholder removed.
- Verification script `verify-pricing-dogfood.mjs` passes end-to-end.
- F-119 YAML populated.
- No `stripe` import outside deferred calls to Plan 205-07's library layer (seed scripts may import `lib/markos/billing/stripe-sync.ts` but do not add new Stripe call sites).
