# Plan 205-03 — Pricing Knowledge Store + Watch List + Alerts + PRC-06 stub

## Objective

Ship the `markos_pricing_knowledge` store, change-magnitude diff classifier, competitor watch list with allowlist + robots.txt admission, manual-entry path + tenant-initiated single-URL fetch (NO bulk crawl per RESEARCH Approved Decision #4), and the alert emit path. Register `MARKOS-AGT-PRC-06` (Competitive Price Watcher) as `experimental (monitor only)` per RESEARCH §PRC Agent Allocation.

## Current code path this migrates

- No prior code path — net-new Pricing Knowledge store.
- `lib/markos/mcp/cost-meter.cjs` — used by `single-url-fetch.ts` as the budget admission gate for per-URL fetch (keeps fetch under MCP cost cap).
- Scheduled bulk crawler is **explicitly NOT** shipped in this phase (approved deferral).

## must_haves

- `lib/markos/pricing/knowledge-store.ts` exports `insertKnowledge(input)`, `getKnowledge(knowledge_id)`, `listKnowledgeForSubject(tenant, subject_name, business_type)`, `admitByQualityThreshold(row)` implementing the `source_quality_score >= 0.7 AND extraction_confidence >= 0.6` gate.
- `lib/markos/pricing/knowledge-diff.ts` exports `classifyChangeMagnitude(prev, next): PricingChangeMagnitude` with a documented field-level diff rubric.
- `lib/markos/pricing/watch-list.ts` admission: cannot set `fetch_cadence != 'manual_only'` if `allowlisted=false` OR `robots_ok=false` (enforces DB CHECK + code-level guard).
- `lib/markos/pricing/single-url-fetch.ts` fetches one URL per call, respects robots.txt, records `source_quality_score` + `extraction_confidence`, stores raw payload only as content-addressed `raw_payload_ref` (SHA-256), rate-limited (min 30s between fetches per `watch_url`).
- `lib/markos/pricing/alerts.ts` emits `markos_pricing_alerts` rows on `major` or `minor` change_magnitude.
- `.agent/markos/agents/markos-prc-06-competitive-price-watcher.md` stub exists with `status: experimental (monitor only)`.
- F-114 YAML populated with 6 operations (+ error envelopes).
- Vitest suite covers change-magnitude classification + watch-list admission + single-URL fetch with mocked HTTP.

  <action>
    `lib/markos/pricing/single-url-fetch.ts`:
    - `fetchSingleUrl(opts: { tenantId, watch_id, actor_id, redis, mcpCostMeter })`: tenant-initiated only; NOT called from pg_cron. Sequence:
      1. Load watch row; if `allowlisted=false || robots_ok=false`, throw `PRICING_FETCH_ADMISSION_FAILED`.
      2. Rate-limit: check Redis key `pricing_fetch:<tenant>:<watch_id>`; if set within last 30s, throw `PRICING_FETCH_RATE_LIMITED`. Set key with TTL 30.
      3. Fetch `robots.txt` for the URL host; respect `Disallow` rules for the exact path. If disallowed, mark `watch.robots_ok=false` and throw `PRICING_FETCH_ROBOTS_DISALLOWED`.
      4. Call `mcpCostMeter.admit({ tenantId, estimated_usd_micro: DEFAULT_SINGLE_URL_FETCH_COST_MICRO })`; on reject, throw `PRICING_FETCH_BUDGET_EXCEEDED`.
      5. `fetch(watch_url, { redirect: 'follow', signal: AbortSignal.timeout(15000) })` with `User-Agent: MarkOS-PricingIntelligence/0.1 (+markos.dev)`.
      6. Compute `sha256Fingerprint` of response body; if equal to `watch.last_fingerprint`, return `{ changed: false }`.
      7. Store raw payload content-addressed under `raw_payload_ref` (hash only; NEVER expose raw to product surface — RESEARCH §Architecture Patterns Pattern 4).
      8. Return `{ changed: true, fingerprint, raw_payload_ref, fetched_at }`.

    - `DEFAULT_SINGLE_URL_FETCH_COST_MICRO = 5_000` constant (half a cent budgeted per fetch).
    - `SINGLE_URL_FETCH_RATE_LIMIT_MS = 30_000`.
    - Explicitly NOT exposing a bulk-scheduled variant. Any attempt to call `fetchManyUrls` or similar must fail at import time (export is deliberately absent).

    No `stripe` import.
  </action>
