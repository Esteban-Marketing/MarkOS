## Spacing Scale

Authoring rule: every `padding`, `margin`, `gap`, `inset` in scope MUST cite a `--space-*` token. No arbitrary px. Off-grid values are auto-FAIL.

| Token | Value | DESIGN.md citation | Usage in this phase |
|-------|-------|--------------------|---------------------|
| `--space-none` | 0 | `spacing.none` | Reset margins on `<h1>`, `<p>`, `<table>` |
| `--space-xxs` | 2px | `spacing.xxs` | Badge inner padding, state-chip group inner padding |
| `--space-xs` | 8px | `spacing.xs` | Table row `gap`, form field gap (label â†’ input â†’ help), inline icon-text adjacency, evidence badge group gap |
| `--space-sm` | 16px | `spacing.sm` | Card vertical rhythm, notice padding-block, table `th`/`td` padding-block, mobile horizontal page padding, wizard step padding |
| `--space-md` | 24px | `spacing.md` | Card padding (via `.c-card`), gap between major sections, modal padding, page horizontal padding at smâ€“md viewport |
| `--space-lg` | 32px | `spacing.lg` | Page horizontal padding â‰Ą md, evidence-rail padding, inter-section gap within pricing dashboard + recommendation detail |
| `--space-xl` | 48px | `spacing.xl` | Page vertical padding â‰Ą lg, pricing dashboard top padding |
| `--space-xxl` | 96px | `spacing.xxl` | Reserved â€” not used in this phase |

**Allowed exceptions (DESIGN.md documented):**
1. `1px` for hairline borders (`var(--color-border)`) â€” preferred over shadows on dark surfaces.
2. `2px` for focus ring width and offset (`var(--focus-ring-width)`, `var(--focus-ring-offset)`).
3. `4px` for `.c-notice` `border-inline-start` accent â€” composed via `.c-notice` primitive; modules never declare this directly.
4. `max-width: 1280px` for admin/pricing operator surfaces (`--w-container`).
5. Wizard step-indicator dot size `8px` â€” visual affordance for a progress indicator; sizing value not spacing; cited with rationale comment.
