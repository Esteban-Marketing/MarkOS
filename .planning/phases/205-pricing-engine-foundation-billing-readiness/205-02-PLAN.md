# Plan 205-02 - Tenant Cost Model and Pricing Floor

**Phase:** 205 - Pricing Engine Foundation + Billing Readiness  
**Status:** Planned

## Objective

Implement the minimum viable Tenant Cost Model so pricing recommendations cannot exist without margin context.

## Scope

- Support SaaS, eCommerce, and Services cost-model families.
- Compute pricing floor, target margin, margin health, confidence, and missing-input warnings.
- Map existing billing usage ledger and agent-run cost data into the cost model where available.
- Treat BYOK as a cost/supply signal until approved recommendations decide whether it affects price.

## Acceptance Criteria

- Recommendations block or degrade when cost model is missing.
- Cost model setup can be completed by an operator in under 10 minutes.
- Pricing below the floor is labeled as an explicit strategy, not a normal suggestion.
- Tests cover SaaS, eCommerce, Services, missing cost fields, and BYOK posture.
