# Plan 205-02 — Tenant Cost Model and Pricing Floor

## Objective

Ship `PricingCostModel` CRUD + snapshot write path + pricing-floor computation for SaaS, eCommerce, and Services business types, wired into the wizard state machine consumed by the operator UI (Plan 205-06). Seed the admission gate: inserting a `PricingRecommendation` (Plan 205-04) MUST be rejected with error code `PRICING_COST_MODEL_MISSING` when the tenant has no `status='ready'` cost model snapshot.

Scope: library + 3 API routes + F-113 contract body + 3 Vitest suites. UI lives in 205-06.

## Current code path this migrates

- `lib/markos/billing/entitlements.ts` — unchanged; this plan is additive.
- `lib/markos/billing/pricing-catalog.ts` — remains a billing-side adapter; Plan 205-07 retargets it to read approved recommendations. This plan does not touch it.
- `lib/markos/mcp/cost-table.cjs` — keeps its static caps as a fallback; Plan 205-05 adds `resolveTenantDailyCap(tenantId)` that reads from recommendations (not from this cost-model plan).
- `lib/markos/contracts/pricing.ts` — imported; never redefined.

## must_haves

- `lib/markos/pricing/cost-model.ts` exports `computePricingFloor(model)` returning `{ pricing_floor_usd_micro: number; confidence: number; missing_inputs: string[] }` for all 3 business types.
- `insertCostModelSnapshot(tenantId, input)` inserts an append-only row and updates `updated_at`; returns the new `snapshot_id`.
- BYOK posture module returns `'recommendation_input'` as default (per RESEARCH Approved Decision #5).
- Wizard state machine supports: `blank → saas_inputs → complete` | `blank → ecom_inputs → complete` | `blank → services_inputs → complete`; illegal transitions rejected.
- F-113 YAML fully populated with 3 operations + error envelopes; `x-markos-phase` marker removed.
- Vitest suite covers SaaS/eCom/Services floor math + missing-input degradation (status demotes `ready → partial → missing`) + BYOK branches + integer invariant.
- All money fields stored + returned as `usd_micro` integers; all percent-like fields stored as basis points (`bps`). No float money anywhere.
- No `stripe` import in any file.

  <action>
    Create `lib/markos/pricing/cost-model.ts`. Exports:

    - `computePricingFloor(model: PricingCostModel): { pricing_floor_usd_micro: number; confidence: number; missing_inputs: string[] }`:
      - **SaaS branch** (`business_type === 'saas'`): require `saas_infra_usd_micro_per_month`, `saas_llm_api_usd_micro_per_run`, `saas_support_usd_micro_per_tenant_month`, `saas_payment_fee_bps`, `saas_target_gross_margin_bps`. Floor = `(saas_infra + saas_support + assumed_runs_per_month * saas_llm_api_usd_micro_per_run) * (10_000 + saas_target_gross_margin_bps) / 10_000 / (10_000 - saas_payment_fee_bps) * 10_000`. `assumed_runs_per_month = 1000` (documented constant `DEFAULT_ASSUMED_RUNS_PER_MONTH`). All arithmetic via `addUsdMicro` / `subUsdMicro` from `money.ts`.
      - **eCommerce branch**: require `ecom_cogs_usd_micro_per_unit`, `ecom_fulfillment_usd_micro_per_unit`, `ecom_returns_bps`, `ecom_platform_fee_bps`. Floor = `(cogs + fulfillment) * (10_000 + returns_bps) / 10_000 / (10_000 - platform_fee_bps) * 10_000`.
      - **Services branch**: require `svc_fully_loaded_labor_usd_micro_per_hour`, `svc_utilization_bps`, `svc_overhead_multiplier_bps`. Floor = `labor * overhead_multiplier_bps / 10_000 / utilization_bps * 10_000`.
    - `missing_inputs`: array of the field names whose value is null; every missing required field contributes -0.1 to confidence (min clamp 0); all required fields present = confidence 1.0; any missing present = `status='partial'`; majority missing = `status='missing'`.
    - `insertCostModelSnapshot(supabase, tenantId, cost_model_id, input)`: validates via `PricingCostModel.parse`, computes floor, writes snapshot row, returns `snapshot_id`. Append-only — never UPDATEs prior snapshots.
    - `getLatestCostModel(supabase, tenantId, cost_model_id)`: queries `markos_pricing_cost_models_latest` view.
    - `assertCostModelReadyForRecommendation(supabase, tenantId)`: returns `void`; throws `PRICING_COST_MODEL_MISSING` if no cost model exists with `status='ready'` for the tenant. This is the admission gate Plan 205-04 calls from its recommendation insert path.
    - `isStale(model, now)`: returns true if `now - model.updated_at > 90 days` → callers should demote to `status='stale'`.

    All computations in usd_micro integers. Reject NaN/float via `money.ts`. No `stripe` import.
  </action>
