# Plan 205-01 — Pricing Contracts, Schema, RLS, F-ID/Migration Allocation

## Objective

Ship the locked Pricing Engine contract surface that every other 205 plan imports from, and lay down seven contiguous migrations (107-113) with row-level security, append-only history tables, and tenant-isolation tests. This is the **Wave 1 gate**: no 205 plan other than 205-01 may merge until the RLS tests are green and the Zod schemas in `lib/markos/contracts/pricing.ts` are exported.

This plan **creates** the five Zod schemas (`PricingCostModel`, `PricingKnowledge`, `PricingRecommendation`, `PriceTest`, `PricingEngineContext`) verbatim from `205-RESEARCH.md` §Proposed Contract Direction, **reserves** F-112..F-119 as declarative YAML scaffolds, and **stubs** the evidence-pack `pricing` slot so Phase 209 (EvidenceMap) has a landing spot. No business logic yet — logic lands in plans 205-02 through 205-08.

## Current code path this migrates

- `lib/markos/contracts/` — currently houses AgentRun v2 (`agent-run-v2.ts`) and billing (`billing/contracts.ts`); adds a sibling `pricing.ts` exporting the 5 Zod schemas.
- `lib/markos/billing/contracts.ts` — untouched by this plan. Plan 205-07 will ALTER billing tables via migration 113 to add `pricing_recommendation_id` FK.
- `supabase/migrations/` — highest on disk = `100_crm_schema_identity_graph_hardening.sql`; Phase 207 reserves 101-106; this plan ships the contiguous 107-113 block.
- `lib/markos/governance/evidence-pack.ts` — existing evidence-pack aggregates billing/approval/identity/audit/provider-sync; we add a `pricing: []` empty slot. 205-04 populates it.
- `lib/markos/mcp/approval.cjs` — NOT modified here. Plan 205-04 registers 4 new token names into the existing store.

## must_haves

- `lib/markos/contracts/pricing.ts` exports exactly the 5 Zod object schemas from 205-RESEARCH.md: `PricingCostModel`, `PricingKnowledge`, `PricingRecommendation`, `PriceTest`, `PricingEngineContext`; plus the supporting enums `PricingSubject`, `BusinessType`, `RecommendationState`, `RecommendationType`, `RiskLevel`, `Reversibility`, `PricingChangeMagnitude`, `PricingCostModelStatus`, `PriceTestState`, `PriceTestStopCondition`, `PricingEvidence`, `PricingOption`.
- Helper module `lib/markos/pricing/money.ts` exports `fromUsdMicro(n)` and `toUsdMicro(str)`; every pricing file uses this — no inline `/ 1_000_000` or `* 1_000_000` math outside this file.
- 7 migrations (107-113) exist under `supabase/migrations/` with matching rollbacks under `supabase/migrations/rollback/`; every new pricing table has RLS `USING (tenant_id = current_setting('app.tenant_id', true))`.
- `markos_agent_runs.pricing_engine_context` gains CHECK constraint: either `{}`, or `{"placeholder":"{{MARKOS_PRICING_ENGINE_PENDING}}"}`, or a schema-valid `PricingEngineContext` — placeholder and real fields are mutually exclusive.
- 8 F-ID YAMLs (F-112..F-119) scaffolded with `x-markos-phase: 205-<plan>-PLAN` markers that later plans remove on completion.
- Cross-tenant RLS regression test green for all 6 new pricing tables.
- Evidence-pack adds the `pricing` key returning `[]`.
- No import of `stripe` anywhere in this plan's files (library, migrations, tests).

    - `toUsdMicro(input: string | number): number` — parses `"$12.34"`, `"12.34"`, or `12.34` into `12_340_000`; rejects NaN and non-finite numbers with `PRICING_MONEY_INVALID_INPUT`.

    - `microToStripeCents(n: number): number` — returns `Math.round(n / USD_MICRO_PER_CENT)`; asserts `Number.isInteger(n)`.
    - `stripeCentsToMicro(c: number): number` — returns `c * USD_MICRO_PER_CENT`; asserts `Number.isInteger(c)`.

    All functions assert `Number.isInteger` on usd_micro inputs and throw `PRICING_MONEY_NON_INTEGER` on float. No float-returning helpers exist; formatting returns a string.
