# Plan 205-04 — Recommendation + PriceTest Approval Engine + PRC agent stubs

## Objective

Ship the two state machines (`RecommendationState` with 8 states; `PriceTestState` with 8 states), the approval-token bridge reusing `lib/markos/mcp/approval.cjs`, and the `pricing_engine_context` writer that lands data on `markos_agent_runs` for Phase 207 consumption. Register 4 PRC agent stubs (MARKOS-AGT-PRC-01..04) per RESEARCH §PRC Agent Allocation — PRC-05 deferred to Phase 211, PRC-06 owned by Plan 205-03.

## Current code path this migrates

- `lib/markos/mcp/approval.cjs` — untouched (reused as-is). This plan registers 4 new tool names in a new sibling module.
- `lib/markos/governance/evidence-pack.ts` — `pricing` slot created empty by Plan 205-01; this plan populates it with approved recommendations.
- `markos_agent_runs.pricing_engine_context` — column exists from migration 105 (Phase 207); CHECK added by Plan 205-01 migration 113. This plan writes to it.
- `markos_agent_tasks` — owned by Phase 207 Plan 207-04 (migration 104). Approval workflows create task rows via the existing RPC.

## must_haves

- `lib/markos/pricing/recommendation-state.ts` exports `RECOMMENDATION_TRANSITIONS` (legal DAG) and `assertTransitionAllowed(from, to)` throwing `PRICING_RECOMMENDATION_INVALID_TRANSITION`.
- `lib/markos/pricing/price-test-state.ts` mirrors the recommendation pattern with 8 states + `PRICE_TEST_TRANSITIONS`.
- `lib/markos/pricing/approval-bridge.ts` exports `approveRecommendation`, `activatePriceTest`, `rollbackPriceTest` — each binds to a distinct approval-token tool name.
- 4 tool names registered in `lib/markos/mcp/approval-tokens-registry.ts`: `pricing.recommendation.approve`, `pricing.test.activate`, `pricing.page.publish`, `pricing.billing.sync`. Token namespaces MUST NOT collapse into a single `pricing.mutate` (RESEARCH §Common Pitfalls #6).
- `lib/markos/pricing/agent-run-context-writer.ts` exports `writePricingEngineContext(runId, ctx)` that validates via `PricingEngineContext.parse` and updates `markos_agent_runs.pricing_engine_context` — obeys placeholder/real mutual-exclusion.
- 4 PRC agent stubs exist at `.agent/markos/agents/markos-prc-01..04-*.md`. PRC-01/02/03 `status: planned`. PRC-04 `status: experimental` (draft generator).
- `affects_existing_subscribers=true` requires secondary approval (tested in vitest state-machine test).
- F-115 YAML populated with 8 operations.
- Task handoff via `markos_agent_tasks` — every `proposed → approved/rejected` transition creates a task row referencing the recommendation.

      - Create task row in `markos_agent_tasks` for `proposed/modified/deferred` transitions (handoff to operator) — call existing Phase 207 RPC `markos_agent_tasks_insert`.
    - `listRecommendations(supabase, tenantId, { state? })`.
    - `getRecommendation(supabase, rec_id)`.
    - `archiveRecommendation(supabase, rec_id, actor_id)`.

      3. Insert with `state='proposed'`.
    - `transitionPriceTest(supabase, test_id, { toState, actor_id, reason, approval_token? })`:
      - `assertTransitionAllowed`.
      - `active` transition requires `approval_token` bound to `pricing.test.activate`.
      - On `active → stopped|rolled_back`, write `activated_at`/`stopped_at` and append history seq.
    - `evaluateAndMaybeStop(supabase, test_id, observedMetrics)`: calls `evaluateStopConditions`; if `shouldStop`, transitions to `stopped` with `reason: 'stop_condition_breached'`.

    Both stores call `agent-run-context-writer.writePricingEngineContext` (Task 4) when invoked inside an AgentRun (run_id passed in opts).

    No `stripe` import.
