# Plan 208-03 - Unified Task Board

**Phase:** 208 - Human Operating Interface  
**Status:** Planned  
**Requirements:** TASK-01, TASK-03, RUN-05, QA-01..15

## Objective

Turn agent outputs, escalations, approvals, failures, connector recovery, and follow-ups into one task system.

## Scope

- Define persisted task records for source agent, priority, owner, status, due date, evidence, campaign, revenue context, and blockers.
- Migrate the fixture-backed task runner toward the v2 substrate.
- Add grouping by priority, source, status, owner, and campaign/revenue context.

## Acceptance Criteria

- Tasks can originate from AgentRun, ApprovalGate, ConnectorInstall, PricingRecommendation, CRM, billing, support, or learning.
- Every task has evidence or a reason evidence is missing.
- Task state changes are append-only and auditable.

## Verification

- Reducer and persistence tests cover creation, assignment, completion, retry, and blocked states.
