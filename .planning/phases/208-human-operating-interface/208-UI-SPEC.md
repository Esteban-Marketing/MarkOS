# Phase 208 — UI Design Contract

> Visual and interaction contract for the **MarkOS Operator Cockpit**: six daily-driver surfaces shipped across plans 208-01 through 208-06.
>
> Phase 208 is the **first heavy-UI phase** to consume the 206 (compliance) and 207 (orchestration) doctrine. Every approval gate, every run-state badge, every cost-posture chip, every recovery row, every narrative section binds verbatim to a contract field defined upstream. No new doctrine is invented in this phase — only its visual expression.
>
> Future translation gates (P209 evidence, P210 connectors, P211 dispatch, P212 learning) are explicit placeholder states inside this UI; each is rendered via `<PlaceholderBanner>` with named `placeholder_state` literal. Phase 208 ships a polished cockpit shell whose unfinished substrate is honestly visible to the operator.
>
> Authority chain: DESIGN.md v1.1.0 → 213.x adoption-wave decisions D-08..D-15 → 206-UI-SPEC (mutation-class origin) → 207-UI-SPEC (orchestration substrate origin) → this document. Generated by gsd-ui-researcher. Status: draft (checker upgrades to approved once 6-pillar audit passes).

## Spacing Scale

Authoring rule: every `padding`, `margin`, `gap`, `inset` in scope MUST cite a `--space-*` token. No arbitrary px. Off-grid values are auto-FAIL.

| Token | Value | DESIGN.md citation | Usage in this phase |
|-------|-------|--------------------|---------------------|
| `--space-none` | 0 | `spacing.none` | Reset margins on `<h1>`, `<h2>`, `<p>`, `<table>`, `<ul>` |
| `--space-xxs` | 2px | `spacing.xxs` | Badge inner padding, status-dot offset, brief-item urgency-glyph adjacency |
| `--space-xs` | 8px | `spacing.xs` | Brief-item icon-text adjacency, approval-card metadata gap, recovery-row impact-list gap, narrative-bullet leader gap, task-row priority chip group gap |
| `--space-sm` | 16px | `spacing.sm` | Card vertical rhythm, notice padding-block, table `th`/`td` padding-block, mobile horizontal page padding (max-width ≤ 640px), brief-section header gap, narrative-section spacer |
| `--space-md` | 24px | `spacing.md` | Card padding (via `.c-card`), gap between brief sections, modal padding, page horizontal padding at sm–md viewport, approval-modal body padding, recovery-page section gap |
| `--space-lg` | 32px | `spacing.lg` | Page horizontal padding ≥ md, inter-section gap within Morning Brief grid, Task Board column-group gap, narrative-section gap |
| `--space-xl` | 48px | `spacing.xl` | Page vertical padding ≥ lg, Morning Brief top padding, weekly narrative time-window header padding |
| `--space-xxl` | 96px | `spacing.xxl` | Reserved — not used in this phase |

**Allowed exceptions (DESIGN.md documented):**
1. `1px` for hairline borders (`var(--color-border)`) — preferred over shadows on dark surfaces (DESIGN.md "Borders over shadows").
2. `2px` for focus ring width and offset (`var(--focus-ring-width)`, `var(--focus-ring-offset)`); never suppressed.
3. `4px` for `.c-notice` `border-inline-start` accent — composed via `.c-notice` primitive; modules never declare this directly.
4. `max-width: 1280px` for cockpit container (`--w-container`). Brief and Recovery use `--w-container`; Task Board and Narrative may extend to full viewport with internal `--w-prose` (720px) constraint on text columns.
5. `max-width: 560px` for confirmation modals (`--w-modal`).
6. **Mobile touch target `44px`** via `--h-control-touch` on `.c-button` for `(pointer: coarse)` viewports — already declared globally in `styles/components.css` per 213.2 cross-cutting fix; this phase consumes it without re-declaration.

**Forbidden (auto-FAIL):**
- Any third typeface (not JetBrains Mono or Inter).
- Inline `font-size`, `font-weight`, `color` literals — use tokens only.
- Hard-coded dollar amounts (e.g. `$99`, `$150.00`) — use `<Money />` + `--fs-code` or `{{MARKOS_PRICING_ENGINE_PENDING}}`.
- Tier name strings `"Growth Monthly"`, `"Starter"`, `"Professional"` — banned in all JSX/CSS (205 inheritance).
- Soft sentence openers banned by CLAUDE.md: `synergy`, `leverage`, `empower`, `unlock`, `transform`, `revolutionize`, `supercharge`, `holistic`, `seamless`, `cutting-edge`, `innovative`, `game-changer`, `next-generation`, `world-class`, `best-in-class`, `reimagine`, `disrupt`, `just` (as softener). No exclamation points in any product surface copy.
