# Plan 208-01 - Operating Shell Information Architecture

**Phase:** 208 - Human Operating Interface  
**Status:** Planned  
**Requirements:** TASK-01..05, RUN-07, QA-01..15

## Objective

Reframe the app shell from a generic control plane into an operator decision system.

## Scope

- Define navigation for Morning Brief, Task Board, Approval Inbox, Recovery Center, and Weekly Narrative.
- Map existing operations, CRM, billing, MCP, webhook, and governance surfaces into the new IA.
- Preserve activation gates for SaaS Suite and future growth modules.
- Define mobile reactive scope.

## Acceptance Criteria

- First screen routes operators toward decisions, not passive metrics.
- Existing useful surfaces are reused instead of duplicated.
- Mobile scope is approvals, brief, inbox, notifications, and quick tasks.

## Verification

- UI route map and accessibility checklist are produced before implementation.
