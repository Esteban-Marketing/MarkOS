# Plan 208-05 - Connector Recovery and Impacted-Agent Visibility

**Phase:** 208 - Human Operating Interface  
**Status:** Planned  
**Requirements:** CONN-04, TASK-01, RUN-05, QA-01..15

## Objective

Make broken data sources visible, owned, and recoverable.

## Scope

- Show connector status, last sync, missing scopes, credential failure, rate limits, and recovery steps.
- Show impacted agents, blocked runs, blocked tasks, and stale evidence risk.
- Create or link recovery tasks from ConnectorInstall failures.

## Acceptance Criteria

- Connector failures pause dependent agent work when data would be unsafe.
- Operator sees exact recovery action and impacted workflows.
- Recovery state is evidence-linked and auditable.

## Verification

- Recovery fixtures cover revoked token, missing scope, stale sync, rate limit, and backfill failure.
