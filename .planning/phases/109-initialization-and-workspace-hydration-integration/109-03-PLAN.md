### Task 2: Add packSelection block and update both response objects

**File:** `onboarding/backend/handlers.cjs`

**Action:** In `handleApprove`, (1) declare `let packSelection = null` alongside the existing `let skeletonsSummary` declaration, (2) insert the resolve + seed-write block inside the existing skeleton try, after the seed read and before `generateSkeletons`, (3) update the `generateSkeletons` call to pass `undefined, undefined, packSelection`, and (4) add `packSelection: packSelection` to both `json(res, 200, {...})` response objects.

**Details:**

**Step A — Declare packSelection before the try block.**

Find this line (currently ~line 2383):
```javascript
    let skeletonsSummary = { generated: [], failed: [] };
```

Change to:
```javascript
    let skeletonsSummary = { generated: [], failed: [] };
    let packSelection = null; // Phase 109 — declared here so both json() calls outside the try can read it
```

**Step B — Insert resolve + seed-write block inside the existing skeleton try.**

Find the existing seed read block inside the try (lines ~2385–2388):
```javascript
      let seed = {};
      if (fs.existsSync(SEED_PATH)) {
        seed = JSON.parse(fs.readFileSync(SEED_PATH, 'utf8'));
      }
```

Insert the following block IMMEDIATELY after the closing `}` of the `if (fs.existsSync(SEED_PATH))` block — before the existing `const skeletonResults = ...` line:

```javascript
      // Phase 109: resolve pack selection and persist to seed (D-01, D-02, D-06, D-08) ─────────
      try {
        packSelection = resolvePackSelection(seed);
        seed.packSelection = packSelection; // D-08: written as-is, no transformation
      } catch (resolveErr) {
        console.warn('[POST /approve] resolvePackSelection failed:', resolveErr.message);
        // packSelection remains null; seed.packSelection not set; continue (D-06)
      }

      if (packSelection !== null) {
        try {
          fs.writeFileSync(SEED_PATH, JSON.stringify(seed, null, 2), 'utf8'); // D-02
        } catch (writeErr) {
          console.warn('[POST /approve] Failed to persist packSelection to seed:', writeErr.message);
          // Non-fatal — approval continues without persisted packSelection (D-06)
        }
      }
      // End Phase 109 ────────────────────────────────────────────────────────────────────────────
```

**Step C — Update the `generateSkeletons` call.**

Current (line ~2397):
```javascript
      const skeletonResults = await generateSkeletons(seed, approvedDrafts);
```

New:
```javascript
      const skeletonResults = await generateSkeletons(seed, approvedDrafts, undefined, undefined, packSelection); // Phase 109
```

**CRITICAL:** Use `undefined` (not `null`) for the 3rd and 4th args — `undefined` triggers the default parameter values (`MARKOS_LOCAL_DIR` and `TEMPLATES_DIR`). Passing `null` would override the defaults with `null` and cause `path.join(null, ...)` to throw.

## Threat Model

- **Seed file write**: `JSON.stringify(seed, null, 2)` uses standard pretty-print with no interpolation of user data. The seed is an object already in memory that was read from the same file moments before. `SEED_PATH` is a controlled constant from `path-constants.cjs`, not user-supplied. Write is wrapped in try/catch per D-06.
- **resolvePackSelection input**: `seed` was read with `JSON.parse(fs.readFileSync(...))` — structured data, not an eval context. `resolvePackSelection` does its own canonicalization internally. No injection vector.
- **Response payload**: `packSelection` is the raw return value of `resolvePackSelection()` or `null` — it contains only `basePack`, `overlayPack`, `overrideReason`, `resolvedAt` (all strings or null). No user-controlled data is inserted.
