## Tasks

### Task 1: Extend `resolveSkeleton` signature and add overlay-first check

**File:** `onboarding/backend/agents/example-resolver.cjs`

**Action:** Modify the `resolveSkeleton` function (currently at line 175) to accept an optional 4th parameter `overlaySlug = null`. When `overlaySlug` is truthy, perform an overlay path check using the existing `readFileSafe` helper before the existing base path logic. If the overlay file contains content, return it immediately (Replace strategy, D-03). If absent, fall through silently to existing behavior (D-06).

**Details:**

Current signature (line 175):
```javascript
function resolveSkeleton(discipline, businessModel, basePath = DEFAULT_BASE) {
```

New signature:
```javascript
function resolveSkeleton(discipline, businessModel, basePath = DEFAULT_BASE, overlaySlug = null) {
```

Insert the following block immediately after the `if (!slug) { return ''; }` guard — before the existing `const filePath = path.join(...)` base path check:

```javascript
  // Phase 109: industry overlay check (D-03 — Replace strategy, D-06 — silent fallback)
  if (overlaySlug) {
    const overlayPath = path.join(basePath, 'SKELETONS', 'industries', overlaySlug, discipline, 'PROMPTS.md');
    const overlayContent = readFileSafe(overlayPath);
    if (overlayContent) {
      return overlayContent;
    }
    // Overlay file absent → fall through to base skeleton silently (GOV-01 diagnostics are Phase 110)
  }
```

The rest of the function body remains UNCHANGED — the existing `filePath` construction, `directContent` check, `readFallbackContent` chain are all preserved as-is.

**`readFileSafe` is already defined in this file** (line ~61). Do NOT add a new helper or a separate `fs.existsSync` call — `readFileSafe` already returns `''` on missing files.

No changes to `module.exports` — `resolveSkeleton` is already exported.

## Threat Model

- **Path traversal via overlaySlug**: `resolvePackSelection()` in `pack-loader.cjs` canonicalizes its `overlayPack` slug via `canonicalizeValue()` (strips non-alphanumeric chars except hyphens). The slug values are constrained to known keys in `INDUSTRY_ALIAS_MAP` (travel, it, marketing-services, professional-services). `path.join` does not prevent traversal on its own, but the slug can only ever contain lowercase letters and hyphens — no `../` sequences are possible from the aliased values. No user-supplied raw string reaches this code path.
- **No new file writes in this plan** — read-only filesystem access only.
