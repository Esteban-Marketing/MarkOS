# Phase 213.5 — UI Design Contract

> Visual and interaction contract for redesigning the two Phase 200 marketing surfaces —
> `/integrations/claude` (Claude Marketplace landing page) and `/integrations/claude/demo`
> (in-browser MCP playground) — from orphan class names (`.btn`, `.hero`, `.tool-grid`,
> `.demo-form`, `.demo-result`, `.demo-error`, `.demo-loading`) into the canonical
> MarkOS DESIGN.md v1.1.0 visual contract: Kernel Black surface, Protocol Mint signal,
> JetBrains Mono + Inter, strict 8px grid, `.c-*` primitive composition, bracketed-glyph
> state coding, voice classifier 100/100 preserved.
>
> **This is a token-adoption redesign.** DOM tree, props, client fetch wiring, and MCP
> allow-list tool slugs in copy are preserved verbatim (D-20/D-21). Only class names and
> copy strings that violate canon are changed. Generated by gsd-ui-researcher.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — vanilla CSS Modules consuming `var(--*)` tokens from `app/tokens.css` + composing `.c-*` primitives from `styles/components.css` |
| Preset | not applicable |
| Component library | none — primitives defined in `styles/components.css` (Phase 213.1, v1.1.0) |
| Icon library | none — no icon elements in scope; bracketed glyphs carry every state signal |
| Heading font | JetBrains Mono (`var(--font-mono)`) — inherited via `<h1>`, `<h2>`, `<h3>` from `globals.css` |
| Body font | Inter (`var(--font-sans)`) — inherited via `<p>` and form elements from `globals.css` |
| Default theme | dark (`color-scheme: dark`) — `globals.css` global; no local override needed |
| Form authoring posture | Primitive-only. Every `<input>`, `<button>`, `<label>` MUST compose `.c-input`, `.c-button{,--primary,--tertiary}`, `.c-field` + `.c-field__{label,help,error}`. Modules carry only layout/spacing concerns. |

## Spacing Scale

Authoring rule: every `padding`, `margin`, `gap`, `inset` in scope MUST cite a `--space-*` token. No arbitrary px. Off-grid values are auto-FAIL.

| Token | Value | DESIGN.md citation | Usage in this phase |
|-------|-------|--------------------|---------------------|
| `--space-none` | 0 | `spacing.none` | Reset margins on headings; `<fieldset>` reset |
| `--space-xxs` | 2px | `spacing.xxs` | Tight adjacency only (protocol-chip pseudo gap) |
| `--space-xs` | 8px | `spacing.xs` | Tool-card internal gap (slug↔description), form field label↔input↔help gap, inline icon-text adjacency |
| `--space-sm` | 16px | `spacing.sm` | Hero section vertical rhythm, demo form field gap, result-card padding-block, mobile horizontal padding |
| `--space-md` | 24px | `spacing.md` | Value-props section gap, landing section vertical gap, result-card padding-inline, form field group gap |
| `--space-lg` | 32px | `spacing.lg` | Hero card padding (`c-card--feature` internal), inter-section spacing, landing page padding-inline ≥ md |
| `--space-xl` | 48px | `spacing.xl` | Page padding-block (all breakpoints ≥ lg), section breathing room |
| `--space-xxl` | 96px | `spacing.xxl` | Landing page top padding-block at ≥ xl viewport (hero push-down) |

**Exceptions (allowed by DESIGN.md):**
1. `1px` for hairline borders (`var(--color-border)`) — preferred over shadows per DESIGN.md "Elevation & Depth."
2. `2px` for focus ring width and offset (`var(--focus-ring-width)`, `var(--focus-ring-offset)`) — global, not locally authored.
3. `4px` for `.c-notice` `border-inline-start` accent (Phase 213.1 banner formula).

**Banned values (auto-FAIL — orphan orphan-class layout values to eliminate):**
- `10px`, `12px`, `13px`, `14px`, `15px`, `18px`, `20px`, `22px`, `28px` — all off-grid and present in legacy bespoke selectors.

| Range | Token group | These surfaces |

**Accent reserved-for list (10% slice — both surfaces combined):**
1. **Single primary CTA per surface** (mint fill via `.c-button--primary`): landing → "Try the demo"; demo → "Draft message".
2. **Focus rings** — globally inherited via `:focus-visible` in `globals.css` (2px solid mint, 2px offset).
3. **Protocol chip slug text** — `.c-chip-protocol` mint text on `surface-raised` for each tool name: `[draft_message]`, `[plan_campaign]`, etc.
4. **Tertiary text-button color** — "Read the quickstart", inline "the quickstart" link, "Open the demo sandbox" (final CTA secondary) — all via `var(--color-primary-text)` through `.c-button--tertiary`.
5. **`::selection`** — `var(--color-primary-subtle)` (global).

NOT mint: subheadings, value-prop body, tool descriptions, install steps, form labels, card borders.

**Forbidden values (auto-FAIL):**
- `#000000` pure black.
- `#FFFFFF` pure white outside `[data-theme="light"]`.
- Any hex value not on the above list — all values cited via `var(--*)` tokens.
- Gradients of any kind — `radial-gradient`, `linear-gradient` — banned (DESIGN.md "Anti-patterns").
- `box-shadow` with `rgba(...)` or any color — cards use 1px border hairline only.
- `transform: translateY(Npx)` hover-jiggle — no hover-translate on any element.
