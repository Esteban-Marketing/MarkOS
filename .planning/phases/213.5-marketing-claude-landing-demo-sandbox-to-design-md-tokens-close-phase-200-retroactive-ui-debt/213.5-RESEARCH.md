## Risks and Pitfalls

- **429 branch missing in onSubmit:** `demo/page.tsx` currently has NO `response.status === 429` check — falls through to generic error. Plan 02 Task 02 MUST insert this before the generic `!response.ok` branch or DS-6 fails. [VERIFIED: route.ts confirmed 429 response; page.tsx confirmed no 429 branch]
- **Second `.c-button--primary` in landing:** L-6 permits 2 occurrences (hero + final-cta sections are scroll-separated, not simultaneously visible). The a11y test must verify count = 2, not ≤ 1.
- **`<code>` vs `<span>` for tool slugs:** `page.tsx` tool grid uses `<code>{tool.name}</code>` — must change to `<span className="c-chip-protocol">`. `.c-chip-protocol` uses `::before`/`::after` for brackets; do NOT double-wrap in `<code>`.
- **`aria-label` on `<Link>` CTAs preserved:** Both `<Link>` CTAs in the hero have `aria-label` attrs — do not remove them when swapping className. They are not in scope but adjacent to the className change.
- **MCP install URL `<code>` → `<code className="c-code-inline">`:** Only the `className` changes; text content `https://markos.dev/api/mcp/session` is preserved verbatim (D-21).
- **`max-width: var(--w-modal)` on `.sandbox`:** Verify `--w-modal` is defined in `app/tokens.css` before authoring. If token is absent, use `max-width: 40rem` via a local token — do not hard-code px.
- **Voice classifier target discrepancy:** `page.tsx` JSDoc says `score >= 85`; CONTEXT.md D-11 says `100/100`. Respect D-11 (higher bar). Existing copy is clean — no regression expected.
- **Off-grid forbidden values in legacy HTML:** `page.tsx` install section has a `<code>` element inside `<li>` — no CSS on it currently. When `.c-code-inline` is added, it inherits primitive padding; verify no off-grid values surface via primitive's own rules (already v1.1.0 canonical — safe).
