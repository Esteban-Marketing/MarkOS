## Top 3 Priority Fixes

1. **`max-width: 75rem` literal in `.landing` class** — breaks the single-source-of-truth rule; `var(--w-container)` resolves to 1280px (80rem) per `tokens.css` line 151, while `75rem` = 1200px is an off-token value. A future token update to `--w-container` would not cascade. Fix: change `page.module.css` line 19 from `max-width: 75rem` to `max-width: var(--w-container)`.

2. **Success state missing `[ok]` bracketed glyph and `.c-notice c-notice--success`** — UI-SPEC color table explicitly cites `--color-success` for demo draft-success via `.c-notice c-notice--success` with `[ok]` glyph. CONTEXT spec states `[ok] Draft generated.` Omitting it means the success state is visually confirmed only by the result card appearing — no paired text glyph, which weakens the DESIGN.md non-negotiable rule 6 ("state color never the only signal"). RESEARCH.md noted this as Claude's Discretion and recommended omitting it, but the UI-SPEC color table treats it as a declared element. Fix: add `{result && <p role="status" className="c-notice c-notice--success">[ok] Draft generated.</p>}` above the result section in `demo/page.tsx`. Update X-8 test to assert `[ok]` presence.

3. **X-8 test does not assert `[ok]` glyph** — the test checks `[err]` and `[warn]` but not `[ok]`, leaving a gap in the X-8 AC coverage. The RESEARCH.md "all 3 present" assertion in the verification map does not match the test file. Fix: add `assert.ok(src.includes('[ok]'), '[ok] glyph required for success state')` to the X-8 test block (requires fix 2 first).

PASS with one finding.

States confirmed present:

Deduction finding (1 point):
