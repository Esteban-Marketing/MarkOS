# Phase 201.1 Plan 05: Edge-Config Slug Cache Hardening Summary

Jittered-TTL slug cache + per-instance single-flight coalescer + transitional-410 on rename with read-after-write verification. Closes review concern H5 in three parts.

## What Was Built

### slug-cache.cjs additions (jitter + transitional)

Five new exports appended to the existing 94-line file (additive — no existing exports removed):

- `computeJitteredTtl()` — returns `45 + Math.floor(Math.random() * 31)` (range [45, 75]s)
- `writeSlugToEdgeJittered(slug, tenantId, deps)` — calls `writeSlugToEdgeRaw` with jittered TTL
- `writeSlugToEdgeRaw(slug, value, ttlSeconds, deps)` — low-level PATCH with optional `ttl` field in items payload
- `writeTransitionalRename(oldSlug, newSlug, deps)` — writes `__renamed:{newSlug}` at `markos:slug:{oldSlug}` with 90s TTL
- Constants: `SLUG_CACHE_TTL_BASE_SECONDS=45`, `SLUG_CACHE_TTL_JITTER_SECONDS=30`, `TRANSITIONAL_RENAME_TTL_SECONDS=90`, `TRANSITIONAL_PREFIX='__renamed:'`

### single-flight.cjs (new, 30 LoC)

`createSingleFlight()` returns a `{ coalesce(key, fn), inflightSize(), clear() }` object. An in-flight `Map<key, Promise>` is cleared via `.finally()` on settle (both resolve and reject). Module exports a default `singleFlight` instance scoped to the Vercel function invocation lifetime.

### middleware.ts diff (single-flight wiring + 410 branch)

First-party slug branch rewritten:

1. Imports `{ readSlugFromEdge, writeSlugToEdgeJittered, TRANSITIONAL_PREFIX }` from slug-cache.cjs
2. Imports `{ singleFlight }` from single-flight.cjs
3. After `readSlugFromEdge`: if value starts with `TRANSITIONAL_PREFIX` → return 410 with `Location`, `Sunset`, `Cache-Control`, `X-Markos-Renamed-From`, `X-Markos-Renamed-To` headers
4. Normal cache hit path unchanged (returns tenant-id headers)
5. Cache miss path wrapped in `singleFlight.coalesce('slug:'+slug, ...)` — Supabase round-trip is shared within warm instance
6. Cache backfill uses `writeSlugToEdgeJittered` instead of old fixed-TTL `writeSlugToEdge`

### tenants.cjs renameTenantSlug body

Augmented from 2-step (UPDATE + invalidate/write) to 5-step D-104 flow:

1. UPDATE `markos_tenants` SET slug = newSlug
2. `writeTransitionalRename(oldSlug, newSlug, deps)` — pins 90s transitional marker
3. `writeSlugToEdgeJittered(newSlug, tenantId, deps)` — backfills jittered cache for new slug
4. Poll `readSlugFromEdge(oldSlug, deps)` up to 3× (50/100/200ms backoff); sets `confirmed=true` on first hit; logs warn and proceeds on miss (fail-soft)
5. `enqueueAuditStaging` with `action: 'tenant_slug.renamed'`, `source_domain: 'tenancy'`, payload includes `{ old_slug, new_slug, transitional_pin_confirmed }`

Returns `{ ok: true, transitional_pin_confirmed: boolean }`.

### Vercel Edge Config per-item TTL note

The PATCH body includes `item.ttl = ttlSeconds` in the items array. The Vercel Edge Config PATCH `/v1/edge-config/{id}/items` API accepts per-item `ttl` as of the API version documented in Vercel's platform docs (verified via plan spec). If the underlying edge-config plan does not support per-item TTL, the `ttl` field is silently ignored by the API and the entry is written with the default global TTL — the jitter logic still runs on every write, so the distribution is maintained across the cohort even if the actual server-side TTL falls back to the global value.
