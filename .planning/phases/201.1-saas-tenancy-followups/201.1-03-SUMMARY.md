### Task 2 — reserved-slugs.cjs refactor + admin handlers + F-106 + tests

**reserved-slugs.cjs additions** (49 LoC baseline preserved unchanged):

```javascript
loadReservedSlugsFromDb(client)   // SELECT slug FROM markos_reserved_slugs; 5-min TTL cache
clearReservedSlugCache()           // sets expires_at=0; called by admin add/remove + tests
getDefaultProfanitySet()           // obscenity RegExpMatcher (bad-words fallback, noop+warn last resort)
isReservedSlugAsync(slug, client)  // sync floor + DB overlay + profanity pass
```

Mandatory fallback contract (W-1.3 — grep verifiable):
```javascript
// try { require('obscenity') } catch { require('bad-words') }
```
Both-fail path emits `console.warn('[reserved-slugs] both obscenity and bad-words failed to load; profanity check disabled…')`.

**obscenity API** — version 0.4.6, `require` path. Correct construction:
```javascript
const matcher = new RegExpMatcher({
  ...englishDataset.build(),
  ...englishRecommendedTransformers,
});
matcher.hasMatch(str); // → boolean
```
`englishRecommendedTransformers` provides leet-speak, confusables, and `toAsciiLowerCase` — so `Fuck-slug`, `fuckslug`, and `fucksluga` all match without manual lowercasing.
