## Resolver + domains.cjs diffs

**resolver.cjs — `resolveTenantByDomain`:**
- Signature: `(client, domain, opts)` — `opts.allowGrace: boolean`.
- When `allowGrace=true`: DB query uses `.or('status.eq.verified,and(status.eq.failed,last_verified_at.gte.{cutoff})')` where cutoff = `now() - 24h`.
- Always computes `withinGrace: boolean` in JS on the returned row (status=failed + within window).
- Return shape gains `withinGrace` field.

**domains.cjs — `pollDomainStatus`:**
- Pre-reads previous row (`status, last_verified_at, tenant_id, org_id`) before writing patch.
- Sets `last_verified_at = now` on every `verified=true` result (not just first transition).
- If `prev.status === 'verified'` AND `!verified`: fires `fireByodFailureAlert` (fail-soft try/catch).

## M2 Closure Attestation

Review concern **M2**: "If domain transitions verified->failed (cert expiry, DNS drift), middleware drops it -> tenant offline. No grace window, no alert in plan summary. Real-world: certs expire silently."

**Closed by this plan:**

1. **24h grace window:** `isWithinByodGraceWindow(row)` + `resolveTenantByDomain({allowGrace:true})` + middleware sets `x-markos-byod-grace=1`. Tenant remains served in degraded mode for 24h after last successful verification.
2. **Alert hook:** `fireByodFailureAlert` fires on `verified->non-verified` transition in `pollDomainStatus`. Inserts `markos_webhook_deliveries` row with `status='queued'` and `event_type='byod.verification_lost'`. Audit row always emitted.
3. **New event registered:** `byod.verification_lost` added to `engine.cjs` WEBHOOK_EVENTS.
4. **Tests prove behavior:** grace window unit tests (5 cases) + alert queueing unit tests (4 cases) + middleware structural assertions (3 cases) + resolver unit tests (4 cases).
