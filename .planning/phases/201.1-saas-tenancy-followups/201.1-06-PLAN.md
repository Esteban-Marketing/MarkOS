<objective>
Reconcile GDPR Article 17 (right to erasure) with the per-tenant SHA-256 hash chain by implementing a **pseudonymize-with-tombstone** policy. Closes review concern **M4** ("Erasure (Art. 17) requires PII removal. Audit log rows reference user_id, IPs, etc. Redacting those breaks `row_hash`. No mention of erasure policy.").

Purpose: Phase 201's audit log is append-only with RLS RESTRICTIVE blocking UPDATE+DELETE. This makes immutability guarantees clean BUT creates a direct conflict with GDPR Art. 17: when a user requests deletion of their personal data, PII fields embedded in audit row payloads (user_id, email, IP) must be redacted. Cross-AI review M4 flagged this and recommended either (a) pseudonymize-in-place with documented chain exception, or (b) shadow PII vault. CONTEXT D-106 picks (a).

The 5-step erasure protocol (per CONTEXT D-106):
1. Write a NEW audit row with `action='audit.pii_erased'`, payload includes `original_row_id` + `redaction_marker`. This is the **tombstone**.
2. Replace the original row's payload PII fields with `{"__redacted":"<reason>","at":"<iso-ts>","by":"<actor_id>"}`.
3. Recompute the original row's `row_hash` using the new payload + locked canonical spec from Plan 01.
4. Leave `prev_hash` of the NEXT row UNTOUCHED (the chain references prev_hash → next_row.prev_hash; only the redacted row's row_hash drifts).
5. The verifier learns: when row N's expected_row_hash != actual_row_hash AND a tombstone audit row exists for row N before any subsequent mismatch, report this as `redaction_tombstone_detected` (chain still considered ok). If no tombstone, report `tampering_suspected`.

Output: (1) Migration 93 publishing `erase_audit_pii(bigint, text, text) returns void` SQL function with tenant-admin RLS gate; (2) Node-side wrapper `lib/markos/audit/erasure.cjs::eraseAuditPii`; (3) augmented `chain-checker.cjs::verifyTenantChain` that detects tombstones; (4) authenticated tenant endpoint `api/tenant/audit/erase-pii.js`; (5) 4 dedicated test files (erasure unit, tombstone-aware verifier positive, tombstone-aware verifier negative, **atomicity fault-injection** — the last added by W-2).

**Backward-compat note:** Per CONTEXT phase_201_substrate_constraints, "old rows already in markos_audit_log must remain verifiable. Migration 88 / D-103 must include a one-shot re-canonicalization+re-hash audit run for any pre-existing row that fails the new spec, BEHIND a feature flag, with a tombstone audit row announcing the recompute." This plan implements that one-shot recanonicalization helper (`recanonicalize_legacy_audit_row(bigint)`) BEHIND a feature flag (`MARKOS_AUDIT_RECANONICALIZE_ENABLED=1`) that is OFF by default. Plan 11 (D-108) staging-smoke harness exercises it. The recanonicalize emits its own tombstone (`action='audit.recanonicalized'`) so the chain verifier accepts the drift without flagging tampering.
</objective>

-- Phase 201.1 D-106 (closes M4): right-to-erasure vs hash chain reconciliation.
-- Pseudonymize-with-tombstone policy: original row's payload is redacted in place,
-- row_hash is recomputed, and a NEW audit row announces the redaction.
-- Chain verifier (lib/markos/audit/chain-checker.cjs) cross-references the tombstone.
--
-- ATOMICITY CONTRACT (W-2 closure):
-- erase_audit_pii is a single PL/pgSQL function call. PL/pgSQL provides an implicit
-- transaction around the function body, so if ANY step (tombstone INSERT in step 3
-- or the payload UPDATE in step 6) raises, the ENTIRE function rolls back, including
-- the tombstone. This guarantees the chain never ends up with a tombstone but no
-- redaction (or vice versa). Atomicity is verified by test/audit/erasure-atomicity.test.js
-- via fault-injection.

create extension if not exists pgcrypto;

-- ============================================================================
-- 1. Schema: redacted_at + redacted_by columns (operator hints, not part of canonical payload)
-- ============================================================================
alter table markos_audit_log
  add column if not exists redacted_at timestamptz,
  add column if not exists redacted_by text;

create index if not exists idx_markos_audit_log_redacted_at on markos_audit_log(redacted_at) where redacted_at is not null;

comment on column markos_audit_log.redacted_at is 'Phase 201.1 D-106: timestamp at which PII was erased. NULL means original row.';
comment on column markos_audit_log.redacted_by is 'Phase 201.1 D-106: actor_id who triggered the erasure (matches the tombstone audit row actor_id).';

-- ============================================================================
-- 2. RLS: replace the migration-82 block-UPDATE policy with a more nuanced one.
-- Direct UPDATEs are still blocked. Only the erase_audit_pii (security definer) fn
-- can write through.
-- ============================================================================
-- Note: migration 82 already created markos_audit_log_block_update RESTRICTIVE.
-- We keep it. erase_audit_pii uses SECURITY DEFINER and runs as the function owner
-- (typically the supabase admin role) which bypasses RLS for table operations
-- performed inside the function body.

-- ============================================================================
-- 3. erase_audit_pii — pseudonymize-with-tombstone fn (atomic per implicit txn)
-- ============================================================================
create or replace function erase_audit_pii(
  p_audit_row_id bigint,
  p_redaction_marker text,
  p_actor_id text
) returns table (id bigint, tombstone_id bigint, new_row_hash text)
  language plpgsql
  security definer
  set search_path = public
as $$
declare
  v_row record;
  v_redacted_payload jsonb;
  v_new_canonical text;
  v_new_row_hash text;
  v_tombstone_id bigint;
begin
  -- 1. Fetch the row.
  select id, tenant_id, org_id, source_domain, action, actor_id, actor_role, payload, occurred_at, prev_hash, row_hash
    into v_row
    from markos_audit_log
    where markos_audit_log.id = p_audit_row_id;

  if v_row.id is null then
    raise exception 'erase_audit_pii: audit row not found: %', p_audit_row_id;
  end if;
  if v_row.tenant_id is null then
    raise exception 'erase_audit_pii: row has null tenant_id (illegal state)';
  end if;
  -- Refuse double-redaction.
  if v_row.payload ? '__redacted' then
    raise exception 'erase_audit_pii: row % already redacted', p_audit_row_id;
  end if;

  -- 2. Per-tenant advisory lock — serialize against append_markos_audit_row.
  perform pg_advisory_xact_lock(hashtext('audit:' || v_row.tenant_id));

  -- 3. Append the tombstone row via the standard writer (this advances the chain).
  --    If the subsequent UPDATE (step 6) fails, the implicit-txn rollback removes
  --    this tombstone too — that's the atomicity property tested by erasure-atomicity.test.js.
  select t.id into v_tombstone_id
    from append_markos_audit_row(
      v_row.tenant_id,
      v_row.org_id,
      'governance',
      'audit.pii_erased',
      p_actor_id,
      'owner',
      jsonb_build_object(
        'original_row_id', p_audit_row_id,
        'redaction_marker', p_redaction_marker,
        'reason', p_redaction_marker
      ),
      now()
    ) as t;

  -- 4. Build the redacted payload (deterministic shape per CONTEXT D-106).
  v_redacted_payload := jsonb_build_object(
    '__redacted', p_redaction_marker,
    'at', to_char(now() at time zone 'utc', 'YYYY-MM-DD"T"HH24:MI:SS.MS"Z"'),
    'by', p_actor_id
  );

  -- 5. Recompute row_hash using the locked canonical spec (Plan 01).
  v_new_canonical :=
    '{"action":'      || to_jsonb(v_row.action)::text
    || ',"actor_id":' || to_jsonb(v_row.actor_id)::text
    || ',"actor_role":' || to_jsonb(v_row.actor_role)::text
    || ',"occurred_at":' || to_jsonb(to_char(v_row.occurred_at at time zone 'utc', 'YYYY-MM-DD"T"HH24:MI:SS.MS"Z"'))::text
    || ',"payload":'  || markos_canonicalize_audit_payload(v_redacted_payload)
    || ',"tenant_id":' || to_jsonb(v_row.tenant_id)::text
    || '}';

  v_new_row_hash := encode(digest(v_row.prev_hash || v_new_canonical, 'sha256'), 'hex');

  -- 6. UPDATE the row in place. Service-definer auth bypasses RESTRICTIVE policy.
  --    If this UPDATE fails (e.g., a fault-injection trigger raises), the entire
  --    function rolls back per implicit-txn — proven by erasure-atomicity.test.js.
  update markos_audit_log
    set payload = v_redacted_payload,
        row_hash = v_new_row_hash,
        redacted_at = now(),
        redacted_by = p_actor_id
    where markos_audit_log.id = p_audit_row_id;

  id := p_audit_row_id;
  tombstone_id := v_tombstone_id;
  new_row_hash := v_new_row_hash;
  return next;
end;
$$;

create or replace function recanonicalize_legacy_audit_row(
  p_audit_row_id bigint,
  p_actor_id text
) returns table (id bigint, tombstone_id bigint, old_row_hash text, new_row_hash text)
  language plpgsql
  security definer
  set search_path = public
as $$
declare
  v_row record;
  v_new_canonical text;
  v_new_row_hash text;
  v_tombstone_id bigint;
begin
  select id, tenant_id, org_id, action, actor_id, actor_role, payload, occurred_at, prev_hash, row_hash
    into v_row
    from markos_audit_log
    where markos_audit_log.id = p_audit_row_id;

  end if;

  perform pg_advisory_xact_lock(hashtext('audit:' || v_row.tenant_id));

  select t.id into v_tombstone_id
    from append_markos_audit_row(
      v_row.tenant_id, v_row.org_id, 'system', 'audit.recanonicalized',
      p_actor_id, 'system',
      jsonb_build_object(
        'original_row_id', p_audit_row_id,
        'reason', 'phase_201.1_d103_canonicalizer_lock'
      ),
      now()
    ) as t;

  v_new_canonical :=
    '{"action":'      || to_jsonb(v_row.action)::text
    || ',"actor_id":' || to_jsonb(v_row.actor_id)::text
    || ',"actor_role":' || to_jsonb(v_row.actor_role)::text
    || ',"occurred_at":' || to_jsonb(to_char(v_row.occurred_at at time zone 'utc', 'YYYY-MM-DD"T"HH24:MI:SS.MS"Z"'))::text
    || ',"payload":'  || markos_canonicalize_audit_payload(v_row.payload)
    || ',"tenant_id":' || to_jsonb(v_row.tenant_id)::text
    || '}';

  v_new_row_hash := encode(digest(v_row.prev_hash || v_new_canonical, 'sha256'), 'hex');

  update markos_audit_log
    set row_hash = v_new_row_hash,
        redacted_at = now(),
        redacted_by = p_actor_id
    where markos_audit_log.id = p_audit_row_id;

  id := p_audit_row_id;
  tombstone_id := v_tombstone_id;
  old_row_hash := v_row.row_hash;
  new_row_hash := v_new_row_hash;
  return next;

async function verifyTenantChain(client, tenant_id) {
  if (!client || typeof client.from !== 'function') throw new Error('verifyTenantChain: client required');
  if (typeof tenant_id !== 'string' || !tenant_id) throw new Error('verifyTenantChain: tenant_id required');

  const { data, error } = await client

    .select('id, tenant_id, action, actor_id, actor_role, payload, occurred_at, prev_hash, row_hash, redacted_at')
    .eq('tenant_id', tenant_id)
    .order('id', { ascending: true });
  if (error) throw new Error(`verifyTenantChain: select failed: ${error.message}`);
  const rows = data || [];
  if (rows.length === 0) return { ok: true, row_count: 0, breaks: [] };

  // Phase 201.1 D-106 (closes M4): pre-load tombstones to distinguish redaction from tampering.
  const tombstoneByOriginalId = new Map(); // original_row_id (string|number) -> tombstone row_id
  for (const row of rows) {
    if (row.action === 'audit.pii_erased' || row.action === 'audit.recanonicalized') {
      const origId = row.payload && row.payload.original_row_id;
      if (origId !== undefined && origId !== null) {
        tombstoneByOriginalId.set(Number(origId), row.id);
      }
    }
  }

  const genesis = sha256Hex('genesis:' + tenant_id);
  let expected_prev = genesis;
  const breaks = [];

  for (const row of rows) {
    if (row.prev_hash !== expected_prev) {
      breaks.push({
        row_id: row.id,
        expected_prev_hash: expected_prev,
        actual_prev_hash: row.prev_hash,
        actual_row_hash: row.row_hash,
        reason: 'prev_hash mismatch',
      });
      expected_prev = row.row_hash;
      continue;
    }

    const canonical = canonicalJson({
      action:      row.action,
      actor_id:    row.actor_id,
      actor_role:  row.actor_role,
      occurred_at: typeof row.occurred_at === 'string' ? row.occurred_at : new Date(row.occurred_at).toISOString(),
      payload:     row.payload || {},
      tenant_id:   row.tenant_id,
    });

    const expected_row_hash = computeRowHash(row.prev_hash, canonical);
    if (expected_row_hash !== row.row_hash) {
      // D-106: cross-reference the tombstone map.
      if (tombstoneByOriginalId.has(row.id)) {
        breaks.push({
          row_id: row.id,
          expected_row_hash,
          actual_row_hash: row.row_hash,
          tombstone_row_id: tombstoneByOriginalId.get(row.id),
          reason: 'redaction_tombstone_detected',

          row_id: row.id,
          expected_row_hash,
          actual_row_hash: row.row_hash,
