# Phase 201.1 Plan 02: Atomic Signup Rate-Limit (H6 Closed) Summary

Atomic Postgres `increment_signup_rate` RPC replaces JS read-then-write upsert in `recordSignupAttempt`, closing the concurrent-attacker bypass at the DB layer.

## What Was Built

### Migration 91 — `increment_signup_rate(text, timestamptz) returns int`

`supabase/migrations/91_markos_signup_rate_limit_atomic.sql` creates a single SQL function:

```sql
insert into markos_signup_rate_limits (ip_hash, window_start, attempt_count, updated_at)
  values (p_ip_hash, p_hour_bucket, 1, now())
on conflict (ip_hash, window_start) do update
  set attempt_count = markos_signup_rate_limits.attempt_count + 1,
      updated_at = now()
returning attempt_count;
```

Grant: `authenticated, service_role, anon` (anon required for pre-auth `/api/auth/signup`). Rollback: `drop function if exists`.

### rate-limit.cjs — recordSignupAttempt rewritten

Before (race window):
```javascript
await client.from('markos_signup_rate_limits').upsert({ attempt_count: 1 }, ...);
// Parallel callers all write attempt_count=1, limit bypassed
```

After (atomic):
```javascript
const { data, error } = await client.rpc('increment_signup_rate', { p_ip_hash, p_hour_bucket });
if (error) throw new Error(`recordSignupAttempt: rpc failed: ${error.message}`);
return { attempt_count: typeof data === 'number' ? data : 0 };
```

### rate-limit.ts — return type updated

`Promise<void>` → `Promise<{ attempt_count: number }>` to match new RPC-based implementation.

## H6 Closure Attestation

Review concern H6: "Plan 03 rate-limit race deployed before Plan 08 — concurrent attackers can exceed 5/hour."

Status: **CLOSED**.

- The attack vector (N parallel requests all writing `attempt_count=1`) is eliminated at the Postgres layer via `ON CONFLICT DO UPDATE SET attempt_count = attempt_count + 1`.
- No JS app-layer locking is needed.
- The Plan 03 + Plan 08 deploy-ordering workaround is moot — `recordSignupAttempt` invokes the atomic fn directly.
- 50-concurrent regression test in `test/auth/rate-limit-atomic.test.js` provides ongoing protection.
