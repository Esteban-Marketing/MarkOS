### Task 2: Staging-smoke harness scaffold

**scripts/staging-smokes/lib/runner.cjs** (~20 LoC): shared `runSmoke()` + `summarize()` utilities used by the orchestrator.

**scripts/staging-smokes/run-all.cjs** (~50 LoC): sequential orchestrator; imports 8 smokes, calls `runSmoke()` for each, logs `[ok]`/`[err]`/`[skip]` per result, exits non-zero if any smoke fails (skipped exit 78 treated as PASS in v1).

**scripts/staging-smokes/01-08-*.cjs** (8 files, ~45 LoC each): each exports `{ run, SMOKE_NAME, SMOKE_DESCRIPTION }`. `runDry()` returns a trivial mock pass. `runLive()` returns `{ ok: false, skipped: true, message: 'STAGING_SMOKE_LIVE_NOT_IMPLEMENTED — fill in for v4.0.0-release prep' }` with POSIX exit code 78.

Total scaffold LoC: ~420 across 10 .cjs files.

**scripts/staging-smokes/README.md**: runbook with table of 8 smokes, mode docs, npm run commands, CI gate description, and per-smoke live-mode fill-in references.

**.github/workflows/staging-smokes.yml**: triggers on `workflow_dispatch` (dry/live choice input) and `push: tags: 'v4.0.0-*'`. NOT on `pull_request`. Passes 9 STAGING_* secrets as env vars to the run step. Runs `npm run smokes:staging` (dry) or `npm run smokes:staging:live` based on mode input.

**package.json**: added `"smokes:staging"` and `"smokes:staging:live"` scripts.
