# Phase 201.1 Plan 08: GDPR Signed-URL Hardening (D-102, H3 + M5 closure) Summary

**One-liner:** Server-mediated GDPR download proxy with single-use nonce, 24h TTL, audience claim, and 2 GiB size-bomb cap — review concerns H3 and M5 verifiably closed.

## What Was Built

### Migration 95 (Task 1)

`supabase/migrations/95_markos_gdpr_export_consumed.sql`:
- Adds `nonce text not null`, `audience_tenant_id text not null`, `reissued_at timestamptz` columns to `markos_gdpr_exports`.
- Backward-compat backfill: existing rows get `nonce = id`, `audience_tenant_id = tenant_id`.
- Creates `markos_gdpr_export_consumed(export_id, nonce, consumed_at, consumed_by_session_id, consumed_by_user_id)` with `PRIMARY KEY (export_id, nonce)` — single-use enforcement at the DB layer.
- RLS: `service_role` only. Index on `(export_id)` for the consume-check path.
- Rollback drops table + 3 columns.

Migration slot 95 was confirmed free (88 = mcp_sessions, 89 = mcp_cost_window, 96 = neuro_literacy — avoided as required).

### gdpr-export.cjs Refactor (Task 2)

Key changes:
- `SIGNED_URL_TTL_SECONDS = 86400` (was `604800` — H3 part 4 closed).
- `GDPR_EXPORT_MAX_BYTES = 2_147_483_648` default; overridable via `GDPR_EXPORT_MAX_BYTES_OVERRIDE` env (M5 closed).
- `generateExportBundle` now mints `nonce = randomBytes(32).toString('hex')` (64 hex chars), sets `audience_tenant_id = tenant_id`, stores `signed_url` on the row server-side only. **Return value no longer includes `signed_url`** — caller receives `{ export_id, nonce, expires_at, object_key }` (H3 part 1 closed).
- M5 byte-counter: PassThrough `.on('data')` tracks cumulative bytes; if cap crossed, `archive.abort()` is called, partial object deleted, `markos_gdpr_export_consumed` row marked oversize, audit `gdpr_export.oversize_aborted` emitted, HTTP 413 thrown from `generateExportBundle`.
- New `consumeExportNonce(client, input)`: SELECT export row → verify `audience_tenant_id` → check expiry → check nonce match → INSERT into `markos_gdpr_export_consumed` (PK violation = replay) → audit emit (H3 part 2 + part 3 closed).
- New `reissueExport(client, input)`: owner RBAC via `markos_org_memberships`, rotates nonce, extends TTL 24h, emits `gdpr_export.reissued`.

### New Endpoints

`api/tenant/gdpr/download.js` — `POST /api/tenant/gdpr/download`:
- Requires `X-Markos-Tenant-Id + X-Markos-User-Id + X-Markos-Session-Id` headers.
- Calls `consumeExportNonce`; maps reasons: `consumed|expired` → 410, `audience_mismatch` → 403, `not_found` → 404.
- On success: fetches bytes from R2/S3 server-side via `S3Client.GetObjectCommand`, streams to client as `application/zip`. URL never leaves the server.

`api/tenant/gdpr/reissue.js` — `POST /api/tenant/gdpr/reissue`:
- Re-auth gate: queries `markos_sessions_devices` to verify session still valid.
- Calls `reissueExport`; returns `{ export_id, nonce, expires_at }`.
- Maps `owner role required | audience_mismatch` → 403, `not found` → 404.

### F-107 Contract

`contracts/F-107-gdpr-export-download-v1.yaml`: OpenAPI 3.1 covering both endpoints with full response codes (200, 400, 401, 403, 404, 410, 502).

Note: F-89 was already taken by `F-89-mcp-oauth-v1.yaml` (Phase 202 MCP OAuth). This plan uses F-107.
