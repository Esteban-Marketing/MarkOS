# Phase 201.1 Plan 10: Playwright Tenancy Golden-Path Spec Summary

Single Playwright spec covering the 7-step tenancy hot path (signup email fill and submit, magic-link callback navigation, subdomain middleware routing, sessions check, invite flow, tenant switcher, offboard schedule and slug confirm) with >=6 combined `page.fill` + `page.click` action drives; closes review concern M7 and reverses QA-06 NA for the tenancy hot path.

### 7-step spec content

File: `test-playwright/tenancy-golden-path.spec.ts`

The spec uses a single browser context (cookies persist across navigations) and calls `installTenancyMocks(page)` at the top of the test before any navigation.

| Step | Description | Drive level | Key selectors |
|------|-------------|-------------|---------------|
| 1 | Signup form fill + submit | Full (mandatory) | `input[name="email"]`, `button[type="submit"]` |
| 2 | Magic-link callback navigation | Full (mandatory) | `page.goto(baseUrl/api/auth/callback?token_hash=...)` |
| 3 | Subdomain post-auth navigation | Full (mandatory) | `page.goto(slug.localhost:3000/settings/sessions)` |
| 4 | Invite flow fill + submit | Full preferred / render-check allowed | `#invite-email`, `button:has-text("Invite member")` |
| 5 | Tenant switcher open | Full preferred / render-check allowed | `details[aria-labelledby="switcher-heading"] summary` |
| 6 | Member list render check | Render-check | `h1#members-heading` |
| 7 | Offboard schedule + slug confirm | Full (mandatory) | `button:has-text("Delete workspace")`, `#confirm-name`, `button:has-text("Delete workspace permanently")` |

**Total page.fill + page.click occurrences: 7** (W-5 minimum bar of 6 met).

### Selector adjustments needed against Phase 201 surfaces

Four adjustments were made versus the plan's indicative spec body:

1. **Signup form is email-only.** `app/(marketing)/signup/page.tsx` has only `input[name="email"]` — no slug or displayName fields. Org/tenant provisioning happens post-callback via `provisionOrgAndTenantOnVerify`. Step 1 adjusted to single fill + submit.
2. **Callback param is `token_hash`, not `token`.** `api/auth/callback.js` reads `urlParams.searchParams.get('token_hash')`. Spec uses `token_hash=` in the URL.
3. **Callback redirects to `/onboarding`.** `api/auth/callback.js` redirects to `${baseUrl}/onboarding`. Step 2 assertion updated to match `/onboarding|/dashboard|/app|/welcome`.
4. **TenantSwitcher uses `<details>/<summary>` with no `data-testid`.** `components/markos/tenant/TenantSwitcher.tsx` renders `<details aria-labelledby="switcher-heading">`. Spec targets `details[aria-labelledby="switcher-heading"] summary` with a render-check fallback path for step 5.
5. **Danger page confirm input has `id="confirm-name"` but no `name` attribute.** `app/(markos)/settings/danger/page.tsx` uses `id="confirm-name"`. Spec uses `#confirm-name` selector (no `name` attr needed).

No `data-testid` attributes were added to any Phase 201 surface — alternate working selectors were found for all mandatory steps.

### Step degradations

- **Step 4 (invite):** Full action drive attempted with `#invite-email` fill + `button:has-text("Invite member")` click. Render-check fallback on `h1#members-heading` if API returns non-201 (seat quota). The fallback is environment-dependent, not a spec shortcut.
- **Step 5 (switcher):** Full action drive attempted via `details[aria-labelledby="switcher-heading"] summary` click. Render-check fallback if switcher is not rendered in the layout shell on test routes. Documents `console.warn` if switcher absent.
- **Step 6 (member list):** Render-check on `h1#members-heading`. Allowed per W-5.

Steps 1, 2, 3, 7 are unconditional full action drives with no fallback paths.

### Known limitations

- Tests run against ephemeral local Next.js dev server with in-process route mocks; no live Supabase required for the mock path.
- `<slug>.localhost:3000` subdomain simulation requires Next.js dev server to handle wildcard host headers (set `NEXT_PUBLIC_APEX_DOMAIN=localhost:3000`).
- Step 2 falls back to direct navigation when `SUPABASE_SERVICE_ROLE_KEY` is absent, leaving magic-link token verification unchecked on that CI run.
- Broader Playwright harness (additional browsers, sad paths, multi-tenant) deferred per D-111.

## M7 Closure Attestation

Review concern M7: "Tenancy phase is exactly where E2E pays rent. Unit tests cannot catch SameSite/cookie scope/middleware/edge-config issues."

This plan closes M7 for the tenancy hot path. The single spec exercises signup, magic-link callback, subdomain middleware routing, sessions, invite, switcher, and offboard under Playwright — all failure modes that unit tests cannot catch.

**QA-06 NA declaration reversed for the tenancy hot path.** QA-06 remains NA for the broader Playwright harness (additional surfaces, browsers, sad paths) — that scope is still deferred to a future testing-infra phase per D-111.

- **Found during:** Task 1 surface inspection

**5. [Rule 1 - Bug] Danger confirm input uses id not name**
