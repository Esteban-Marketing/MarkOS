# Plan 220-03 - MarketingEvent Workflow and Attribution

**Phase:** 220 - SaaS Community, Events, PR, Partnerships, Developer Marketing, and Growth Agent Surface  
**Status:** Planned  
**Requirements:** SG-06, LOOP-06, LOOP-08, QA-01..15

## Objective

Create event marketing workflows from planning through attribution.

## Scope

- Define `MarketingEvent` with event type, date, platform, promotion plan, speakers, reminders, replay, no-show/attendee sequences, and attribution.
- Link event actions to tasks, approvals, content, CRM, and revenue evidence.

## Acceptance Criteria

- Event workflows are not passive calendars.
- Attendee/no-show/replay actions are approval-aware.
- Event results feed attribution and learning.
