# Phase 13 — Plan 03: Conversational Interview & Routing

## Objective
Implement the dynamic, context-aware gap-filling interview. Once Step 0 finishes, render the Business Model selector. Then, the LLM constructs conversational questions covering only the missing (Red/Yellow) fields based on the business model.

## Tasks

### 1. Business Model Routing & Question Logic
<action>
Create `onboarding/backend/prompts/grouping-prompt.js`:
Export `getGroupingPrompt(businessModel, missingFields)` to generate 1 natural-language question that covers 2-3 fields at once (e.g., "competitors" and "market_maturity"). It should only ask for fields that are in `missingFields` array, which comprises fields scored 'Red' or 'Yellow'.
In `server.cjs` add `POST /api/generate-question`: Takes `businessModel`, `missingFields` array, and `previousAnswers`, calling LLM via `llm-adapter.cjs` and returning a generated string question.
</action>

### 2. Conversational Frontend Interface
<action>
In `onboarding/index.html` and `onboarding.js`:
- Remove Steps 2, 3, 4, 5, 6 DOM structures manually to rely on dynamic conversational interface.
- Add an explicit "Step 1: Business Model" card in the UI (a direct `<select id="businessModel">` form element). This must be answered before anything else.
- Add a new conversational container `<div id="interview-container">` simulating a chat window.
- The UI fetches a question from `/api/generate-question` based on the Red/Yellow keys and renders it as an AI message. User types a free-form response, which is sent to a new `/api/parse-answer` endpoint.
- Add `POST /api/parse-answer` to `server.cjs` which aggregates the free-text response, uses `schema-extractor.cjs` (from Plan 02) partial-mapping mode, updates the seed data in-memory, recalculates scores, and returns updated Red/Yellow fields.
- Continue fetching questions until no Red/Yellow fields remain (or the user bails out).
</action>
