# Phase 13 — Plan 06: Step 7 Review Dashboard

## Objective
Revamp Step 7 from its current AI Draft Review role into a full 30-field schema dashboard with inline editing, source badges (`[Chat]`, `[File]`, `[Web]`), confidence indicators, and a dual-tab layout (Tab 1: Schema Review, Tab 2: Draft Previews).

## Tasks

### 1. Build Tabbed Interface & Schema Review UI
<action>
In `onboarding/index.html` `onboarding.css` and `onboarding.js`:
- Replace existing `Step 7` content with a tabbed component: `<div class="tabs">` with `tab-schema` (active) and `tab-drafts`.
- In `tab-schema`: Build a dashboard grid iterating directly over the `onboarding-seed.schema.json` categories (Company, Product, Audience, Competition, Market, Content).
- Generate a repeating CSS Grid card layout. Each card represents a schema field, displaying its internal title, its populated string/array value, and a small badge spanning its source mapping state.
- Update `onboarding.css` to add `badge-source-*` (e.g. `.badge-source-chat` with green tint, `.badge-source-file` with blue tint, `.badge-source-web` with orange tint) based on the `13-UI-SPEC.md` colors.
</action>

### 2. Inline Editing & Source Tags
<action>
In `onboarding/onboarding.js` and `onboarding/index.html`:
- For each generated field card in `tab-schema`, add an inline `contenteditable` span or standard `<input class="inline-edit">`.
- Trigger an `onblur` event that syncs the manually modified value back up to the primary application seed JSON map, updating its source state to `[Manual]`.
- Ensure Step 7's final submit explicitly checks `completeness_score` limits from the actual `onboarding-seed` and executes the original POST to `/api/submit` to run `orchestrator.cjs`.
- Tab 2 (`tab-drafts`) should retain the existing logic that renders MIR/MSP previews exactly as it does today.
</action>
