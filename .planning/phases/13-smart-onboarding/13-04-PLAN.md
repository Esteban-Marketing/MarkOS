# Phase 13 — Plan 04: AI Spark & Cascade Strategy

## Objective
Implement the LLM cascade fallback architecture (BYOK → Ollama → Fallback) and the "Spark ✨" icon generating per-field inline suggestions based on collected context.

## Tasks

### 1. Implement LLM Run-Time Cascade
<action>
Modify `onboarding/onboarding-config.json` to add:
`"ollama_model": "llama3:8b"`
Modify `onboarding/backend/agents/llm-adapter.cjs`:
- If `process.env.OPENAI_API_KEY` (or Anthropic/Gemini) exists, run the current BYOK flow (Tier 1).
- If no keys exist, make a POST to `http://localhost:11434/api/generate` with `model: config.ollama_model` and `prompt: ...` (Tier 2).
- Return an error code (e.g. `{ error: 'NO_AI_AVAILABLE' }`) if neither BYOK nor Ollama responds successfully (Tier 3).
</action>

### 2. Implement AI Spark Context Generation
<action>
Create `onboarding/backend/prompts/spark-prompt.js`:
Export `getSparkPrompt(fieldName, contextString)`.
Instruct the LLM to provide 3 distinct, concise alternatives (as an array of strings in JSON) for the given `fieldName` strictly using the `contextString`.
In `server.cjs`, add `POST /api/spark-suggestion`: accepts `fieldName` and `currentState` object. It packages `currentState` (scraped + files + previous answers) as `contextString`. Call `llm-adapter.cjs` with `getSparkPrompt` and return `{ suggestions: [...] }`.
</action>

<acceptance_criteria>

### 3. Frontend Spark Interaction
<action>
In `onboarding/index.html` and `onboarding.js`:
- Within the Chat UI inputs and manual forms, append a `✨` icon (using CSS SVG inline) next to active text fields.
- On click, show a loading tooltip. Make a request to `POST /api/spark-suggestion` sending the specific field name and global `onboarding-seed` context.
- Render the 3 returned strings as a small popover. Clicking an option replaces the input value.
- If backend responds with `NO_AI_AVAILABLE`, disable the `✨` button and set `title="Enable AI: add an API key or install Ollama"`.
</action>
