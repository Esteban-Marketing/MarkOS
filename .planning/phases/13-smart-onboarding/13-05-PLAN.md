# Phase 13 — Plan 05: Auto-Enrichment & "Skip Chat" Bailout

## Objective
Post-scrape, run a targeted web search to find competitors and market trends if missing, presenting an AI-proposed confirmation card. Also, implement the persistent "Skip chat -> Show full form" bailout action allowing users to bypass the conversational interface.

## Tasks

### 1. Auto-Enrichment Engine
<action>
Create `onboarding/backend/prompts/enrichment-prompt.js`:
Export `getEnrichmentPrompt(companyName, industry)`. Instruct LLM to generate list of 3 competitors and 1 major market trend based on the input text. Return `{ competitors: [{ name, differentiator }], biggest_trend: '' }` JSON.

Create `onboarding/backend/enrichers/competitor-enricher.cjs`:
Read `onboarding-config.json` for `tavily_api_key`. If exists, run search for `<companyName> competitors` and `<industry> market leaders 2026`. Feed raw search results to `llm-adapter.cjs` using `enrichment-prompt.js` and return structured JSON.

In `server.cjs`, add `POST /api/competitor-discovery`: accepts `companyName` and `industry`. If both exist, calls `competitor-enricher.cjs` and returns `{ success: true, enrichedData }`.
</action>

### 2. Present Enrichment Card
<action>
In `onboarding/onboarding.js`:
- After Business Model selection (from Step 03), if `competition.competitors` array is empty or low confidence, trigger `POST /api/competitor-discovery`.
- On success, render a special AI Chat Card: *"We found that your likely competitors are [A], [B], [C]. Does this sound right?"* with input fields allowing the user to confirm/edit these values before updating the seed.
</action>

### 3. "Skip Chat" Bailout Action
<action>
In `onboarding/index.html`:
- Render a persistent button `<button id="btnBailout">Skip chat → Show full form</button>` in the chat container footer.
- Wrap the original Step 1-6 form fields into a `<section id="manual-form-bailout" class="d-none">`.

In `onboarding/onboarding.js`:
- Add an event listener to `btnBailout`. When clicked, hide the chat container and remove `d-none` from `manual-form-bailout`.
- Populate all form fields (`companyName`, `industry`, `mission`, `painPoint1`, etc.) from the current global JSON state (which contains scraped/answered data).
</action>
