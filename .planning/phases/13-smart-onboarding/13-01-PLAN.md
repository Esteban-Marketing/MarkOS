# Phase 13 — Plan 01: Omni-Input Gate & Extractors

## Objective
Build the Step 0 Omni-Input Gate that accepts a website URL and/or uploaded files, returning raw extracted text for downstream LLM mapping. Includes the terminal-style loading mechanism in the UI.

## Tasks

### 1. Build Scraper and Parsers
<action>
Create modular parsers in `onboarding/backend/parsers/` and `onboarding/backend/scrapers/`.
- `tavily-scraper.cjs`: Expect URL. Use Tavily API (fetch) with `include_raw_content: true`, `max_results: 15`, scraping homepage and max depth 2 links (`/about`, `/pricing`, `/product`, `/blog`). Limit timeout to 30000ms.
- `pdf-parser.cjs`: Use `pdf-parse` library to extract text from buffer.
- `docx-parser.cjs`: Use `mammoth` library to extract raw text from buffer.
- `csv-parser.cjs`: Use `csv-parse/sync` to parse buffering strings.
- `text-parser.cjs`: Basic string decoding (UTF-8) for TXT and MD, stripping basic Markdown syntax.
Add an Express endpoint `POST /api/extract-sources` in `server.cjs` that accepts multipart/form-data with a `url` field and `files[]` array. Max files 5, max total size 10MB.
</action>

<action>
