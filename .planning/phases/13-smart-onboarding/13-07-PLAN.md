# Plan: Phase 13 Gap Closure (Smart Onboarding Hardening)

This plan addresses the critical blockers identified during UAT, specifically the `NO_AI_AVAILABLE` failures and the "Step 3 progression trap."

## Gaps to Close
1.  **AI Resilience**: System crashes or shows broken placeholders when no LLM is available.
2.  **Dashboard Trap**: "Finish/Publish" button is hidden if draft generation fails.
3.  **Conversational Dead-end**: Chat input is disabled when AI fails, leaving the user stranded.

## Proposed Changes

### 1. LLM Adapter: Add Static Fallback
- Modify `onboarding/backend/agents/llm-adapter.cjs` to include a `getFallbackResponse(systemPrompt, userPrompt)` function.
- If all providers fail (or `NO_AI_AVAILABLE` is detected), return a structured mock response instead of `ok: false`.
- For specific prompts (MIR/MSP), return highly descriptive "template" placeholders.

### 2. Orchestrator: Graceful Error Collection
- Modify `onboarding/backend/agents/orchestrator.cjs` to ensure it never returns `ok: false` for the entire batch if one agent fails.
- Collect errors but ensure `drafts` object is always populated (even if with fallbacks).

### 3. Frontend: Decouple Progression
- Modify `onboarding/onboarding.js`:
    - Ensure `publishBar` is shown even if `result.success` is false (as long as `lastSlug` exists).
    - Ensure "Next" button in the Dashboard enables "Review & Finish" even if AI drafts are empty.
    - Add a "Manual Entry Mode" toggle/state that bypasses AI generation if it fails 3 times.

### 4. UI: Improved Error States
- Update `index.html` to include a "Proceed without AI" button in the error state of the Drafts grid.

## Verification Steps
- Set `ANTHROPIC_API_KEY=""` and stop Ollama.
- Run onboarding.
- Confirm Step 3 (Dashboard) is reachable.
- Confirm "Review Drafts" shows placeholders and allow "Publishing" to MIR.
