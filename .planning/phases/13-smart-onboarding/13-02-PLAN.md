# Phase 13 — Plan 02: Confidence Scoring & LLM Extraction

## Objective
Take raw text streams extracted from Step 0 and map them to the `onboarding-seed.schema.json` v2.1 via an LLM. Assign Red/Yellow/Green confidence scores for each mapped field.

## Tasks

### 1. Build Strict JSON Schema Extractor
<action>
Create `onboarding/backend/prompts/extraction-prompt.js`:
Export a function `getExtractionPrompt(schema, text)` that enforces strict JSON map return. "Output ONLY valid JSON matching this schema: [SCHEMA]. Do not add keys. If a field is unknown, output null. Never explain."

Create `onboarding/backend/extractors/schema-extractor.cjs`:
Read `onboarding-seed.schema.json`. Use `llm-adapter.cjs` to pass the `extraction-prompt.js` prompt + the raw text (from Step 0).
Export `extractToSchema(rawText)`. Include a try/catch JSON parse block with 1 automatic retry if output fails to parse.
</action>

- onboarding/backend/agents/llm-adapter.cjs
- onboarding/onboarding-seed.schema.json
- .planning/phases/13-smart-onboarding/13-CONTEXT.md
</read_first>

<acceptance_criteria>
- `extraction-prompt.js` exports `getExtractionPrompt(schema, text)`.
- `schema-extractor.cjs` exports `extractToSchema(rawText)` which returns a JSON object perfectly matching the schema structure.
- `schema-extractor.cjs` retries exactly 1 time if `JSON.parse` fails.
</acceptance_criteria>

### 2. Implement R/Y/G Confidence Scorer
<action>
Create `onboarding/backend/confidences/confidence-scorer.cjs`.
Export `scoreFields(extractedJson)`.
It evaluates each field independently (ignoring keys not in schema) and assigns a score:
- Green (>=85%): Field has detailed content (e.g., arrays have 2-3 items, text is > 10 chars).
- Yellow (50-84%): Field is populated but sparse (text < 10 chars, array has 1 item, or vague).
- Red (<50%): Field is null, empty string, or empty array.

Modify `server.cjs` to add `POST /api/extract-and-score`. It should call `extractToSchema` with the aggregated text from Step 0, then run `scoreFields`, and return `{ data: extractedJson, scores: calculatedScores }` to the frontend.
</action>

<acceptance_criteria>
- `confidence-scorer.cjs` exports `scoreFields(extractedJson)` returning an object of identical shape but values are `{ value, score: 'Green'|'Yellow'|'Red' }`.
- `server.cjs` has endpoint `POST /api/extract-and-score` returning the scored mapping.
</acceptance_criteria>
