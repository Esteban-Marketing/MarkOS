# Plan 207-05 - Cost Estimate, Actual Cost, Billing, and Pricing Context

**Phase:** 207 - AgentRun v2 Orchestration Substrate  
**Status:** Planned  
**Requirements:** RUN-02, BILL-01, PRC-09, QA-01..15

## Objective

Make run-level cost visible before, during, and after execution.

## Scope

- Add estimated cost and actual cost fields to AgentRun v2 contracts.
- Map provider attempts and run close events to billing usage events.
- Preserve model, provider, prompt version, token, latency, and tool usage lineage.
- Add Pricing Engine placeholder/context rules for pricing-sensitive agent work.
- Ensure cost failures create tasks or budget warnings.

## Acceptance Criteria

- Estimated and actual costs are visible at run level.
- Billing usage events can be derived idempotently from run close/provider attempt events.
- Pricing-sensitive runs carry Pricing Engine context or `{{MARKOS_PRICING_ENGINE_PENDING}}`.

## Verification

- Cost reconciliation tests cover estimates, actuals, retries, and failed runs.
- Billing-normalizer fixtures accept AgentRun v2 close and provider attempt payloads.
