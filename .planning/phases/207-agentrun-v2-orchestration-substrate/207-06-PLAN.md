# Plan 207-06 — Adoption Path for CRM, MCP, Onboarding, and Future Agent Readiness

## Objective

Wire the existing three run-producing domains (CRM playbooks, MCP pipeline, onboarding run-engine) to the v2 substrate WITHOUT breaking their contracts. Ship `markos_agent_registry` with readiness posture for planned SaaS/PRC/growth agents per SG-10. Complete migration 106 + F-111 YAML. Establish compatibility integration tests proving CRM and MCP continue to work while speaking v2.

## Current code path this migrates

- **`lib/markos/crm/playbooks.ts`** — still imports `createRunEnvelope`, `assertTransitionAllowed`, `recordSideEffect` from `onboarding/backend/agents/run-engine.cjs`. Adapter injects v2 persistence without changing the import surface.
- **`lib/markos/mcp/pipeline.cjs`** — 10-step middleware pipeline stays intact. A new `run-emitter.cjs` registers an AgentRun around each tool-call so MCP invocations become first-class runs in `markos_agent_runs`.
- **`onboarding/backend/agents/run-engine.cjs`** — becomes a thin compatibility shim: keeps `RUN_STATES`, `ALLOWED_TRANSITIONS`, and the in-memory helpers (used by existing tests), BUT now also exports pass-throughs to v2 primitives from `lib/markos/agents/run-api.ts`. Legacy imports keep working.
- **`api/crm/copilot/playbooks.js`** — unchanged; its adapter layer in this plan picks up v2 silently.

## must_haves

- Migration 106 complete: `markos_agent_registry` with 6 columns + RLS + indexes.
- `lib/markos/agents/registry-v2.ts` seed data lists planned agents per D-207-06 and CONTRACT-LOCK §10, including: PRC-* (pricing), SAS-* (SaaS Suite), SG-* (growth), CRM-* (CRM copilot), MCP-* (MCP tools), ONB-* (onboarding).
- CRM playbooks continue to pass their existing tests while also writing v2 run/event rows.
- MCP tool invocation produces a `markos_agent_runs` row with `trigger_kind: 'mcp'`, cost estimate/actual via 207-05, and side-effect guard via 207-04.
- `onboarding/backend/agents/run-engine.cjs` keeps its v1 exports and adds v2 re-exports — no breaking change.
- 2 new endpoints: `GET /api/tenant/agents` (list registry) + `GET /api/tenant/agents/{agent_id}`.
- F-111 YAML complete.
- 2 integration tests prove round-trip: CRM playbook → AgentRun row; MCP tool-call → AgentRun row with cost.
