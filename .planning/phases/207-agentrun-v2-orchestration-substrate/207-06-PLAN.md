# Plan 207-06 - Adoption Path for Existing and Future Agents

**Phase:** 207 - AgentRun v2 Orchestration Substrate  
**Status:** Planned  
**Requirements:** RUN-01..08, MCP-01, PRC-06, SAS-10, SG-10, QA-01..15

## Objective

Move existing agent surfaces onto AgentRun v2 while preventing future target agents from becoming runnable too early.

## Scope

- Identify first adopters: CRM copilot, MCP marketing tools, claim audit tools, and onboarding agents.
- Define a registry field for `planned`, `research_ready`, `contract_ready`, `executable`, and `disabled`.
- Block PRC, SAS, PLG, ABM, and other future growth agents from executable status until their phase contracts exist.
- Define migration notes for in-memory run engine users.

## Acceptance Criteria

- Existing agent surfaces have an adoption order.
- Future target agents are discoverable but not runnable without readiness gates.
- Agent registry status aligns with `.agent/markos` and `.planning/REQUIREMENTS.md`.

## Verification

- Registry tests prove planned agents cannot execute.
- Migration checklist maps every current agent entrypoint to v2 adoption status.
