**Phase:** 207 - AgentRun v2 Orchestration Substrate  
**Status:** Planned  
**Requirements:** RUN-01..08, TASK-01, BILL-01, QA-01..15

## Objective

Define the durable AgentRun v2 object family before runtime code expands.

## Scope

- Reserve fresh contract IDs and migration IDs for AgentRun v2, AgentRunEvent, AgentChain, AgentChainEdge, AgentSideEffect, and AgentFailure.
- Extend or replace the current `markos_agent_runs`, `markos_agent_run_events`, and `markos_agent_side_effects` model without losing tenant isolation.
- Add fields for trigger, priority, source surface, model, tokens, estimated cost, actual cost, retry classification, approval policy, task link, and policy metadata.
- Define RLS, audit, rollback, and support-visibility posture.

## Acceptance Criteria

- Every v2 run object is tenant-scoped and audit-ready.
- Existing v1 lifecycle data has a migration or compatibility path.
- Fresh IDs avoid incoming-document F-ID collisions.
- Schema supports CLI, UI, MCP, and agent runtimes as consumers.
