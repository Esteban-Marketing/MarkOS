# Plan 207-03 - Priority, Concurrency, Retry, Timeout, Pause, Cancel, and DLQ Policy

**Phase:** 207 - AgentRun v2 Orchestration Substrate  
**Status:** Planned  
**Requirements:** RUN-03, RUN-04, RUN-05, QA-01..15

## Objective

Make scheduling and failure behavior explicit before agent volume grows.

## Scope

- Define P0-P4 priority behavior, tenant concurrency limits, and starvation protection.
- Define retry classes, retry-after behavior, timeout policy, cancellation, pause/resume, and DLQ entry.
- Define which failures create tasks immediately and which can retry silently.
- Add emergency behavior for security, compliance, live spend leak, and revenue-critical cases.

## Acceptance Criteria

- Priority tiers map to documented queue behavior.
- Retry and DLQ states are visible on runs and tasks.
- Paused/canceled work does not continue external side effects.
- P0 behavior is powerful but bounded to prevent starvation.

## Verification

- State machine tests cover allowed and forbidden transitions.
- Scheduler tests cover tenant concurrency and starvation caps.
