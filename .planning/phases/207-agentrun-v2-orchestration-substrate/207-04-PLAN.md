# Plan 207-04 - ApprovalGate and Task Handoff Contracts

**Phase:** 207 - AgentRun v2 Orchestration Substrate  
**Status:** Planned  
**Requirements:** RUN-05, RUN-06, TASK-01, TASK-04, QA-01..15

## Objective

Ensure every run output, blocked action, approval need, and failure can become human-visible work.

## Scope

- Define run-to-task and run-to-approval links.
- Define standard task handoff payloads for failure, recovery, external mutation, evidence gap, pricing gap, and connector degradation.
- Add approval policy metadata for external-world mutations.
- Ensure rejection reasons and edits flow back into run state.

## Acceptance Criteria

- No agent output disappears without task, approval, archive, or explicit low-risk policy.
- External side effects require approval metadata and decision history.
- Failed and rejected runs produce visible owner/action context.

## Verification

- Fixture tests prove run failure creates task payloads.
- Approval tests prove mutation attempts cannot proceed without approval state.
