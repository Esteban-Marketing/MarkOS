# Plan 207-02 — Durable Run API and Event Stream

## Objective

Ship the durable `POST /api/tenant/runs` creation endpoint, `GET /api/tenant/runs/{run_id}` read endpoint, and `GET /api/tenant/runs/{run_id}/events` SSE stream that every surface (CLI, UI, MCP, webhook) reads from. Complete migration 101 (state enum + v2 columns) and F-106 YAML. Replace `onboarding/backend/agents/run-engine.cjs` in-memory helpers with a durable adapter that writes to Supabase `markos_agent_runs` + `markos_agent_run_events`.

## Current code path this migrates

- `onboarding/backend/agents/run-engine.cjs` — v1 in-memory stores (`createInMemoryEventStore`, `createInMemorySideEffectLedger`) remain for tests but gain a durable parallel path `createSupabaseEventStore` in `lib/markos/agents/run-events.ts`.
- `supabase/migrations/53_agent_run_lifecycle.sql` — extended via migration 101 (state check expanded; v2 columns added).
- `api/crm/copilot/playbooks.js` — unchanged in this plan; 207-06 wires adapter.

## must_haves

- Migration 101 complete: state check constraint includes all 15 states from CONTRACT-LOCK §5; columns `priority, chain_id, parent_run_id, agent_id, agent_registry_version, trigger_kind, source_surface, approval_policy` exist.
- POST `/api/tenant/runs` creates a run and returns the AgentRun envelope; emits `agent_run_transitioned requested→queued` event.
- GET `/api/tenant/runs/{run_id}` returns the AgentRun + last 100 events.
- GET `/api/tenant/runs/{run_id}/events` streams SSE with `AgentRunEvent` JSON per line; monotonic `seq` per run.
- F-106 YAML `paths` block is complete (3 paths), `x-markos-phase: 207-01-PLAN` placeholder removed.
- `lib/markos/agents/run-api.ts` exports `createRun(input)`, `getRun(runId)`, `transitionRun(runId, toState, reason)` calling `isTransitionAllowedV2`.

    Create 3 modules:

    **`lib/markos/agents/run-store.ts`** — durable Supabase store. Exports:
    - `createSupabaseRunStore(client)` returning `{ insertRun, getRun, updateRunState, listRuns, getRunEvents, appendRunEvent }`.
    - Every write goes through a Postgres transaction; `appendRunEvent` computes next `seq` via `select coalesce(max(seq),0)+1 from markos_agent_run_events where run_id = $1 for update`.
    - Every read respects tenant RLS (caller passes tenant_id + actor's Supabase JWT).

    **`lib/markos/agents/run-events.ts`** — event emitter with validation. Exports:
    - `emitRunEvent({ runId, tenantId, eventType, fromState, toState, actorId, correlationId, reason, payload })` that validates via `AgentRunEvent.parse` and calls `appendRunEvent`.
    - `subscribeRunEvents(runId, { since_seq })` returns an async iterator for SSE streaming — polls Supabase every 500ms or uses Supabase realtime channel.

    **`lib/markos/agents/run-api.ts`** — high-level API. Exports:
    - `createRun(input: z.input<typeof AgentRun>)` validates, inserts into `markos_agent_runs` with initial state `queued`, emits `agent_run_queued` event, returns the AgentRun.
    - `getRun(runId, tenantId)` returns `{ run, events: AgentRunEvent[] }` (last 100 events).
    - `transitionRun({ runId, tenantId, toState, actorId, reason })` — calls `isTransitionAllowedV2(fromState, toState)`; on `false`, emits `agent_run_transition_denied` and throws `AGENT_RUN_INVALID_TRANSITION`; on `true`, updates state + emits `agent_run_transitioned`.
    - `cancelRun({ runId, tenantId, actorId, reason })` — transitions to `canceled` from any non-terminal state, emits `agent_run_canceled`.
  </action>

  <action>
    Create 3 Vercel API routes:

    **`api/tenant/runs/index.js`** — POST only. Auth via bearer (Supabase JWT OR markos_cli_api_keys). Body validated via `AgentRun.parse` minus server-assigned fields. Calls `createRun()`. Returns 201 with AgentRun JSON.

    **`api/tenant/runs/[run_id]/index.js`** — GET + DELETE. GET calls `getRun()`. DELETE calls `cancelRun()` with actor from bearer. Returns 200 JSON `{ run, events }` for GET, 204 for DELETE.

    **`api/tenant/runs/[run_id]/events.js`** — GET only; serves SSE (`Content-Type: text/event-stream`). Accepts `?since_seq=<N>` query. Uses `subscribeRunEvents(runId, { since_seq })` iterator; emits one `data: <json>\n\n` per event. Closes stream when run reaches terminal state (`completed`, `archived`, `canceled`, `dlq`, `rejected`).

    All endpoints enforce tenant scoping — `run.tenant_id` must match authenticated principal.
  </action>
