# Plan 207-02 - Durable Run API and Event Stream

**Phase:** 207 - AgentRun v2 Orchestration Substrate  
**Status:** Planned  
**Requirements:** RUN-01, RUN-07, RUN-08, CLI-01, MCP-01, QA-01..15

## Objective

Create the shared run API and event stream that CLI, UI, MCP, and agents can all consume.

## Scope

- Define create, read, list, cancel, pause, resume, and event-stream contracts.
- Align `markos run`, `markos status`, task UI, and MCP resources around one run/event substrate.
- Ensure events are append-only, ordered, support-visible, and safe for SSE or polling.
- Preserve trace IDs and request correlation for debugging and SOC2 evidence.

## Acceptance Criteria

- `markos run/status` can consume the same run state as the UI.
- Run events expose enough information for support without leaking secrets.
- Failed and waiting states are explicit, not inferred from logs.
- API contracts include auth, rate limit, and tenant scoping.

## Verification

- Contract tests cover event ordering, tenant scoping, and forbidden access.
- CLI/UI/MCP consumer fixtures all parse the same event payload.
