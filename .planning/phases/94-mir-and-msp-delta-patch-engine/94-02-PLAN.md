Expected preview engine inputs:
- context_pack.filters and request-level tailoring filters
- evidence items with implication, freshness, authority class, and confidence metadata
- contradiction objects or warnings from the Phase 93 context pack
- existing target note snapshot when available

Expected engine outputs:
- section_key and target heading path
- diff mode such as `section_replace`, `section_append`, or `metadata_refresh`
- before_excerpt, after_excerpt, and change_rationale
- evidence IDs attached to each changed block
- downgrade metadata when a concrete patch cannot be justified

<threat_model>
- T-94-05 (wrong target resolution): evidence could map to the wrong MIR/MSP block and produce misleading updates. Mitigation: resolve targets through stable section keys, destination mapping, and filter-aware heuristics backed by tests.
- T-94-06 (cross-artifact bleed): the engine could update more than the affected section or drift across unrelated notes. Mitigation: keep patch generation narrowly scoped to one resolved target block at a time with explicit diff modes only.
- T-94-07 (hidden contradiction collapse): mixed evidence could be synthesized into a clean patch without exposing conflict. Mitigation: carry contradictions forward into warnings and downgrade logic instead of flattening them.
</threat_model>
