# Phase 11 — Plan 02: Example Resolver Utility

**Phase:** 11 — Rich Business-Model Examples

## Objective

Create `example-resolver.cjs` — a shared utility that resolves the correct model-specific sibling example file for a given template, wraps its content in the standardised injection block, and is imported by both `mir-filler.cjs` and `msp-filler.cjs`.

## Deliverables

- `onboarding/backend/agents/example-resolver.cjs` — **NEW**
  - `resolveExample(templateName, businessModel, slug, basePath)` → injection block string
  - Slug normalization: `Agents-aaS` → `agents-aas`
  - Graceful degradation: returns `''` if model unknown or file missing

## Acceptance Criteria

- [x] Reads `_TEMPLATE-{model}.example.md` from given `basePath`
- [x] Wraps content in `## 📌 Reference Example ({Model})` header
- [x] Returns empty string (not a throw) for missing files
- [x] Exported as `{ resolveExample }`
