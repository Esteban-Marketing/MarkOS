# Plan 217-01 - SaaS Revenue Metric Definitions and Source Precedence

**Phase:** 217 - SaaS Revenue Intelligence, SAS Agents, API/MCP/UI Readiness  
**Status:** Planned  
**Requirements:** SAS-09, LOOP-06, QA-01..15

## Objective

Define SaaS revenue metrics and source precedence before dashboards or agents use them.

## Scope

- Define MRR, ARR, NRR, GRR, churn, expansion, contraction, cohorts, forecast, and waterfall metrics.
- Define precedence across MarkOS billing, processor, accounting, CRM, product usage, and manual corrections.
- Define reconciliation and degraded states.

## Acceptance Criteria

- Every revenue metric has formula, source, timestamp, and confidence/reconciliation state.
- Conflicting sources resolve predictably.
- Revenue claims can link to evidence.

## Verification

- Formula tests cover edge cases: trials, pauses, discounts, refunds, upgrades, downgrades, partial periods, and churn.
