P217 Revenue Metric Formula Table (RESEARCH §Domain 1 — used to seed saas_revenue_metric_definitions):
| Metric Key | Formula | Source Precedence (priority order) |
|------------|---------|-----------------------------------|
| mrr | Sum of (monthly normalized recurring revenue per active subscription) | [billing, processor, accounting, crm, manual] |
| arr | MRR × 12 | [derived_from_mrr] |
| new_mrr | Sum of MRR from subscriptions started within period | [billing, processor] |
| expansion_mrr | Sum of MRR increase from existing subscriptions (upgrades/seat adds) | [billing, processor] |
| contraction_mrr | Sum of MRR decrease (downgrades/seat reductions; stored negative) | [billing, processor] |
| churn_mrr | Sum of MRR from canceled subscriptions (stored negative) | [billing, processor] |
| reactivation_mrr | Sum of MRR from reactivated subscriptions | [billing, processor] |
| net_new_mrr | new + expansion + contraction + churn + reactivation | [derived_waterfall] |
| nrr | ((MRR_start + Expansion − Contraction − Churn) / MRR_start) × 100 | [derived_from_waterfall] |
| grr | ((MRR_start − Contraction − Churn) / MRR_start) × 100 | [derived_from_waterfall] |
| churn_rate_logo | Churned customers / Customers at period start × 100 | [billing] |
| churn_rate_revenue | Churn MRR / MRR_start × 100 | [derived_from_mrr] |
| ltv | ARPU / Churn Rate (monthly) | [derived] |
| cohort_retention | MRR retained per acquisition cohort at T+1, T+3, T+6, T+12 | [billing, subscription_history] |
| forecast_3m | 3-month linear extrapolation from trailing 6-month MRR trend | [derived_from_mrr_history] |

5-tier source-precedence model (RESEARCH §Domain 1):
1. billing (P215 tenant_billing_subscriptions) — authoritative for lifecycle → reconciled
2. processor (Stripe/Mercado Pago signed event) — authoritative for payment facts → reconciled if matches billing
3. accounting (QuickBooks/Siigo/Alegra export) — authoritative for invoice facts → reconciled if within tolerance
4. crm (CRM subscription record) — informational → pending_reconciliation
5. manual (operator-entered) — override → manually_corrected (requires approval)
