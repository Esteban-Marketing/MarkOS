# Plan 217-02 - SaaSMRRSnapshot and Revenue Intelligence Records

**Phase:** 217 - SaaS Revenue Intelligence, SAS Agents, API/MCP/UI Readiness  
**Status:** Planned  
**Requirements:** SAS-09, EVD-01..06, QA-01..15

## Objective

Create the durable records that power SaaS revenue intelligence.

## Scope

- Define SaaSMRRSnapshot, revenue movements, cohorts, forecast inputs, and reconciliation evidence.
- Link snapshots to subscriptions, invoices, product usage, support, and health.
- Preserve tenant scoping and audit posture.

## Acceptance Criteria

- Revenue records can explain movement, not only totals.
- Snapshots are evidence-linked and time-bounded.
- Forecast and cohort data state confidence and source.

## Verification

- Snapshot tests cover new, expansion, contraction, churn, reactivation, and reconciliation.
