# Plan 217-04 - `/v1/saas/*` API Contracts

**Phase:** 217 - SaaS Revenue Intelligence, SAS Agents, API/MCP/UI Readiness  
**Status:** Planned  
**Requirements:** SAS-10, API-01, QA-01..15

## Objective

Expose SaaS Suite data through fresh, tenant-safe API contracts.

## Scope

- Reserve fresh F-IDs for SaaS activation, plans, subscriptions, invoices, health, support, product usage, revenue, and agents.
- Define auth, RLS, rate limit, audit, pagination, idempotency, and error envelopes.
- Include read and mutation boundaries.

## Acceptance Criteria

- SaaS APIs are activation-gated and tenant-scoped.
- Mutating endpoints require approval or explicit low-risk policy.
- Contract IDs do not collide with incoming placeholder IDs.

## Verification

- OpenAPI contract tests and route authorization tests are specified.
