# Plan 27-02: Winners and Injection Validation

## Wave 1 — Prove the Anchoring Rules Matter

<task id="1" name="Audit execution-facing prompt assumptions">
  <read_first>
    - .agent/prompts/paid_media_creator.md
    - .agent/prompts/email_lifecycle_strategist.md
    - .agent/prompts/seo_content_architect.md
    - .protocol-lore/CONVENTIONS.md
  </read_first>
  <action>
    Audit the execution-facing prompt set and record which prompts:
    - require winner catalogs
    - require MIR injection paths
    - still point at template paths instead of approved local state
    - need clearer boot-time expectations
  </action>
  <acceptance_criteria>
    - The prompt layer has an explicit inventory of execution prerequisites
    - Misaligned or ambiguous prompt dependencies are visible instead of hidden in prompt prose
  </acceptance_criteria>
</task>

<task id="2" name="Validate winners-catalog boot requirements across flows">
  <read_first>
    - .protocol-lore/WORKFLOWS.md
    - .agent/get-shit-done/workflows/execute-phase.md
    - .mgsd-local/MSP/
  </read_first>
  <action>
    Define how creator and executor flows should behave when winner catalogs are present, missing, stale, or mislocated. Prefer explicit checks or documented prerequisites over silent fallback behavior.
  </action>
  <acceptance_criteria>
    - Winner anchoring becomes a validated boot requirement for relevant execution flows
    - Unsupported or missing anchor scenarios are explicit rather than accidental
  </acceptance_criteria>
</task>

<task id="3" name="Protect anchor rules with tests or integrity checks">
  <read_first>
    - test/protocol.test.js
    - .protocol-lore/CODEBASE-MAP.md
  </read_first>
  <action>
    Add integrity checks or tests that protect the most important winner-catalog and prompt-injection assumptions from regression.
  </action>
  <acceptance_criteria>
    - Regressions in critical anchor paths are detectable by the test suite or integrity checks
    - The anchor contract is enforced by code or tests, not only by documentation
  </acceptance_criteria>
</task>

## Verification

Manual review and integrity checks should be able to show which execution prompts require approved local state, which require winners catalogs, and what happens when those prerequisites are absent.
