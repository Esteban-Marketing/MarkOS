# Plan 214-03 - Subscription Lifecycle State Machine

**Phase:** 214 - SaaS Suite Activation and Subscription Core  
**Status:** Planned  
**Requirements:** SAS-03, TASK-01..05, RUN-01..08, QA-01..15

## Objective

Define subscription lifecycle transitions and safety gates.

## Scope

- Model trial, active, past_due, paused, canceled, reactivated, upgraded, and downgraded states.
- Define allowed transitions, rollback posture, lifecycle events, and operator tasks.
- Identify which transitions require approval by default.

## Acceptance Criteria

- Subscription lifecycle transitions are explicit and testable.
- Mutating lifecycle actions create AgentRun/task/approval/audit records.
- Invalid transitions fail closed.

## Verification

- State-machine tests cover valid, invalid, approval-required, and rollback paths.
