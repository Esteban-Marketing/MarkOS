# Phase 202 — UI Design Contract

> Visual and interaction contract for two frontend surfaces introduced in Phase 202: the tenant-scoped `/settings/mcp` dashboard and the public-but-authenticated `/oauth/consent` screen. Every token **must match** the Phase 201 MarkOS admin-surface vocabulary exactly — MarkOS has one visual language, and 201 established it. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (hand-authored CSS Modules) |
| Preset | not applicable |
| Component library | none (native HTML + `<dialog>` element) |
| Icon library | none in 202 (Phase 201 ships no icon lib; introducing one here would break consistency) |
| Display font | Sora (semibold 600) — `'Sora', 'Space Grotesk', sans-serif` |
| Body font | Space Grotesk — `'Space Grotesk', system-ui, sans-serif` |
| Co-located styling | `page.module.css` next to `page.tsx` (mirrors 201 exactly) |

**Source of truth for tokens:** `app/(markos)/settings/sessions/page.module.css`, `app/(markos)/settings/members/page.module.css`, `app/(markos)/settings/danger/page.module.css`, `app/(markos)/invite/[token]/page.module.css`. These four files are the Phase 201 ground truth. Planner and executor MUST import values from them — do not re-invent. If a value is not listed in this spec, it does not exist in 202.

## Spacing Scale

8-point scale, inherited verbatim from Phase 201. Multiples of 4 only.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon-to-text gap |
| sm | 8px | Seat-bar / cost-bar inner gaps, chip padding |
| md | 12px | Form field internal padding, dialog body margins |
| lg | 16px | Default element spacing, table cell padding, card gap |
| xl | 24px | Page padding, section gap between cards |
| 2xl | 32px | Card internal padding |
| 3xl | 40px | Standalone consent-card padding |
| 4xl | 48px | Consent page vertical padding |

**Exceptions:**
- `min-height: 44px` on every interactive control (button, input, select, `<details>` summary) — WCAG 2.2 AA tap target.
- `min-height: 48px` on primary filled CTAs (matches `invite/[token]` acceptButton) — slightly taller so "Approve access" reads as the primary action.
- Dialog padding: `28px` (not 32px) — matches `danger/page.module.css` `.dialog`.

## Typography

3 sizes on S1, 4 sizes on S2. 2 weights total.

| Role | Size | Weight | Line Height | Family | Usage |
|------|------|--------|-------------|--------|-------|
| Display / h1 | 28px | 600 | 1.2 | Sora | Page heading (both surfaces) |
| Subheading | 18px | 600 | 1.3 | Sora | "Which workspace?", "Pending sessions" sub-sections (S1 cost-breakdown subtitle) |
| Body | 16px | 400 | 1.6 | Space Grotesk | Subheading paragraph, consent scope duration line, empty-state |
| Micro / label | 14px | 400 or 600 | 1.5 | Space Grotesk | Table cells (400), table `<th>` + form labels + caption (600), button labels (600) |
| Caption (eyebrow) | 14px | 600 | 1 | Space Grotesk | Table `<caption>` text-transform: uppercase, letter-spacing 0.08em, color `#0f766e` |
| Mono-inline | 13px | 400 | 1.4 | `ui-monospace, SFMono-Regular, Menlo, Consolas, monospace` | `redirect_uri` display on consent card — **single exception** to body font |

**Weights allowed:** 400 (Space Grotesk body) + 600 (Sora headings, labels, captions, CTA text). No 500, no 700. No italics.

**Accent reserved-for list (explicit):**
1. Approve access CTA (filled, S2)
2. Refresh usage CTA (S1 ghost hover / focus treatment) — button text color on hover only; base button is neutral
3. Focus ring `outline: 2px solid #0d9488; outline-offset: 2px` on every `:focus-visible` interactive element
4. Cost-meter fill bar
5. Deny button focus ring (outline color only)

Accent is **never** used for body text, never for generic borders, never for table rules.

**Contrast verification:**
- Body `#0f172a` on `#ffffff` = 16.1:1 (AAA)
- Secondary `#475569` on `#ffffff` = 8.0:1 (AAA)
- Caption `#0f766e` on `#ffffff` = 5.0:1 (AA large + normal)
- CTA white `#ffffff` on `#0d9488` = 3.1:1 — **large text only** (button 16px+ bold qualifies; standard body would fail AA; never use accent as a body background)
- Destructive `#9a3412` on `#ffffff` = 7.2:1 (AAA)
- Warn text `#78350f` on `#fef3c7` = 8.4:1 (AAA)
- Error text `#991b1b` on `#fef2f2` = 8.1:1 (AAA)

## Component Inventory — Surface 2: `/oauth/consent`

### IA outline (top to bottom)

Centered card (max-width 560px) on a plain page. **No workspace shell.** Mirrors `invite/[token]` layout posture.

1. **Error region** (conditional) — `role="alert"` above the main card when consent request is invalid/expired.
2. **Consent card** — `aria-labelledby="consent-heading"`.
   - h1 "Authorize {client_name}" (Sora 28px).
   - Client details row: 48×48 logo + client_name (14px 600) + `redirect_uri` on second line (13px mono `#64748b`).
   - "Authorize {client_name} is requesting access to:" preamble.
   - Scope chips: horizontal flex-wrap, each chip = `#e6fffb` bg, `#0f766e` text, 12px radius, 6px 12px padding.

   - `<details><summary>What is MCP?</summary>` expandable with 2-sentence plain-English explainer.
   - Action row: Approve access (filled accent) + Deny (neutral outline). Stacked full-width on mobile.

### Tone rules (applied to all 202 copy)

Per Brand Stance (Q-B, `obsidian/brain/Brand Stance.md`):
- **Developer-native, quietly confident.** No marketing superlatives, no exclamation points, no "Awesome!" / "Oops!" / "Yay!" framing.
- Prefer **specification over persuasion.** "30 tools." beats "An incredibly powerful tool suite."
- Error copy names the problem + next action. Never apologizes, never blames the user.
- Use **sentence case** for every heading and button ("Approve access", not "Approve Access"). Matches Phase 201 convention.
- Short sentences. Monosyllabic where natural. Match the MarkOS voice.

## Accessibility Contract (WCAG 2.2 AA per surface)

### Shared requirements

- [ ] Every interactive element has `:focus-visible` outline 2px `#0d9488` offset 2px.
- [ ] Every button has `type="button"` or `type="submit"` explicitly.
- [ ] Every input/select has an associated `<label>` (wrapping `.field` pattern from members page).
- [ ] Every `<table>` has a `<caption>` and every `<th>` has `scope="col"` or `scope="row"`.
- [ ] Color contrast ≥ 4.5:1 for body text, ≥ 3:1 for large text and UI components (verified in Color section).
- [ ] All functionality reachable via keyboard only.
- [ ] Tab order matches visual order.
- [ ] Toast region has `role="status" aria-live="polite"`.
- [ ] Error region has `role="alert"` (immediate announce).

- [ ] Respects `prefers-reduced-motion: reduce` — meter fill, toast slide-in, and hover darken all drop to instant.
- [ ] Touch target ≥ 44×44 CSS px (WCAG 2.2 AA 2.5.8).
- [ ] No `<div>` used where a semantic element exists. Sections wrap in `<section>`. Sidebars not applicable in 202.
