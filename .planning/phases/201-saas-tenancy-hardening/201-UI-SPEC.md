## Interaction Patterns

### Forms
- Input focus ring: `outline: 2px solid #0d9488`, `outline-offset: 2px`.
- Input error state: `border-color: #f87171`, inline error text below at 14px, color `#9a3412`.
- All inputs: 44px min-height (touch target parity with buttons).
- Submit button disabled state while BotID or async op is in flight; spinner replaces label (not added beside it).

### Status Badges
- Pattern: 8px dot + label, inline-flex.
- Pending/polling: amber dot (`#b45309`) + "Verifying…"
- Verified/active: teal dot (`#0f766e`) + label
- Error/failed: red dot (`#f87171`) + label
- Revoked/inactive: gray dot (`#94a3b8`) + label

### Toast Notifications
- Position: bottom-right, 16px from edge.
- Width: `min(360px, calc(100vw - 32px))`.
- Border-radius: 12px.
- Success: white card, teal left border (4px), 14px text.
- Error: white card, red left border (4px).
- Auto-dismiss: 4 seconds. No dismiss button required (user research deferral).

### Polling (SSL status)
- Frontend polls every 30 seconds using `setInterval` + `fetch`.
- Shows spinner badge while polling.
- Stops polling when status reaches `verified` or `failed`.
- No WebSocket required — polling is sufficient for a low-frequency lifecycle event.

### Passkey Prompt
- Shown only on second successful magic-link login — evaluated server-side, passed as a prop.
- Rendered as an inline dismissible card at the top of the dashboard, not a modal.
- Not shown if user already has a registered passkey credential.
- Dismissal persists for 30 days via a cookie (`markos_passkey_prompt_dismissed`).

### Offboarding Read-Only State
- When `tenant.status = 'offboarding'`: all mutating forms and buttons are `disabled` with `aria-disabled="true"`.
- Persistent amber banner renders above the content area (not inside the card).
- Settings nav remains navigable — no pages are hidden; all data is readable.

### Org/Tenant Switcher
- Keyboard accessible: arrow keys navigate options, Enter selects, Escape closes.
- Switch triggers a full page navigation (no SPA transition) to maintain tenant context isolation.
- No optimistic state — wait for server response before updating chrome.

## Implementation Notes for Executor

1. **CSS Modules only.** No Tailwind, no shadcn, no styled-components. All new surfaces get a `page.module.css` alongside `page.tsx` following the billing + layout-shell precedent.

2. **Font loading.** Sora + Space Grotesk must be loaded via `next/font/google` in the root layout. Do not add a `<link>` tag. Confirm they are already loaded before adding duplicates.

3. **Color variables.** Consider extracting the repeated hex values to CSS custom properties in a `globals.css` or a `tokens.module.css` if not already done. Do not inline them per-component.

4. **Vanity login parameterization.** The vanity login page reads `primary_color` from a server prop. Use CSS custom properties set inline on the root element (`style={{ '--accent': primaryColor }}`), not inline style per button.

5. **Tenant switcher dropdown.** Build without a library. The existing project has no dropdown primitive. Use a `<details>/<summary>` or a controlled `useState` + `onBlur` dismiss pattern. Do not reach for a headless UI library.

6. **Modal pattern.** Use `<dialog>` element with `showModal()` / `close()`. No custom modal overlay div. This gives native focus trap and Escape-key close for free, satisfying WCAG 2.2 requirements.

7. **BotID invisible gate.** The BotID script fires pre-submit. No visible CAPTCHA widget renders. The submit button is the only UX surface for the gate (spinner state + blocked error state).

8. **Countdown display.** Use `Math.ceil((offboarding_initiated_at + 30 days - now) / (1000 * 60 * 60 * 24))` days remaining. Display as plain text: "{N} days remaining". No animated progress bar required in phase 201.

9. **CNAME code block.** Wrap CNAME records in `<pre><code>` with a copy button. Copy button uses `navigator.clipboard.writeText()` with a "Copied" confirmation flash (1.5s).

10. **Responsive breakpoint.** Follow existing breakpoint at `max-width: 960px` (from billing CSS) and `max-width: 980px` (from layout-shell CSS). Sidebar collapses to top nav strip on mobile. All settings cards go single-column.
