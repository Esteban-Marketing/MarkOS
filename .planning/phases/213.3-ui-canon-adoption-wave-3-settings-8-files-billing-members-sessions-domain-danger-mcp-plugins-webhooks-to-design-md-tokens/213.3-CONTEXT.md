### Form primitives + state signals (cross-cutting — mirrors 213.2)
- **D-11:** Every form input (search, filter, name field, secret rotation, member-invite email, etc.) MUST compose `.c-input` from `styles/components.css`. Labels use `.c-field__label`. Help text uses `.c-field__help`. Error messages use `.c-field__error` (auto-prepends `[err] ` via `::before`). Wrap each label+input+help/error trio in `.c-field`. NO bespoke form CSS — modules contain only layout/spacing/composition concerns, never re-style inputs.
- **D-12:** Submit / primary actions compose `.c-button.c-button--primary` (single primary CTA per surface per DESIGN.md). Secondary actions use `.c-button.c-button--secondary` or `.c-button.c-button--tertiary`. Destructive actions (e.g., "Delete tenant", "Revoke session", "Disable webhook subscription", "Rotate signing secret") use `.c-button.c-button--destructive`. Confirm-modals (delete confirmations) compose `.c-modal` + `.c-backdrop`.
- **D-13:** Settings surface containers compose `.c-card` (default, 8px radius) for distinct sections. NO `c-card--feature` (32px radius reserved for hero auth panel + landing-page features). Page background is `var(--color-surface)` (Kernel Black). NO gradient backgrounds, NO drop shadows, NO 28px radii.
- **D-14:** Tables (members list, sessions list, webhooks list, MCP tool list, plugins list, billing usage rows, webhook deliveries) use vanilla semantic `<table>` with token-cited row dividers (`border-bottom: 1px solid var(--color-border)`) — NO bespoke table component this wave. Row state badges (active/inactive/failed/pending) compose `.c-badge--{state}` with bracketed-glyph prefix.
- **D-15:** Notice/banner state surfaces (billing alerts, sessions revoke warnings, danger-zone confirmations, MCP cost-meter alerts, plugin warnings, webhook delivery failures, domain rotation grace, member invite-pending banners) compose `.c-notice c-notice--{state}` per **D-09b**. NO inline workarounds. Banner glyph in JetBrains Mono.
- **D-16:** Status indicators on table rows / cards use `.c-status-dot--{live,idle,error}` paired with bracketed-glyph text (`[ok] Active` / `[warn] Pending` / `[err] Failed`). NEVER color-only signals.
