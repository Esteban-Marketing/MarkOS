# Phase 213.3 ‚Äî UI Design Contract

> Visual and interaction contract for redesigning the eight MarkOS settings surfaces ‚Äî billing, members, sessions, domain, danger, mcp, plugins, webhooks (list + detail) ‚Äî from the legacy "Sora + Space Grotesk + teal `#0d9488` + `#ffffff`/`#f8fbfd` light bg + 28px radii + `0 18px 45px rgba()` drop-shadows + `Please` softeners" implementation into the canonical MarkOS visual contract defined in `/DESIGN.md` v1.1.0 (dark Kernel Black `#0A0E14` surface, Protocol Mint `#00D9A3` signal, JetBrains Mono + Inter, strict 8px grid, borders over shadows, flat). Generated by gsd-ui-researcher.

**This is a redesign, not a token-swap port.** All 396 inline-hex occurrences across 17 files are replaced with token citations or canonical alpha-tint formulas. Every banner-shaped surface composes `.c-notice c-notice--{state}` (DESIGN.md v1.1.0 primitive ‚Äî landed via PR #2). Form inputs compose `.c-input` + `.c-field` + `.c-field__{label,help,error}`. Every CTA composes `.c-button{,--primary,--secondary,--tertiary,--destructive}`. Containers compose `.c-card`. Inline hex, gradients, and drop shadows are removed wholesale. The redesign mirrors Phase 213.1 chrome + Phase 213.2 auth posture exactly.

## Spacing Scale

Authoring rule: every `padding`, `margin`, `gap`, `inset`, `top`, `right`, `bottom`, `left` in scope MUST cite a `--space-*` token. **No arbitrary px.** Off-grid values are auto-FAIL.

| Token | Value | DESIGN.md citation | Usage in this phase |
|-------|-------|--------------------|---------------------|
| `--space-none` | 0 | `spacing.none` | Reset margins on `<h1>`, `<p>`, `<table>` |
| `--space-xxs` | 2px | `spacing.xxs` | Badge inner padding, chip/protocol-chip inline gaps, status-dot hit-area offset |
| `--space-xs` | 8px | `spacing.xs` | Table row `gap`, form field gap (label‚Üîinput‚Üîhelp), chip group gap, inline icon-text adjacency, status-dot label gap |
| `--space-sm` | 16px | `spacing.sm` | Settings-card vertical rhythm, notice padding-block, table `th`/`td` padding-block, settings-page left nav link padding, mobile horizontal page padding |
| `--space-md` | 24px | `spacing.md` | Card padding (default via `.c-card`), gap between major sections, modal padding, page horizontal padding at sm‚Äìmd viewport |
| `--space-lg` | 32px | `spacing.lg` | Modal padding (via `.c-modal`), page horizontal padding ‚â• md, sidebar padding (via `.c-sidebar`), inter-section gap within large surfaces (mcp, webhooks) |

| `--space-xxl` | 96px | `spacing.xxl` | Reserved ‚Äî not used in settings |

**Allowed exceptions (DESIGN.md documented):**

4. `8px` for cost-meter / seat-bar track height ‚Äî Claude's Discretion (visual affordance for progress bars; not a spacing value, a sizing value; cited as `height: 8px` with rationale comment).

## Copywriting Contract

Settings-surface copy must match the developer-native, AI-first, quietly confident voice from `obsidian/brain/Brand Stance.md`. Rules identical to Phase 213.2:

1. No banned lexicon: `synergy, leverage, empower, unlock, transform, revolutionize, supercharge, holistic, seamless, cutting-edge, innovative, game-changer, next-generation, world-class, best-in-class, reimagine, disrupt, just (as softener)`.
2. No exclamation points.
3. No emoji (bracketed glyphs only).
4. No "Please" softeners.
5. Errors carry remediation per DESIGN.md "Errors carry remediation."
6. Notices lead with bracketed glyph in JetBrains Mono: `[warn] Subscription expires in 14 days.`
