#### 4. Light theme opt-in — verify token cascade

**Test:** Add `data-theme="light"` to the document root in browser DevTools and load /settings/billing and /settings/webhooks/[sub_id]. Verify:
- Surfaces shift to `#F4F4F0` (Lino) background
- Mint text uses `--color-primary-text` = `#00805F` (not raw `#00D9A3`)

**Expected:** No contrast failures. Bracketed glyphs `[ok]/[warn]/[err]/[info]` remain visible. Filter chip active state (webhooks detail) uses `#00805F` not `#00D9A3` via the `--color-primary-text` → light-theme override.

**Why human:** Light-theme cascade requires browser to resolve CSS custom property overrides in `[data-theme="light"]` block. The filter chip at line 169 of webhooks/[sub_id]/page.module.css uses raw `--color-primary` (not `--color-primary-text`) — this should be confirmed to pass contrast on light theme. On dark theme it is AAA (10.5:1). On light theme, `#00D9A3` on `rgb(0 217 163 / 0.10)` over `#F4F4F0` = approximately 1.6:1 (FAIL). However the `--color-primary-subtle` background provides context contrast rather than the `--color-primary-text` text color itself. This edge case requires human verification to confirm light-theme WCAG compliance on the filter chip.
