## Sub-Component Extraction Decision

**Decision: Extracted `SessionsPageView` to `app/(markos)/_components/SessionsPageView.tsx`.**

Rationale: `page.tsx` is a `'use client'` component with internal `useState` and `useEffect` — Storybook cannot control internal state (`revokeTarget`, `showRevokeAll`, `sessions`) without extraction. Pattern mirrors `MembersPageView` (Phase 213.3-02) and `LoginCard` (Phase 213.2-01). `page.tsx` retains all fetch/handler logic; `SessionsPageView` receives all state as props.
