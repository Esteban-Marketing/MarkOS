# Plan 215-03 - Colombia Mercado Pago and DIAN Provider Path

**Phase:** 215 - SaaS Billing, Payments, and Multi-Country Compliance  
**Status:** Planned  
**Requirements:** SAS-05, SAS-06, COMP-01, QA-01..15

## Objective

Define the Colombia billing path without hard-coding one provider into the core model.

## Scope

- Map Mercado Pago subscription/payment capability during implementation research.
- Define Siigo/Alegra DIAN provider interface, certificate handling posture, CUFE/QR fields, sandbox/production state, and rejection handling.
- Model IVA, retentions, legal invoice numbering, and correction behavior.

## Acceptance Criteria

- Colombia path has processor, legal billing, accounting, tax, and evidence model.
- DIAN rejections create P1 tasks.
- Provider-specific complexity stays behind provider config objects.

## Verification

- Country-specific fixture tests cover successful invoice, rejected invoice, retry, correction, and task creation.
