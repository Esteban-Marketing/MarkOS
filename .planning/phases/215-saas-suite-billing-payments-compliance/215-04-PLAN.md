# Plan 215-04 - Processor Webhook Routing Through Existing Webhook Engine

**Phase:** 215 - SaaS Billing, Payments, and Multi-Country Compliance  
**Status:** Planned  
**Requirements:** SAS-05, WHK-01, QA-01..15

## Objective

Reuse existing webhook guarantees for billing and accounting events.

## Scope

- Route processor/accounting events through durable webhook patterns: signing, replay, DLQ, rate limit, circuit breaker, and observability.
- Define event normalization and idempotency.
- Link webhook events to billing, subscription, accounting, and evidence records.

## Acceptance Criteria

- No processor webhook bypasses MarkOS webhook durability.
- Duplicate processor events are idempotent.
- Failed events are replayable and support-visible.

## Verification

- Webhook tests cover signature, duplicate, DLQ, replay, and event normalization.
