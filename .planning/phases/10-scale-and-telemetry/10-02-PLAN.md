# Phase 10 — Plan 02: PostHog Telemetry (Frontend + Node.js)

## Objective
Implement a robust, centralized analytics and telemetry pipeline using PostHog to track multi-tenant behavior, onboarding completion, agent execution metrics, and AI/Human hybrid workflows across the entire MGSD protocol.

## Pre-requisites
- Phase 12 metadata separation is complete.
- Plan 10-01 Vercel Multi-Tenant routing is complete.
- PostHog account and project API key created and injected via environment variable `POSTHOG_API_KEY`, `POSTHOG_HOST`.

## Implementation Steps

### 1. Frontend Tracking (Onboarding UI)
- Integrate the PostHog JavaScript snippet in `index.html` (pulled from standard PostHog JS library).
- In `onboarding.js` add event captures:
  - `posthog.init('{{POSTHOG_API_KEY}}', { api_host: '{{POSTHOG_HOST}}' })` safely bounded so missing keys don't break the app.
  - `posthog.capture('onboarding_started', { url: window.location.href })` on page load.
  - `posthog.capture('business_model_selected', { business_model: val })` within `onBusinessModelChange`.
  - `posthog.capture('onboarding_step_completed', { step: currentStepIndex })` inside the `nextStep()` function.
  - `posthog.capture('onboarding_completed', { project_slug, business_model })` upon successful API submission.

### 2. Backend Telemetry Logging (MGSD Agents and CLI)
- Create a new module: `onboarding/backend/agents/telemetry.cjs` wrapping the `posthog-node` SDK.
- The telemetry client must capture:
  - Agent starts/completes (`agent_execution_started`, `agent_execution_completed`).
  - LLM Token usage, run duration.
  - Ensure the `distinct_id` is linked to either the `project_slug` or a stable system hash (for local CLI runs).

### 3. Agent Execution Wrapping
- Modify the base `llm-adapter.cjs` to emit a completion event with `promptTokens`, `completionTokens`, and `model` whenever a call finishes.
- The `orchestrator.cjs` then formats and sends these to the `telemetry.cjs` module along with the active project slug.

### 4. Opt-Out Mechanism
- Introduce an environment variable `MGSD_TELEMETRY=false` to completely disable the PostHog initialization, ensuring respectful offline/privacy-conscious usage for agencies who opt out.

## PostHog Taxonomy
1. `onboarding_started`
2. `business_model_selected`
3. `onboarding_step_completed`
4. `onboarding_completed`
5. `agent_execution_started`
6. `agent_execution_completed`

## Verification
- Running the onboarding flow fires network requests to `app.posthog.com/e/...`.
- Vercel API endpoints emit `agent_execution_completed` events into PostHog asynchronously.
- Turning off `MGSD_TELEMETRY=false` blocks all tracking without crashing the app.
