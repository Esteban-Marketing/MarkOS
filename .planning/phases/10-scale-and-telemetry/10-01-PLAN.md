# Phase 10 â€” Plan 01: Vercel Multi-Tenant Architecture Readiness

## Objective
Convert the current Express `server.cjs` onboarding app to a serverless-ready architecture for Vercel deployment, supporting multiple isolated brand contexts (`project_slug`) from a unified hosted API.

## Pre-requisites
- Phase 12 metadata separation (Supabase + Upstash Vector isolation) is complete.

## Implementation Steps

### 1. Extract API Routes for Serverless
Currently, `onboarding/backend/server.cjs` is a monolithic Express app.
- Refactor the endpoints (`GET /api/health`, `POST /api/onboard`, `POST /api/check-db`) into modular handler functions.
- Create an `api/` directory (Vercel standard) at the root to wrap these modules for serverless execution.

### 2. Multi-Tenant Project Slug Routing
Currently, the server assumes a single project directory (`process.cwd()`).
- Modify the frontend to accept or parse a `?client=slug` query parameter.
- Send `project_slug` within every API request payload.
- Update `orchestrator.cjs` to write drafts and seed data into a dynamically resolved path: `.markos-data/${project_slug}/` instead of the local `.markos-local/` when running in hosted mode.

### 3. Vercel Configuration
- Create `vercel.json` at the project root:
  - Define `public/` or `onboarding/` as the static output directory.
  - Define rewrites to map `/api/*` to the Vercel serverless functions.
  - Configure environment variables expected by Vercel (`OPENAI_API_KEY`, `UPSTASH_VECTOR_REST_URL`, `UPSTASH_VECTOR_REST_TOKEN`).

### 4. CORS and Hosted Supabase + Upstash Vector Connect
- Add `cors` middleware to the Express/serverless instances restricting access to the Vercel deployed domain and localhost.
- Ensure `vector-store-client.cjs` defaults to `UPSTASH_VECTOR_REST_URL` in a serverless environment (since local `:8000` won't be accessible).

## Verification
- Vercel local dev (`vercel dev`) runs successfully.
- Submitting the form with `?client=acme-corp` isolates data to the Acme Corp context.
- API endpoints correctly handle multi-tenant paths without filesystem collisions.
