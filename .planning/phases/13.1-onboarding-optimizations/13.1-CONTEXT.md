# 🛠️ Phase 13.1 Context: Onboarding Hardening & Optimization

This phase addresses critical technical debt and architectural improvements identified after the Phase 13 Smart Onboarding Engine rollout.

## 🎯 Objectives

### 1. Upgrade Isolation (P0)
Prevent protocol updates from wiping or corrupting client onboarding data.
- **Action:** Enforce `.mgsd-local/` as the default output path for all MIR/MSP files.
- **Action:** Ensure configuration (`onboarding-config.json`) is isolated from core protocol templates.

### 2. Backend Modularity (P1)
Reduce code redundancy and centralize AI behavior across the onboarding ecosystem.
- **Action:** Consolidated body parsing and JSON serialization into `onboarding/backend/utils.cjs`.
- **Action:** Move all hardcoded LLM prompts from `handlers.cjs` into the dedicated `prompts/` module.
- **Action:** Refactor `server.cjs` and `handlers.cjs` to consume shared utilities.

### 3. Hierarchy of Truth (P1)
Explicitly enforce data trust levels during extraction to prevent hallucinations.
- **Action:** Priority: **Chat > File > Web**.
- **Action:** Label source buckets (`[WEB]`, `[FILE]`, `[CHAT]`) in the extraction prompt.

### 4. Performance & Resilience (P2)
Improve the speed and stability of the extraction pipeline.
- **Action:** Parallelize source processing (URL scraping + file parsing) using `Promise.all`.
- **Action:** Ensure 30s timeouts are consistently applied to all external API calls (Tavily).

## 📊 Success Criteria
- [ ] No client data written to `.agent/` during onboarding.
- [ ] `utils.cjs` is the single source of truth for backend boilerplate.
- [ ] Extraction LLM explicitly references the source hierarchy in its trace.
- [ ] Processing concurrent URL + 2 files takes < 15 seconds.
