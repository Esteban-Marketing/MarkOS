# 📋 Plan 13.1-01: Onboarding Hardening & Optimization

This plan formalizes the technical debt fixes and optimizations for the Smart Onboarding Engine.

## 🎯 Requirements Mapped
- Upgrade Isolation (P0)
- Backend Modularity (P1)
- Hierarchy of Truth (P1)
- Performance Parallelization (P2)

## 🏗️ Technical Approach

### 1. Configuration & Data Isolation
- Move `project_slug` and `mir_output_path` in `onboarding-config.json` away from `.agent/` paths.
- Default to `.markos-local/MIR` and `.markos-local/MSP` for all onboarding output.

### 2. Boilerplate Consolidation
- Use `onboarding/backend/utils.cjs` for all `readBody` and `json` operations.
- Update `server.cjs` and `handlers.cjs` to import these shared helpers.

### 3. Prompt Management
- Move the `getGroupingPrompt` (Step 2) and `getExtractionPrompt` (Step 0) logic into the `backend/prompts/` module.
- Centralize all LLM system prompts (Grouping Agent, Extractor Agent).

### 4. Logic Hardening
- Implement Source Buckets in `handleExtractAndScore`.
- Parallelize `tavilyScraper` and file parsers in `handleExtractSources`.
