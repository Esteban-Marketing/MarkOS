# Plan 211-03 - Approval-to-Dispatch Path

**Phase:** 211 - Content, Social, and Revenue Loop  
**Status:** Planned  
**Requirements:** LOOP-04, TASK-04, RUN-06, CONN-04, QA-01..15

## Objective

Prove one safe path from approved artifact to queued or executed dispatch.

## Scope

- Use the Approval Inbox decision contract before dispatch.
- Define DispatchAttempt with channel, connector, approved payload, provider receipt, and rollback hint.
- Block dispatch if required connector is missing, degraded, or stale.
- Reuse `schedule_post` approval posture where appropriate.

## Acceptance Criteria

- External dispatch never happens without approval or explicit low-risk autonomy.
- Connector state is checked before dispatch.
- Provider receipts or queue state link back to artifact and run.

## Verification

- Dispatch tests cover approved, rejected, connector-blocked, and provider-failure paths.
