# Plan 210-01 - ConnectorInstall Contract and Schema

**Phase:** 210 - Connector Wow Loop and Recovery  
**Status:** Planned  
**Requirements:** CONN-01, CONN-04, QA-01..15

## Objective

Create the tenant-scoped connector substrate for live data and recovery.

## Scope

- Reserve fresh contract and migration IDs.
- Define ConnectorInstall with auth type, scopes, encrypted credential reference, status, sync cadence, retention, recovery state, and dependent agents.
- Define connector status states and audit events.
- Add RLS and deletion/export posture.

## Acceptance Criteria

- Connector records never expose raw credentials.
- Connector status is explicit and support-visible.
- Dependent runs and tasks can reference connector state.

## Verification

- RLS, credential-redaction, and status-transition tests are specified.
