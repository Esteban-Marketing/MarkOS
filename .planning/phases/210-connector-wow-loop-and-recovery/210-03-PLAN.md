# Plan 210-03 - Max-3 Onboarding Connector Recommendation UX

**Phase:** 210 - Connector Wow Loop and Recovery  
**Status:** Planned  
**Requirements:** CONN-02, CONN-03, TASK-02, QA-01..15

## Objective

Recommend a small number of high-value connectors during onboarding.

## Scope

- Define recommendation logic by tenant business type, goals, channels, and available data.
- Limit first recommendations to three connectors.
- Write value-unlock copy and expected wow outcome for each connector.
- Link recommendations to recovery tasks when setup fails.

## Acceptance Criteria

- Onboarding never overwhelms the user with connector sprawl.
- Every recommended connector explains the first useful outcome.
- Failed setup produces visible recovery work.

## Verification

- Fixture tests cover SaaS, eCommerce, services, agency, and low-data tenants.
