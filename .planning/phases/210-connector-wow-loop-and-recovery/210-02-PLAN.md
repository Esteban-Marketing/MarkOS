# Plan 210-02 - Nango and Direct Adapter Decision Matrix

**Phase:** 210 - Connector Wow Loop and Recovery  
**Status:** Planned  
**Requirements:** CONN-06, QA-11, COMP-01

## Objective

Choose connector implementation posture with official API and scope evidence.

## Scope

- Research Nango versus direct adapters for GA4, GSC, CRM, social, product analytics, and support candidates.
- Record OAuth scopes, rate limits, webhooks, retention limits, and data classes.
- Define direct-adapter exceptions for deep APIs that Nango cannot cover.

## Acceptance Criteria

- Connector choices are justified with source evidence.
- Security and privacy posture is recorded for each first-wave connector.
- Future adapter decisions can reuse the matrix.

## Verification

- Research checklist references official docs for every selected provider during execution.
