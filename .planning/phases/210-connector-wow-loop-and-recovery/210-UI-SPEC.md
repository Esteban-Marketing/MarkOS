# Phase 210 — UI Design Contract (hybrid)

> **Phase 210 ships ONE UI page plus five no-UI backend doctrine plans.** This is a
> hybrid scope. The single UI surface is the onboarding connector recommendations
> page at `app/(markos)/onboarding/connectors/page.tsx` (NEW route — onboarding
> directory does not yet exist under `app/(markos)/`). The other five plans
> (210-01, 210-02, 210-04, 210-05, 210-06) ship `.planning/connectors/*.md`
> doctrine, `lib/markos/connectors/*.ts` typed modules + `.cjs` twins,
> `onboarding/backend/connectors/*.cjs` adapters, Supabase migration

> `scripts/connectors/`, an API recovery endpoint at `api/v1/connectors/recovery.js`,
> and tests under `test/connectors/phase-210/**/*.test.js`. None of those five
> plans modify any `app/`, `components/`, `*.stories.tsx`, `page.tsx`,
> `*.module.css`, or `*.css` path other than the single 210-03 page.
>
> The single UI page (Plan 210-03) closes two upstream translation gates:
> 1. The Phase 208 Recovery Center renders the `connector_placeholder` family's
>    `<PlaceholderBanner variant="future_phase_210">` for connector failure
>    routing — once 210-05/06 ship the recovery substrate, install failures
>    from the 210-03 page route into the Recovery Center as real items, NOT
>    placeholder rows.

>    `<PlaceholderBanner variant="future_phase_210">` when

>    210 ships, connector-derived evidence (`EvidenceMap.source_class` ∈
>    connector-derived literals) flows into the same sub-component without
>    schema changes — the placeholder dissolves.

## Spacing Scale

Authoring rule: every `padding`, `margin`, `gap`, `inset` in scope MUST cite a
`--space-*` token. No arbitrary px. Off-grid values are auto-FAIL.

| Token | Value | DESIGN.md citation | Usage in this surface |
|-------|-------|--------------------|---------------------|
| `--space-none` | 0 | `spacing.none` | Reset margins on `<h1>`, `<h2>`, `<h3>`, `<p>`, `<ul>` |
| `--space-xxs` | 2px | `spacing.xxs` | Badge inner padding, `[ok]`/`[warn]`/`[err]`/`[info]` glyph adjacency, scope-row inner spacing |
| `--space-xs` | 8px | `spacing.xs` | Wow-promise eyebrow → wow-promise text gap, scope-list row gap, recommendation card meta-row gap, install-state status-dot offset |
| `--space-sm` | 16px | `spacing.sm` | Card vertical rhythm, notice padding-block, mobile horizontal page padding (max-width ≤ 640px), eyebrow → heading gap |
| `--space-md` | 24px | `spacing.md` | Card padding (via `.c-card`), gap between max-3 recommendation cards, page horizontal padding at sm–md viewport, install-modal padding |
| `--space-lg` | 32px | `spacing.lg` | Page horizontal padding ≥ md, page heading → recommendation list gap |
| `--space-xl` | 48px | `spacing.xl` | Page vertical padding ≥ lg, page top padding before page heading |
| `--space-xxl` | 96px | `spacing.xxl` | Reserved — not used in this phase |

**Allowed exceptions (DESIGN.md documented):**
1. `1px` for hairline borders (`var(--color-border)`) — preferred over shadows on dark surfaces (DESIGN.md "Borders over shadows"). Used between recommendation card sections and on scope-list dividers.
2. `2px` for focus ring width and offset (`var(--focus-ring-width)`, `var(--focus-ring-offset)`); never suppressed.
3. `4px` for `.c-notice` `border-inline-start` accent — composed via `.c-notice` primitive; the page module CSS never declares this directly.
4. `max-width: 720px` for onboarding container (`--w-prose` text-readable width) — onboarding is read-then-act, not data-dense; centered single column on desktop.
5. `max-width: 560px` for confirmation modals (`--w-modal`) when install requires confirm — typically delegated to provider OAuth pages, not authored here.
6. **Mobile touch target `44px`** via `--h-control-touch` on `.c-button` for `(pointer: coarse)` viewports — already declared globally in `styles/components.css` per 213.2 cross-cutting fix; this phase consumes it without re-declaration.

**Accessibility focus order:**
1. `<h1>` "Connect your data" (programmatic focus on route entry).
2. Page descriptor `.t-lead`.
3. For each recommendation card (top → bottom):
   a. `<h2>` connector display name.
   b. "Recommended" `.c-chip--mint` badge (informational; not focusable).
   c. "What you'll get" eyebrow + body (read-only).
   d. "Setup time" eyebrow + body (read-only).
   e. "Permissions" eyebrow + scope `<ul>` (each scope row read-only; raw scope `.c-code-inline` is selectable for copy but not focusable).
   f. "If something fails" eyebrow + body (read-only).
   g. "Connect {provider_display_name}" `.c-button--primary` (primary action focus).
   h. (Post-install) "Open recovery task" `.c-chip-protocol` deep-link (focusable when present).
4. (Page-level) "Skip — start with read-only audit instead" `.c-button--tertiary`.
5. (When error notice present) Retry CTA inside `.c-notice--error`.

ARIA contract:
- `<main>` lands on the page heading; skip-to-content link from layout-shell honors this.
- Each recommendation card is a `<section aria-labelledby="rec-{provider}-h2">` with the `<h2>` carrying that id.
- Install-state notices use `aria-live="polite"` for success/info and `aria-live="assertive"` for error/warning.
- Install-in-progress `.c-status-dot--live` carries `aria-label="Connecting to {provider}"` (state color is never the only signal — bracketed glyph `[info]` always paired).

**Motion:** `.c-status-dot--live` kernel-pulse on install-in-progress (freezes at
full opacity under `prefers-reduced-motion` per DESIGN.md). Card-state
transitions: instant — no slide, no fade, no skeleton. Install-success
notice: persists indefinitely until user navigates away (no auto-dismiss).
Install-error notice: persists indefinitely (operator review required).
`prefers-reduced-motion` block in `page.module.css` collapses any animation
durations to 0ms.

**Mobile breakpoint behavior:** At `max-width: 640px`:
- Single column. Page padding `--space-md`.
- Each recommendation card stacks with `--space-md` gap.
- Inside each card, sections stack vertically.
- "Connect {provider_display_name}" CTA is full-width with `--h-control-touch` (44px) inherited globally.
- Scope `<ul>` items remain vertical; raw `.c-code-inline` wraps with `word-break: break-word` (token-only via `.c-code-inline` primitive).
- Skip CTA full-width at the bottom.
- All `.c-notice` instances span full card width; no horizontal scroll.
