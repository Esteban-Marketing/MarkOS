# Phase 200.1 Plan 08 Summary

Plan `200.1-08` closes review concern `M4` by shipping a deterministic eval-as-test layer for the original 10 MCP tools. The new suite lives under `lib/markos/evals/mcp/`, uses a shared scorer/runner substrate, commits 5 JSON fixtures per tool under `test/fixtures/evals/mcp/`, and evaluates each tool against the same three gates: brand voice, grounded claim reference, and neuro/archetype adherence.

## What changed

- Added `lib/markos/evals/mcp/_lib/scorers.cjs` and `lib/markos/evals/mcp/_lib/scorers.ts`.
- Added `lib/markos/evals/mcp/_lib/runner.cjs`.
- Added 10 per-tool eval entry files under `lib/markos/evals/mcp/`:
  - `draft_message.eval.js`
  - `plan_campaign.eval.js`
  - `research_audience.eval.js`
  - `run_neuro_audit.eval.js`
  - `generate_brief.eval.js`
  - `audit_claim.eval.js`
  - `list_pain_points.eval.js`
  - `rank_execution_queue.eval.js`
  - `schedule_post.eval.js`
  - `explain_literacy.eval.js`
- Added `lib/markos/evals/mcp/all.test.js` as the explicit Node test entrypoint that loads the 10 eval files.
- Added 50 committed fixtures under `test/fixtures/evals/mcp/` (`5` per tool, same scenario matrix across all 10 tools).
- Added `.github/workflows/mcp-evals.yml`.

## Notes

- The brand-voice rubric is now pinned in code from `obsidian/brain/Brand Stance.md` rather than inferred ad hoc. The scorer exports frozen `BRAND_PREFERRED_DEV_VOCAB`, `BRAND_PREFERRED_AI_PHRASES`, `BRAND_PREFERRED_REGISTERS`, `BRAND_PREFERRED`, and `BRAND_ANTI` collections so every tool eval uses the same canon.
- The `brandVoiceScore` implementation is intentionally weighted by the three stance layers instead of using a naive flat-token ratio across the whole combined rubric. That keeps the threshold meaningful while still allowing compact copy to pass when it clearly hits the developer-native, AI-first, and quietly-confident registers.
- `draft_message` and `run_neuro_audit` are not separate tool modules in this repo; they remain inline in `lib/markos/mcp/tools/index.cjs`. Their eval suites call the underlying `runDraft` / `auditDraft` primitives from `bin/lib/generate-runner.cjs`, which is the narrowest deterministic seam the current codebase exposes.
- The original plan text specified `node --test lib/markos/evals/mcp/`, but Node 22 treats that bare directory target as an unsupported directory import in this environment instead of discovering tests. The shipped workflow therefore uses the explicit entry file `lib/markos/evals/mcp/all.test.js`, and sets `NODE_OPTIONS=--test-isolation=none` so the gate does not depend on subprocess spawning behavior.
