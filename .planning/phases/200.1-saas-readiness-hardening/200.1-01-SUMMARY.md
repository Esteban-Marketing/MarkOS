# Phase 200.1 Plan 01 Summary

Plan `200.1-01` landed the webhook SSRF hardening layer across subscribe-time and dispatch-time flows. The new `url-validator` module rejects private and metadata targets, resolves hostnames up front, preserves host/SNI on pinned-IP delivery, and caps redirect chains at `3`.

## What changed

- Added `lib/markos/webhooks/url-validator.cjs` plus `lib/markos/webhooks/url-validator.ts`.
- Wired `engine.subscribe()` through `validateWebhookUrl()` before insert.
- Wired `api/webhooks/subscribe.js` to surface structured `invalid_subscriber_url` failures.
- Wrapped `delivery.cjs` with `dispatchWithValidation()` so each attempt re-validates the destination and follows at most `DEFAULT_MAX_REDIRECTS`.
- Added `test/webhooks/url-validator.test.js` and updated `test/webhooks/engine.test.js` for the current event list and validation behavior.

## Verification

- `node --test --test-isolation=none test/webhooks/url-validator.test.js`
- `node --test --test-isolation=none test/webhooks/engine.test.js test/webhooks/delivery.test.js`

## Outcome

Review concern `H1` is closed for the Phase 200 webhook substrate: subscribe rejects unsafe URLs, dispatch re-checks destinations before egress, and redirect abuse now terminates deterministically instead of trusting the original hostname forever.
