- Token-list approach: `brandStance` defines preferred tokens (verbs, adjectives, framings) and anti-tokens (banned phrases). Score = `100 * (matched_preferred / total_preferred) - penalty * matched_anti`. Stub-quality scorer is fine for this phase; document refinement path.

Claim check scoring:
- Tool output's `claims[]` array (where applicable) must contain at least one `evidence_id` referencing one of `expectedEvidenceIds` from the fixture.

- Tool output text contains at least one of the 9 neuro pillars (e.g. "psychological safety", "loss aversion", "social proof", "reciprocity", "anchoring", "status", "scarcity", "agency", "identity") OR the explicitly expected pillar from fixture (case-insensitive substring).

       exports.brandVoiceScore = function brandVoiceScore(textOutput, opts = {}) {
         const text = String(textOutput || '').toLowerCase();
         const preferred = opts.preferred || exports.BRAND_PREFERRED;
         const anti = opts.anti || exports.BRAND_ANTI;
         const matchedPref = preferred.filter(t => text.includes(t.toLowerCase())).length;
         const matchedAnti = anti.filter(t => text.includes(t.toLowerCase())).length;
         const baseline = preferred.length === 0 ? 0 : (100 * matchedPref / preferred.length);
         const penalty = matchedAnti * 15; // 15 points per anti-token hit
         return Math.max(0, Math.min(100, Math.round(baseline - penalty)));
       };
