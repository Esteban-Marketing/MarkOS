# Phase 213.2 — UI Design Contract

> Visual and interaction contract for redesigning the four MarkOS auth surfaces — `/login`, `/signup`, `/invite/[token]`, and `/oauth/consent` — from the legacy "Sora + Space Grotesk + teal `#0d9488` + `#f8fbfd` light bg + 28px radii + gradients + drop shadows + `Please` softeners" implementation into the canonical MarkOS visual contract defined in `/DESIGN.md` (dark Kernel Black `#0A0E14` surface, Protocol Mint `#00D9A3` signal, JetBrains Mono + Inter, strict 8px grid, borders preferred over shadows, flat). Generated by gsd-ui-researcher; verified by gsd-ui-checker.

**This is a redesign, not a token-swap port.** Every value in the in-scope `.module.css` files is replaced with a token citation from `/DESIGN.md` via `app/tokens.css`, and every input/label/button/card class is rewritten to compose `.c-*` primitives from `styles/components.css`. Inline hex (82 occurrences across 8 files), inline px (excepting documented 1px borders / 2px focus offsets / 4px banner accents), and gradients are removed wholesale. Form authoring is now primitive composition — modules carry only layout/spacing/state-conditional concerns, never re-style inputs/labels/buttons. The redesign mirrors the Phase 213.1 chrome posture exactly.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla CSS Modules consuming `var(--*)` tokens from `app/tokens.css` + composing `.c-*` primitives from `styles/components.css`) |
| Preset | not applicable |
| Component library | none — primitives defined in `styles/components.css` (already shipped, Phase 213.1) |
| Icon library | Lucide (default) · Phosphor Regular (only allowed substitute) — DESIGN.md "Iconography". This phase uses **no icons in chrome**; bracketed glyphs (`[ok]`, `[warn]`, `[err]`, `[info]`) carry every state signal per DESIGN.md "Color blindness" rule + Phase 213.1 Amendment 4 |
| Heading font | JetBrains Mono (`var(--font-mono)`) — DESIGN.md `typography.h1`–`h4`, `display`, `code-*` |
| Body font | Inter (`var(--font-sans)`) — DESIGN.md `typography.body-md`, `lead`, `body-sm`, `label-caps` |
| Default theme | dark (`color-scheme: dark`); light is opt-in via `[data-theme="light"]` |
| Form authoring posture | **Primitive-only.** Every `<input>`, `<button>`, `<label>` MUST compose `.c-input`, `.c-button{,--primary,--secondary,--tertiary,--destructive}`, `.c-field` + `.c-field__{label,help,error}`. No bespoke form CSS in any module. Local module classes carry only layout (`display`, `gap`, `grid`, container width) and state-conditional accents (e.g. error border on a card) |

## Spacing Scale

Authoring rule: every `padding`, `margin`, `gap`, `inset`, `top`, `right`, `bottom`, `left` in scope MUST cite a `--space-*` token. **No arbitrary px.** Off-grid values are auto-FAIL.

| Token | Value | DESIGN.md citation | Usage in this phase |
|-------|-------|--------------------|---------------------|
| `--space-none` | 0 | `spacing.none` | Reset margins on `<h1>`, `<p>`, `<fieldset>`, `<legend>` |
| `--space-xxs` | 2px | `spacing.xxs` | Tight inline gaps within chip/protocol-chip composition; consent radio ring offset |
| `--space-xs` | 8px | `spacing.xs` | Form field gap (label↔input↔help), scope-chip group `gap`, inline icon-text adjacency, button gap inside `.actionRow`, `.brandLockup` `gap` |
| `--space-sm` | 16px | `spacing.sm` | Auth-card vertical rhythm (between heading↔subheading↔form), invite-error notice padding-block, mobile horizontal padding |
| `--space-md` | 24px | `spacing.md` | Auth-card padding (default), gap between major sections in oauth/consent (client row ↔ scopes ↔ tenant picker ↔ duration), invite-error notice padding-inline |
| `--space-lg` | 32px | `spacing.lg` | Page horizontal padding ≥ md, `.c-card--feature` padding (auth hero panels), inter-form spacing in oauth/consent |
| `--space-xl` | 48px | `spacing.xl` | Page vertical padding ≥ lg (replaces legacy `64px 24px` and `48px 24px` page paddings), banner detail breathing room |
| `--space-xxl` | 96px | `spacing.xxl` | Reserved for landing — not used in auth |

**Exceptions (allowed by DESIGN.md):**
1. `1px` for hairline borders (`var(--color-border)`) — explicitly preferred over shadows on dark surfaces.

4. `75ch` body line-length literal — DESIGN.md "Line length: max 75 characters per line."

## Copywriting Contract

Auth-surface copy carries voice load. Phase 213.1 D-07 + Brand Stance.md voice rules apply: developer-native, AI-first, quietly confident, no exclamation marks, no emoji, no banned lexicon, no "Please" softeners. **All text-strings below are reviewed against:**

1. Banned lexicon: `synergy, leverage, empower, unlock, transform, revolutionize, supercharge, holistic, seamless, cutting-edge, innovative, game-changer, next-generation, world-class, best-in-class, reimagine, disrupt, just (as softener)`.
2. No exclamation points.
3. No emoji (bracketed glyphs only).
4. No "Please" softener.
5. Errors carry remediation per DESIGN.md "Errors carry remediation."
