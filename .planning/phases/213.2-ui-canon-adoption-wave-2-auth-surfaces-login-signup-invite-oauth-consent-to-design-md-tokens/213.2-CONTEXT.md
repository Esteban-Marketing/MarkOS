- **D-06:** `node --test` for shape regression (no Vitest migration in this wave). Existing tests under `test/auth/` and `test/tenant-auth/` MUST stay green (verify post-task).

- **D-08:** Token-only authoring — zero inline hex (`#[0-9a-fA-F]{3,8}`) in any modified `.module.css`. Documented exceptions: 1px borders, 2px focus offsets, canonical 12% alpha-tint formulas (`rgb(255 184 0 / 0.12)` warn, `rgb(248 81 73 / 0.12)` error) per DESIGN.md "Badges".
- **D-09:** Mint-as-text uses `var(--color-primary-text)` (theme-aware, AA on light, `LinkText` on forced-colors) per Phase 213.1 commit `fce709d`. NEVER raw `var(--color-primary)` for text contexts.

### Wave structure
- **D-10:** Single wave (Wave 1 of phase 213.2). 4 plans, 1 per surface — surfaces are mutually independent (no shared `.tsx` imports across login/signup/invite/oauth-consent), so this is genuinely parallelizable. However, executor sequencing kept SEQUENTIAL per wave to maintain atomic-commit cadence and avoid hook contention (matches Phase 213.1 D-06).
  - **Plan 213.2-01:** login — module + tsx + Storybook story + a11y test entry
  - **Plan 213.2-02:** signup — module + tsx + layout.tsx + Storybook story + a11y test entry

### Form primitives (cross-cutting)
- **D-11:** Every form input MUST compose `.c-input` from `styles/components.css`. Labels use `.c-field__label`. Help text uses `.c-field__help`. Error messages use `.c-field__error` (auto-prepends `[err] ` via `::before`). Wrap each label+input+help/error trio in `.c-field`. NO bespoke form CSS — modules contain only layout/spacing/composition concerns, never re-style inputs.
- **D-12:** Submit buttons compose `.c-button.c-button--primary` (single primary CTA per surface per DESIGN.md). Secondary actions use `.c-button.c-button--secondary` or `.c-button.c-button--tertiary`. Destructive actions (e.g., "Decline" on oauth/consent) use `.c-button.c-button--destructive`.
- **D-13:** Auth-surface page layouts compose `.c-card` (default) or `.c-card--feature` (hero auth panel) for the form container. Page background is `var(--color-surface)` (Kernel Black). NO gradient backgrounds, NO drop shadows, NO 28px radii.

### State signal (cross-cutting)
- **D-14:** OAuth consent surface displays scope chips using `.c-chip-protocol` for OAuth scope IDs (e.g., `[scope:read]`, `[scope:write]`). Active scope uses `var(--color-primary-text)` (theme-aware). DESIGN.md "Color blindness" rule: pair scope visual with text label, never color alone.
- **D-15:** Invite/[token] error states (404, 410, expired) use bracketed-glyph prefix `[err]` per UI-SPEC AC#9 pattern. Error body composes `.c-card` with `border-color: var(--color-error)` accent.

## Specific Ideas

- **Brand voice anchor (carry-forward from 213.1):** Strip all "Please" softeners. Replace with imperative or neutral copy. Match `obsidian/brain/Brand Stance.md` voice — developer-native, AI-first, quietly confident, no exclamation marks.
- **OAuth consent scope display:** Use `.c-chip-protocol` for scope IDs (e.g., `[markos:read]`, `[markos:write]`, `[markos:admin]`). Squared brackets per signature MarkOS pattern from DESIGN.md.
- **Login + signup error states:** error message renders in `.c-field__error` (auto-prepends `[err] ` via `::before` pseudo-element from `styles/components.css` line 128). Modules MUST NOT reimplement error styling.
- **Invite/[token] expired/used-up state:** 410 Gone error renders inside `.c-card` with `border-color: var(--color-error)` and bracketed `[err]` prefix on the error message. No big red splash; subtle border + token-cited body matches DESIGN.md "Don't shout" principle.
- **Magic-link login (if present):** if login surface has a magic-link flow per `markos login` CLI command (Phase 204), the "check your email" success state composes `.c-card` with token-cited check-glyph (NOT emoji ✓ — use bracketed `[ok]` prefix per DESIGN.md "No emoji" + "Color blindness").
- **Signup tenant-naming UX:** existing surface likely has tenant-slug live-validation. Preserve the wiring; redesign only the visual feedback (use `.c-field__help` for "Available" / `.c-field__error` for "Already taken").
