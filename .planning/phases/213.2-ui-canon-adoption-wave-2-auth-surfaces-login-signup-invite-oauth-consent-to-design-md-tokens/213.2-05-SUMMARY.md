## Decisions Made

- **Single `@media (pointer: coarse)` block (NOT a second block).** CSS specificity and cascade are identical either way, but the single-block form keeps `grep -c '@media (pointer: coarse)' styles/components.css` returning 1 (Phase 213.1 AC#15 contract preserved as a literal-grep guarantee). The 213-1 a11y suite asserts `count == 1` indirectly via the regex `@media\s*\(pointer:\s*coarse\)\s*\{[\s\S]{0,200}\.c-nav-link[\s\S]{0,200}padding-block:\s*11px` which would still match if a second block existed but would yield drift confusion in future audits.
- **`.c-button` uses `min-height: var(--h-control-touch)` (44px direct), NOT `padding-block` lift.** Rationale: `.c-button` has fixed `height: var(--h-control)` 40px declared at line 20 of styles/components.css. A direct `min-height` bump to 44px is the cleanest 4px touch-target lift; padding-block would conflict with the existing `padding: 12px 20px` declaration. `.c-nav-link` is text-driven (no fixed height), so the Phase 213.1 `padding-block: 11px` lift is the correct primitive for that surface.
- **Test count target +39% / AC# count target +200%.** RESEARCH.md target ≥18 tests, achieved 25. CONTEXT D-17 target ≥7 AC# mentions, achieved 21. Over-coverage is intentional — each surface gets dedicated AC#3 + AC#9 + X-4 tests, plus cross-surface AC#11/12/13 multi-file scans, plus AC#15 dual-rule + non-redeclaration scans. Future surface waves can dilute this without dropping below thresholds.
- **Pre-existing test failures: file in deferred-items.md, do not fix.** `test/onboarding-server.test.js` MODULE_NOT_FOUND + `test/tenant-auth/ui-authorization-negative-path.test.js` Access-Denied regex BOTH verified pre-existing via `git stash` reproduction. Neither touches any 213.2 surface file (auth modules, components.css, or test/ui-a11y/). Out-of-scope per executor Rule 5 (scope boundary).
