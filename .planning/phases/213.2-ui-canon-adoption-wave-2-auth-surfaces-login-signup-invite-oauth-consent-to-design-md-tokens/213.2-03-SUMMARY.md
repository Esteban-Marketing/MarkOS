## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] aria-busy boolean coerced to invalid attribute value**
- **Found during:** Task 1 (post-write IDE diagnostic)
- **Issue:** Plan literal `aria-busy={isAccepting || undefined}` triggered jsx-a11y lint: "Invalid ARIA attribute value: aria-busy='{expression}'" (WAI-ARIA expects literal 'true'/'false' string token).
- **Fix:** Migrated to `aria-busy={isAccepting ? 'true' : 'false'}` (matches Plan 02's aria-invalid migration pattern).
- **Files modified:** `app/(markos)/invite/[token]/page.tsx`
- **Commit:** `f94802d`

**2. [Rule 1 - Lint] Nested ternary in button child**
- **Found during:** Task 1 (post-write IDE diagnostic)
- **Issue:** Plan literal `{isAccepting ? 'Accepting…' : isSuccess ? '[ok] Accepted. Redirecting…' : 'Accept invite'}` triggered typescript:S3358 ("Extract this nested ternary operation into an independent statement").
- **Fix:** Hoisted into `let buttonLabel = 'Accept invite'; if (isAccepting) ... else if (isSuccess) ...` and rendered `{buttonLabel}`. Equivalent runtime; lint-clean.
- **Files modified:** `app/(markos)/invite/[token]/page.tsx`
- **Commit:** `f94802d`
