<!-- Critical: pack-loader.cjs resolvePackSelection() code (from research — do NOT change loader) -->

From lib/markos/packs/pack-loader.cjs (lines ~258–271):
```javascript
function resolvePackSelection(familySlug, overlaySlug) {
  if (!overlaySlug) return { type: 'base', pack: getFamilyRegistry()[familySlug] };
  const overlayPath = path.join(__dirname, 'industries', overlaySlug + '.industry.json');
  if (fs.existsSync(overlayPath)) {
    return { type: 'overlay', overlaySlug, baseFamilySlug: familySlug };
  }
  return { type: 'base', pack: getFamilyRegistry()[familySlug] };
}
```

Key implications:
1. File must be at lib/markos/packs/industries/{slug}.industry.json
2. __dirname = lib/markos/packs/ — so path resolves to lib/markos/packs/industries/{slug}.industry.json
3. resolvePackSelection() does NOT parse JSON content — only existence check
4. Schema compliance matters for test 108.10 (Ajv validation) but NOT for runtime resolution

From lib/markos/packs/pack-schema.json (reference):
Required fields: slug, version, displayName, type, aliases, disciplines, completeness,
assets (requires baseDoc + proofDoc), fallbackAllowed, changelog
