<objective>
Ship the **signing-secret rotation orchestrator** in Wave 3. Implements **D-09** (admin-triggered only), **D-10** (dual-sign during 30-day grace), **D-11** (notifications — Plan 203-06 wires email cron; this plan ships the state machine), and **D-12** (rollback only during grace; post-grace the old secret is unrecoverable).

Three layers:
1. **`lib/markos/webhooks/rotation.cjs`** — 4 exports: `startRotation`, `rollbackRotation`, `finalizeExpiredRotations`, `listActiveRotations`. Each delegates to a Supabase RPC (bodies filled in migration 72 via this plan's SQL append).
2. **Dual-sign dispatch wiring** — `delivery.cjs` resolves `sub.secret_v2` and calls `signPayloadDualSign(sub.secret, sub.secret_v2, body)` (Plan 203-04 Task 1 shipped the signing fn). When not in grace, `secret_v2` is null → single V1 header.
3. **Three tenant-admin endpoints** — `rotate` (start), `rotate/rollback`, `rotations/active` (Surface 4 banner fetcher).

Plus migration 72 gets 3 RPC function BODIES (signatures declared in Plan 203-02):
- `start_webhook_rotation(rotation_id, subscription_id, tenant_id, new_secret, grace_ends_at, actor_id)`
- `rollback_webhook_rotation(subscription_id, tenant_id, actor_id)`
- `finalize_expired_webhook_rotations(now)`

Plus F-97 rotation contract + F-72 subscription schema extension.

Purpose: GA requires zero-downtime secret rotation; dashboard Surface 2 Settings panel (Plan 203-09) wires directly to these endpoints; Surface 4 grace banner queries `rotations/active`; daily notification cron (Plan 203-06) uses `listActiveRotations`.
Output: rotation library + migration 72 RPC bodies + 3 endpoints + F-97 contract + F-72 extension + dual-sign dispatch + 2 test suites.
