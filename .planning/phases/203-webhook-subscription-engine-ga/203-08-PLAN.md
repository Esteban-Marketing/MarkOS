<objective>
Ship the **circuit breaker** — Plan 203-04's D-14 (>50% of last 20 deliveries fail = trip) and D-15 (half-open exponential backoff 30s → 60s → 120s → 300s → 600s, capped at 600s) enforced via Upstash Redis sliding-window state per RESEARCH §Pattern 4.

Three integration points:
1. **State library** `lib/markos/webhooks/breaker.cjs` — `recordOutcome` (LPUSH + LTRIM + threshold check), `canDispatch` (state-machine gate), `classifyOutcome` (5xx/timeout → failure; 4xx → NOT failure), `getBreakerState` (read-only for dashboard — Plan 203-09 consumes).
2. **Dispatch wiring** — `delivery.cjs` calls `canDispatch` BEFORE the rate-limit check (breaker short-circuits entire dispatch); on fetch completion, calls `recordOutcome` with success/failure classification.
3. **Integration suite** — `test/webhooks/circuit-breaker.test.js` exercises the full state machine (closed → open → half-open probe success → closed; open → half-open probe fail → open longer).

Why Redis, not in-memory: RESEARCH §Pitfall 2 — Fluid Compute silently loses in-memory state on instance turnover. Upstash Redis sliding-window already proven via 202-04 pattern.

Key accepted tradeoff: The LPUSH/LTRIM race window for ~20-sample 50% threshold is documented as A3 in Assumptions Log — noise floor of a handful of false-positive trips per thousand invocations is tolerable.

Output: breaker library + delivery.cjs integration + 2 test suites.
</objective>

         - `const stateKey = \`\${STATE_KEY_PREFIX}:\${sub_id}\`;`

         - `await redis.ltrim(listKey, 0, WINDOW_SIZE - 1);`

         - If `last20.length < WINDOW_SIZE` → return `{ state: 'closed' }` (need full window).

         - If `rate > TRIP_THRESHOLD`:

           - `const trips = (parsed && typeof parsed.trips === 'number') ? parsed.trips + 1 : 1;`
           - `const backoff = HALF_OPEN_BACKOFF_SEC[Math.min(trips - 1, HALF_OPEN_BACKOFF_SEC.length - 1)];`
           - `const probe_at = Date.now() + backoff * 1000;`
           - `await redis.set(stateKey, JSON.stringify({ state: 'open', trips, probe_at }), { ex: backoff + 3600 });`
           - Return `{ state: 'open', trips, probe_at }`.

         - If `Date.now() >= parsed.probe_at` → return `{ can_dispatch: true, state: 'half-open', trips: parsed.trips }`.
