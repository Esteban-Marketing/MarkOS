<objective>
Ship the **DLQ library + daily purge cron** for Wave 2. The DLQ is NOT a separate Vercel Queue (RESEARCH §Pattern 2 note line 338: "Vercel Queues does NOT have a native DLQ") — it's a tenant-scoped SELECT over `markos_webhook_deliveries` rows with `status='failed' AND dlq_at IS NOT NULL`, window = last 7 days per **D-08**. This plan ships the read + transition + purge library all downstream plans use (203-04 replay, 203-09 dashboard S2 DLQ tab, 203-10 status page metrics) plus the Vercel cron that enforces the 7-day TTL.

Purpose: GA requires human-in-loop replay of failed deliveries (SOC 2 expectation). Without the DLQ lib, 203-04 replay has no row set to iterate; without the purge cron, stale DLQ entries accumulate forever (violates D-08 locked behavior).
Output: dlq.{cjs,ts} library (5 exports), daily purge cron wrapper, vercel.ts cron entry, 2 test suites.
</objective>
