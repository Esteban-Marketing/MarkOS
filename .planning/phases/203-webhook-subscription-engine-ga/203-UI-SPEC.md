# Phase 203 — UI Design Contract

> Visual and interaction contract for the Webhook Subscription Engine GA. Four surfaces: tenant-admin dashboard index (`/settings/webhooks`), per-subscription detail with 3 tabs (`/settings/webhooks/[sub_id]`), public status page (`/status/webhooks`), and a global rotation-grace banner rendered across tenant-admin routes when a rotation is live. Every token **must match** the Phase 201 + Phase 202 MarkOS admin-surface vocabulary exactly — MarkOS has one visual language, and 202-09 (`/settings/mcp`) shipped the canonical dashboard shape this phase mirrors. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

8-point scale, inherited verbatim from Phase 201 + 202. Multiples of 4 only.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon-to-text gap, chip gap in scopeChipGroup |
| sm | 8px | Cost-meter / success-bar inner gaps, chip padding, tab gap |
| md | 12px | Form field internal padding, dialog body margins, tabBar padding-y |
| lg | 16px | Default element spacing, table cell padding, card gap |

| 2xl | 32px | Card internal padding |
| 3xl | 40px | Standalone status page inner padding |
| 4xl | 48px | Status page vertical padding |

**Exceptions:**
- `min-height: 44px` on every interactive control (button, input, select, `<details>` summary, checkbox row, tab button, expand row) — WCAG 2.2 AA tap target.
- `min-height: 48px` on primary filled CTAs (Create subscription, Rotate secret, Replay selected) — matches 202 "Approve access" posture.
- Dialog padding: `28px` (matches 202 `.dialog`).
- Hero number card padding: `20px` inner, `24px` row gap (4 cards side-by-side ≥960px; wrap to 2x2 at 640–959px; stack at <640px).
- Expand-row content padding: `16px 16px 20px` (inline payload preview under a delivery row).

**Accent reserved-for list (explicit — ANY other accent use is a bug):**
1. Primary filled CTAs only: Create subscription · Rotate secret · Replay selected · Save settings · Test fire
2. Focus ring `outline: 2px solid #0d9488; outline-offset: 2px` on every `:focus-visible` interactive element
3. Success-rate meter fill (same treatment as 202 cost-meter fill)
4. Hero success-rate numeral color, but ONLY when value ≥ 99.9% (threshold expressed as CSS variable; below threshold the numeral reverts to `#0f172a`)
5. Active tab underline — 2px solid `#0d9488`, inactive tabs get `1px solid #e2e8f0` bottom border
6. "Closed" (healthy) circuit-breaker chip text — accent on `#e6fffb` chip bg
7. Replay-result success indicator inside inline expand row (success body text in `#0f766e`)

Accent is **never** used for body text, generic borders, table rules, tab backgrounds, or hover states that are not focus-visible.

**Contrast verification (all WCAG 2.2 AA):**
- Body `#0f172a` on `#ffffff` = 16.1:1 (AAA)
- Secondary `#475569` on `#ffffff` = 8.0:1 (AAA)
- Caption `#0f766e` on `#ffffff` = 5.0:1 (AA large + normal)
- CTA white `#ffffff` on `#0d9488` = 3.1:1 — **large text only** (button 16px+ bold qualifies)
- Destructive `#9a3412` on `#ffffff` = 7.2:1 (AAA)
- Warn text `#78350f` on `#fef3c7` = 8.4:1 (AAA)
- Error text `#991b1b` on `#fef2f2` = 8.1:1 (AAA)

Inherited from Phase 201 + 202 shape.

## Component Inventory — Surface 2: `/settings/webhooks/[sub_id]` (Detail)

### IA outline (top to bottom)

1. **Breadcrumb + heading card** — `aria-labelledby="sub-detail-heading"`.
   - Breadcrumb row: `<a href="/settings/webhooks">Webhooks</a> ›` + current URL mono 13px.
   - h1 = subscription URL truncated to fit (full URL readable via `title`); subheading = events subscribed (inline chip cluster) + created-at timestamp.
   - Right-side chip row: circuit-breaker chip + plan-tier RPS chip ("300 rps · Enterprise" or "10 rps · Free").
2. **Tab bar** — `role="tablist"` with 3 tabs.
   - `<button role="tab" aria-selected="true|false" aria-controls="panel-deliveries|panel-dlq|panel-settings">`
   - Active tab: 2px solid `#0d9488` bottom border; inactive: 1px solid `#e2e8f0` bottom border + `#475569` text; hover non-active → `#334155` text.
   - Tabs: **Deliveries** (with count badge "127") · **DLQ** (with count badge, destructive chip variant when > 0) · **Settings**.
3. **Tab panels** — only the active panel renders (no offscreen hidden panels; Next.js route state `?tab=deliveries|dlq|settings` drives selection; default = Deliveries).

#### Panel A — Deliveries (default)

1. Filter row: event-type multi-select dropdown + status filter (All · Success · Failed · Pending) + "Last 24h / 7d / 30d" range chip group (no custom range in 203).
2. Deliveries table — columns: **Timestamp** · **Event** · **Status** · **Latency** · **Attempt** · (expand chevron column). Row click expands inline.
3. **Inline expand region** (per row): renders within a spanning `<tr>` with a single `<td colspan="6">`. Contents:
   - Request URL (mono 13px) + signed headers list (monospace block).
   - Request body preview (`<pre><code>` with `white-space: pre-wrap; max-height: 320px; overflow-y: auto;`).
   - Response status code + response headers + response body preview (same structure).
   - Error reason (if failed, e.g. "HTTP 502 · Upstream responded in 4.8s then closed connection").
   - Footer action row: "Replay this delivery" (ghost accent outline; confirms via small `<dialog>`; single-row replay) + "Copy cURL" link (copies sanitized cURL to clipboard, signature NOT included; copy link reads "Copied." for 2s).
4. Empty state: "No deliveries yet. Fire a test from the index or wait for an event to trigger."

#### Panel B — DLQ

1. Intro line: "7-day replay window. Entries older than 7 days are purged automatically." (14px `#475569`). Right-aligned: "Retained: {N} / {cap}" counter.
2. Batch action bar — sticky at panel top.
   - "Select all" checkbox + N-selected counter.
   - "Replay selected" primary filled accent button (disabled until selection > 0). Label: "Replay ({N})".
   - Replay confirm opens `<dialog>`: "Replay {N} deliveries? Each will be re-signed with the current secret and dispatched now."
   - On success: toast "Replay queued for {N} deliveries."; table refreshes; the replayed entries remain in DLQ but show a new "Replayed {relative_time}" status chip until a terminal result arrives.
3. DLQ table — columns: (checkbox) · **Timestamp** · **Event** · **Final status** · **Attempts** · **Last error** · **Actions**. Row-level Replay button (outline accent) + row-level Delete (outline destructive).
4. Empty state: "No dead-letter entries. All deliveries are succeeding or still retrying."

#### Panel C — Settings

Form sections, all fields use the `members/page.module.css` `.field` label/input pattern.

1. **Endpoint** — URL (text input, required, HTTPS-only validated) · Events subscribed (multi-select checkbox list of event enum).
2. **Rate limit** — RPS cap: shows plan-tier default ("Team plan default: 60 rps") + override input (`<input type="number" min="1">`) with helper text "Cannot exceed plan tier ceiling." Input is disabled/rendered `readOnly` if user lacks admin role.
3. **Signing secret** — dedicated card-within-panel (`.secretPanel`).
   - Current signature header preview: `x-markos-signature-v1=...` (mono 13px, last 8 chars visible, rest masked as `••••`).
   - **Rotate secret** primary filled accent button → opens rotation `<dialog>` with copy: "Rotate signing secret? The new secret will be generated now. Both old and new secrets will sign every webhook for 30 days (the grace window). After grace, the old secret is purged."
   - During grace window: replaces Rotate button with a `.graceStatusPanel` showing:
     - State chip: "Grace active — T-{N} days" (warn variant chip).
     - Countdown text: "Old secret retires on {absolute_date}."
     - **Rollback** button (outline destructive): "Rollback to old secret" → confirm dialog "Rollback rotation? This discards the new secret and restores the old one. The old secret remains live for the remainder of its grace window."
     - Dual-signature preview: two rows each showing header name + masked value.
   - After grace expires: reverts to single-secret panel; rollback is no longer offered.
4. **Danger zone** — card-within-panel with `border: 1px dashed #fca5a5; background: #fef2f2;` (mirrors 201 `danger/page.module.css` `.dangerCard` pattern). "Delete subscription" destructive filled button → confirm dialog "Delete this subscription? Any in-flight deliveries are cancelled. This cannot be undone."
5. **Save** action row at bottom — sticky if viewport < 640px; positioned right-aligned otherwise. "Save changes" primary filled accent. Disabled until form is dirty. Label busy: "Saving…".

**New 203 conventions (3 total, all justified):**
1. `1040px` max-width for S1 + S2 top-level cards — the 6-column subscriptions table does not fit 900px at ≥960px without column truncation; 1040 gives room without breaking the card-posture vocabulary.
2. `16px` nested-card radius — smaller than the 28px top-level radius so nested panels (`.secretPanel`, `.dangerCard`) read as "inside" the outer card. 16px matches `border-radius: 14px` of the 201 rotation/error banners (delta 2px acceptable; both read as "small radius").
3. Dark mono code block (`#0f172a` bg + `#e2e8f0` text) — webhooks inherently surface developer payload content; a dark block is the developer-native cue ("this is machine content") and avoids `#ffffff`-on-`#ffffff` nesting confusion. This ONE dark surface is additive; it never appears outside `<pre><code>`.

Any deviation beyond these 3 conventions is a design extension and must be flagged in UI-SPEC revisions.

### Tone rules (applied to all 203 copy — inherited from Brand Stance)

- **Developer-native, quietly confident.** No marketing superlatives, no exclamation points, no "Awesome!" / "Oops!" / "Yay!" framing.
- Prefer **specification over persuasion.** "HMAC-signed payloads with automatic retry." beats "Bulletproof delivery."
- Error copy names the problem + next action. Never apologizes, never blames the user.
- **Sentence case** for every heading and button ("Rotate secret", not "Rotate Secret"). Matches 201 + 202.
- Short sentences. Monosyllabic where natural. Match the MarkOS voice.
- **Destructive language is literal, not euphemistic.** "Delete subscription" (not "Remove"), "Purged" (not "Cleaned up"), "Cannot be undone" (not "Take care").
- **Timestamps** render via `<time dateTime="{ISO}">{relative}</time>` — relative ("3m ago") when < 24h, absolute date when older. Never "moments ago", "a little while ago", etc.

## Accessibility Contract (WCAG 2.2 AA per surface)

### Shared requirements

- [ ] Every interactive element has `:focus-visible` outline 2px `#0d9488` offset 2px.
- [ ] Every button has `type="button"` or `type="submit"` explicitly.
- [ ] Every input/select has an associated `<label>` (members-page `.field` pattern).
- [ ] Every `<table>` has a `<caption>` and every `<th>` has `scope="col"` (or `scope="row"` for row headers).
- [ ] Color contrast ≥ 4.5:1 for body text, ≥ 3:1 for large text and UI components (verified above).
- [ ] All functionality reachable via keyboard only.
- [ ] Tab order matches visual order.
- [ ] Toast region has `role="status" aria-live="polite"`.
- [ ] Error region has `role="alert"` (immediate announce).
- [ ] Rotation banner has `role="status"` (ambient — not `alert` to avoid announcing on every page nav).
- [ ] No `autoplay`, no motion > 500ms, no flashing content.
- [ ] Respects `prefers-reduced-motion: reduce` — meter fill, toast slide-in, chevron rotate, hover transitions all drop to instant.
- [ ] Touch target ≥ 44×44 CSS px (WCAG 2.2 AA 2.5.8).
- [ ] No `<div>` where a semantic element fits. `<section>`, `<nav>`, `<main>`, `<article>` used appropriately.

### Surface 1 specific

- [ ] `<section aria-labelledby="webhooks-hero-heading">` wraps the hero card.
- [ ] `<section aria-labelledby="webhooks-subs-heading">` wraps the subscriptions card.
- [ ] Each hero number uses `<div role="status">` when the value is machine-driven and will change (optional; announcement should be "polite" only).
- [ ] Success-rate mini-bar: `<div role="meter" aria-valuenow={pct_x10} aria-valuemin={0} aria-valuemax={1000} aria-label="Delivery success rate">` (x10 because aria-valuenow is an integer; 99.9% renders as 999).
- [ ] Status chip (circuit breaker) has `aria-label="Circuit breaker: {state}. {human description}."` so SR users hear the full semantics.
- [ ] Create-sub dialog: native `<dialog>` with `aria-labelledby` on h2. `showModal()` manages focus trap natively. Escape closes.
- [ ] URL cells are the `<a>` elements with `aria-label="Open subscription detail for {url}"` (because the displayed truncated URL may not be unique).

### Surface 2 specific

- [ ] `<nav role="tablist" aria-label="Subscription detail sections">` wraps the tab bar.
- [ ] Each `<button role="tab" aria-selected="..." aria-controls="panel-{id}" id="tab-{id}">`.
- [ ] Each panel `<section role="tabpanel" aria-labelledby="tab-{id}" id="panel-{id}" tabindex="0">` — `tabindex="0"` lets SR users land on the panel.
- [ ] Tab keyboard model: ArrowLeft/Right move focus; Home/End jump to ends; Space/Enter activate (manual activation, not automatic).
- [ ] Inline-expand delivery row: the expand toggle is a `<button>` in the trailing `<td>` (not a click-handler on the row) — a11y-preserving. `aria-expanded="true|false"`, `aria-controls` pointing to the expand row id.
- [ ] Expand row is `<tr id="expand-{delivery_id}">` with a single `<td colspan="6">`. Nested `<pre><code>` retains semantic meaning for SR / copy-paste.
- [ ] DLQ checkboxes: each `<input type="checkbox">` has a visually-hidden `<label>` "Select delivery from {timestamp}" so SR users can identify rows. The Select-all checkbox has `aria-checked="mixed"` when selection is partial.
- [ ] Rotate / Rollback / Delete dialogs: all native `<dialog>` with aria-labelledby, focus returns to the invoking button on close.
- [ ] Secret preview mono block: `<code aria-label="Current signature header value, last 8 characters">` + visually show masked prefix.
- [ ] Plan-tier RPS chip: plain text with `aria-label="Rate limit: {N} requests per second on {plan_tier} plan"`.

### Surface 3 specific

- [ ] `<main>` wraps the entire page (no `(markos)` shell).
- [ ] `<section aria-labelledby="status-heading">` wraps the single card.
- [ ] Hero numbers same a11y as S1.
- [ ] Status line: `<p role="status" aria-live="polite">`.
- [ ] Last-updated: `<time dateTime="{ISO}">` (machine-readable + human-readable text).
- [ ] No auth + no tenant context is surfaced in text — this is a public page.

### Surface 4 specific

- [ ] Banner `<div role="status">` (ambient; does NOT re-announce on every nav).
- [ ] Banner link uses descriptive text ("Review rotation for {url}") not "click here".
- [ ] T-0 variant escalation keeps `role="status"` (announcements of the same banner content are expected to be de-duplicated by AT via content match).
- [ ] Banner is rendered inside the shell, BEFORE the `<main>` of each page, so it's in the natural tab order once (not per-page).
