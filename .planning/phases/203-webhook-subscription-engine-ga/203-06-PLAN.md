<objective>
Ship Wave 3 rotation **notification cron + Surface 4 banner** — the communication layer for D-11.

Daily cron at 04:00 UTC sweeps active rotations, identifies those at stage T-7, T-1, or T-0 (from Plan 203-05's `computeStage`), emails tenant admins (role IN 'owner','admin' via `markos_tenant_memberships`) via **Resend** (reuses 202-10 `RESEND_API_KEY` with same console-log dry-run fallback), dedups via a notifications-sent log so each rotation+stage emits exactly one email, emits `source_domain='webhooks' action='secret.rotation_notified'` audit rows.

Surface 4 (the global grace banner) is a React client component placed in the workspace layout shell; fetches `/api/tenant/webhooks/rotations/active` (Plan 203-05 endpoint) once per session; renders stage-specific locked copy from UI-SPEC; exposes data-stage attribute; NO dismiss button (UI-SPEC security rule).

Purpose: D-11 locks "notifications at T-7 + T-1 + T-0 via email + dashboard banner during grace." Plan 203-05 shipped the rotation state + API; this plan ships the surfaces that communicate the state to admins.

Output: rotation-notify library + cron wrapper + vercel.ts 7th cron + Surface 4 banner (TSX + CSS) + 2 test suites.
</objective>
