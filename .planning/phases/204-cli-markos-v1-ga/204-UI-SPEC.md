# Phase 204 — CLI Design Contract (`markos` v1 GA)

> Visual and interaction contract for the `markos` CLI's 11-command GA surface. CLI-domain adaptation of the standard UI-SPEC: tokens are ANSI escape codes + Unicode/ASCII tiers + exit codes; "components" are output blocks (banner, table, status line, error box, spinner, progress bar, prompt, code-display); "interactions" are flag/argument grammars + interactive prompts. Generated by gsd-ui-researcher; verified by gsd-ui-checker.
>
> **Scope:** All output across the 11 GA commands (`init`, `generate`, `plan`, `run`, `eval`, `login`, `keys`, `whoami`, `env`, `status`, `doctor`) plus shared CLI primitives in `bin/lib/cli/`. The `generate` command's output format (shipped 200-02) is the **established baseline** — this contract codifies it forward, it does not redesign it.
>
> **Authority precedence:** `obsidian/brain/Brand Stance.md` (caveman mode, AI-first, quietly confident, evidence-backed) → `204-CONTEXT.md` D-07/D-10 (output format + error envelope) → `204-RESEARCH.md` Pattern 7 (TTY-adaptive output) → this UI-SPEC. Code already shipped in `bin/lib/cli/output.cjs` + `bin/lib/cli/errors.cjs` is the **as-built** anchor.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (zero-dep CLI per CONTEXT D-08) |
| Preset | not applicable — pure Node.js CJS, hand-rolled ANSI |
| Component library | none — `bin/lib/cli/output.cjs` + `bin/lib/cli/errors.cjs` (shipped 204-01) ARE the component library |
| Icon library | none — Unicode + ASCII tier only (see §Unicode/ASCII Tier) |
| Font | terminal-default monospace (no font assertion; user terminal owns it) |

**Why no third-party UX libs:** CONTEXT D-08 locks `keytar` as the only new dep. All ANSI/table/spinner/progress logic is hand-rolled (≤50 LOC per primitive). This is a feature, not a constraint — it eliminates supply-chain risk, removes font/terminal assumptions, and keeps the CLI install footprint <2 MB.

**Established baseline (DO NOT REDESIGN):**
- `bin/lib/cli/output.cjs` — `EXIT_CODES`, `ANSI`, `shouldUseJson`, `shouldUseColor`, `renderJson`, `renderTable`, `render`
- `bin/lib/cli/errors.cjs` — `formatError`, `exitCodeFor`, `ERROR_TO_EXIT`
- `bin/commands/generate.cjs` (200-02) — table-or-JSON output via `render()`; carry forward as the canonical pattern.

## Token Layer 1: ANSI Color Tokens

ANSI escape codes used. **Frozen in `bin/lib/cli/output.cjs::ANSI`.** No additions without an UI-SPEC amendment.

| Token | Code | Role | Reserved For |
|-------|------|------|--------------|
| `RESET` | `\x1b[0m` | Restore default | End of every styled span |
| `BOLD` | `\x1b[1m` | Emphasis | Table headers, error code, success checkmark |
| `DIM` | `\x1b[2m` | De-emphasis | Hints, secondary text, table separators, `(no rows)` |
| `CYAN` | `\x1b[36m` | Info / column header | Table headers, `INFO` log lines |
| `GREEN` | `\x1b[32m` | Success | `✓` checkmark, `success` status, `markos doctor` PASS |
| `YELLOW` | `\x1b[33m` | Warning | `⚠` symbol, `markos doctor` WARN, deprecation notices, slow-down poll |
| `RED` | `\x1b[31m` | Error / destructive | `ERROR` tag in error box, FAIL in doctor, `revoke` confirm prompt |

**60/30/10 split (CLI analog):**
- **60% dominant** — terminal default foreground (no ANSI). All result data, body text, table cells.
- **30% secondary** — `DIM` for separators, hints, timestamps, secondary metadata. `CYAN` for headers/labels.
- **10% accent** — `GREEN` (success-only) + `YELLOW` (warning-only) + `RED` (error/destructive only).
- **Destructive** — `RED` is exclusively for: error box `ERROR` tag, `markos keys revoke` confirmation prompt, `markos doctor` FAIL row, error-state checkmark `✗`.

**Forbidden:**
- `BLUE` (low contrast on default-blue terminal backgrounds; not in `ANSI` map)
- `MAGENTA` / `BG_*` codes (not in map — not approved)
- 256-color or truecolor escapes (`\x1b[38;5;Nm`, `\x1b[38;2;R;G;Bm`) — incompatible with cmd.exe legacy

**Color-blind safety (WCAG-equivalent for terminal):** Color is **never** the sole signal. Every status MUST dual-encode with an icon/word:
- success → `✓ ` + `GREEN`  (not green text alone)
- warning → `⚠ ` + `YELLOW` + word `WARN`
- error → `✗ ` or `ERROR` tag + `RED`
- info → no icon, plain or `CYAN`

**Accent reserved for:** success checkmarks (GREEN), warnings/doctor-warn (YELLOW), error envelope tag + revoke prompts + doctor-fail (RED), table headers (CYAN+BOLD). Never used to "decorate" arbitrary text.

## Token Layer 2: NO_COLOR + JSON Mode (color suppression)

**`NO_COLOR` env var** (per https://no-color.org): any non-empty value disables ALL color. Implemented in `output.cjs::shouldUseColor` line 43-48.

**JSON mode** (`--json`, `--format=json`, or `!process.stdout.isTTY`): emits one JSON object per line to stdout; NO ANSI codes ever. Implemented in `output.cjs::shouldUseJson`.

**Suppression matrix:**

| Condition | Color? | JSON? | Rationale |
|-----------|--------|-------|-----------|
| TTY + no NO_COLOR + no `--json` | YES | NO | Default human mode |
| TTY + `NO_COLOR=1` | NO | YES (per `shouldUseJson` line 39) | Honor no-color spec; ALSO emit JSON because plain text without color signals is ambiguous |
| TTY + `--json` | NO | YES | Explicit machine mode |
| Non-TTY (pipe/redirect) | NO | YES | Pipes are machine-readable contexts |
| `--format=yaml` | NO | NO (special) | YAML emit via existing `brief-parser.cjs` helpers |
| `--quiet` | (per other rules) | (per other rules) | Suppresses INFO-level stderr only; final result honor JSON/color rules |

## Token Layer 3: Unicode / ASCII Tier

CLI runs on terminals ranging from modern UTF-8 to legacy Windows cmd.exe (`LANG=C`, code page 437). Output MUST gracefully degrade.

**Detection:** `process.env.LANG` contains `UTF-8` OR `process.env.LC_ALL` contains `UTF-8` OR `process.platform !== 'win32'` AND `process.stdout.isTTY` → UTF-8 tier. Else → ASCII tier.

| Symbol Role | UTF-8 Tier (preferred) | ASCII Tier (fallback) | Color |
|-------------|------------------------|------------------------|-------|
| Success | `✓` (U+2713) | `[OK]` | GREEN |
| Failure | `✗` (U+2717) | `[FAIL]` | RED |
| Warning | `⚠` (U+26A0) | `[WARN]` | YELLOW |
| Info / pending | `•` (U+2022) | `*` | CYAN/DIM |
| Spinner frames | `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏` (Braille, 10 frames) | `\|/-\\` (4 frames) | DIM |
| Box top-left | `┌` | `+` | DIM |
| Box top-right | `┐` | `+` | DIM |
| Box bottom-left | `└` | `+` | DIM |
| Box bottom-right | `┘` | `+` | DIM |
| Box horizontal | `─` | `-` | DIM |
| Box vertical | `│` | `\|` | DIM |
| Box double-line (login code) | `═` | `=` | (no color) |
| Progress bar fill | `█` (U+2588) | `#` | GREEN |
| Progress bar empty | `░` (U+2591) | `.` | DIM |
| Arrow / pointer | `→` (U+2192) | `->` | DIM |

**Currently shipped:** `bin/lib/cli/errors.cjs` line 57-64 uses `┌`, `│`, `└` directly. **Action item for auditor:** confirm or add ASCII fallback. Not a Phase 204 blocker — auditor decision.

## Spacing Scale (CLI Analog)

CLI "spacing" = column gaps + indentation + blank lines. Frozen at:

| Token | Value | Usage |
|-------|-------|-------|
| `pad-cell` | 2 spaces | Gap between table columns (`output.cjs:92`: `const sep = '  '`) |
| `pad-indent` | 2 spaces | Nested list indent (e.g. `markos doctor` sub-checks) |
| `pad-banner` | 1 blank line above + 1 below | Section breaks (e.g. between login URL and code display) |
| `pad-eof` | 1 trailing newline | Every command's final output ends with exactly one `\n` (no trailing whitespace; safe for piping) |
| `pad-error-box` | 0 internal vertical pad | Error box is 3 lines: header / body / footer (no internal blank lines; compact) |

**Indentation rule:** 2 spaces, never tabs. Tabs render inconsistently across terminals; 2-space indent matches `bin/install.cjs` precedent.

**Exceptions:**
- `markos login` device code box uses `═` double-line frame with 1-cell horizontal pad inside the box (e.g. `║ WDJB-MJHT ║`).
- `markos generate` brief output (200-02 baseline) has no indentation — flat output.
- Dim separator under table header: 1 line of `-` characters scaled to column width (`output.cjs:102-103`).

## Typography (Monospace)

CLI typography = case + emphasis + character class. Terminal renders all output in user's monospace font; we control case/style only.

| Role | Style | Case | Width Cap |
|------|-------|------|-----------|
| Command name in help | `bold` | lowercase | — |
| Table header | `BOLD + CYAN` | UPPERCASE preferred (column names) | computed (max of header + cell widths) |
| Table cell | plain | as-data | computed |
| Section banner | `BOLD` | Title Case | ≤ terminal width |
| Error code (`UNAUTHORIZED`, `INVALID_ARGS`) | `BOLD + RED` | `SCREAMING_SNAKE_CASE` | — |
| Error message | plain | Sentence case, terse | wrap at 80 cols |
| Hint | `DIM` | lowercase, terse, imperative | wrap at 80 cols, prefix `hint:` |
| OAuth user code | `BOLD` (no color, in box) | `UPPERCASE-DIGITS` (e.g. `WDJB-MJHT`) | exactly 9 chars |
| Timestamps | `DIM` | ISO 8601 UTC (e.g. `2026-04-25T14:32:01Z`) | 20 chars |
| File paths | plain | as-given | ≤ terminal width; wrap on `/` |
| Status enums (`pending`, `running`, `success`, `failed`) | plain or color-paired (see §Status Lines) | lowercase | — |

**Width breakpoints (CLI analog of responsive breakpoints):**
- **<60 cols** (narrow / mobile SSH): drop low-priority columns; multi-line key:value layout instead of table.
- **60–100 cols** (standard): full table, unwrapped.
- **>100 cols** (wide): full table, no truncation; show full UUIDs and full timestamps.

Detection: `process.stdout.columns` (auto via `tty.WriteStream`). Fallback: 80 cols.

**Currently shipped:** `output.cjs::renderTable` computes per-column width from `max(header.length, all cell.length)` — already adaptive. No truncation logic yet; auditor may upgrade for narrow terminals. Not a P204 blocker.

## Color Contract (Status Lines)

| Status | Icon (UTF-8) | Icon (ASCII) | Color | Used In |
|--------|--------------|---------------|-------|---------|
| `success` / `OK` / `passed` | `✓` | `[OK]` | GREEN | doctor PASS, run.completed, login success |
| `running` / `pending` / `in_progress` | `•` | `*` | CYAN | run live event, doctor active check |
| `warn` / `degraded` | `⚠` | `[WARN]` | YELLOW | doctor warn, deprecation notice, slow-down |
| `failed` / `error` / `denied` | `✗` | `[FAIL]` | RED | doctor FAIL, run.failed, revoked key |
| `unknown` / `skipped` | `-` | `-` | DIM | doctor skipped check, missing data |

**Verbosity tiers** (per CONTEXT D-07):

| Flag | stdout | stderr |
|------|--------|--------|
| `--quiet` | final result only | suppressed (errors still shown) |
| (default) | result + 1-line summary | INFO prefixed `info:` (DIM) for non-essential progress |
| `--verbose` | result + full progress | INFO + sub-step details, audit-row ids, request-trace headers |
| `--debug` | result + raw payloads | full HTTP req/res dumps, retry attempts, keychain backend chosen |

**Streams (Unix convention, locked):**
- **stdout** — command result (table or JSON)
- **stderr** — logs, errors, progress (spinner, progress bar), confirmation prompts, hints

JSON output ALWAYS goes to stdout. Progress/spinner ALWAYS goes to stderr (so `markos run --json | jq` works without progress bytes corrupting JSON).

## Output Components

### Component 1: Banner
**Used in:** `markos init` welcome, `markos doctor` summary header, `markos status` section breaks.
**Shape:** 1 blank line + `BOLD` text + 1 blank line. No box.
**Caveman copy:** ≤6 words. No marketing fluff. Examples: `MarkOS doctor`, `Status (tenant: ten_abc123)`.

### Component 2: Table
**As-shipped:** `output.cjs::renderTable`. Header row in `BOLD + CYAN`, separator in `DIM` dashes, cells unstyled, 2-space column gap.
**Empty state:** `(no rows)` in `DIM` (line 70-73 in `output.cjs`). NEVER "No items found! Try X." — caveman mode.
**Used in:** `markos keys list`, `markos env list`, `markos status` (recent runs), `markos run --watch` (event tail).

### Component 3: Status Line
**Shape:** `<icon> <label>: <value>` per line. Icon dual-encoded (color + symbol). Label unstyled. Value can be styled per status.
**Example:**
```
✓ tenant         ten_abc123
✓ profile        default
⚠ quota          98% (running.warn)
✗ rotation       expired 2d ago
```
**Used in:** `markos whoami`, `markos status`, `markos doctor`, `markos env list` (with masked values).

### Component 4: Error Box
**As-shipped:** `errors.cjs::formatError` line 57-65. 3 lines:
```
┌── ERROR_CODE
│ Human message wrapping at 80 cols
└── hint: terse remediation, lowercase
```
**Color rules:** ERROR_CODE in `RED + BOLD`; rest unstyled. In JSON mode: `{"error":"CODE","message":"...","hint":"...","retry_after_seconds":N}` to stderr.
**Mandatory fields:** `error` (code, SCREAMING_SNAKE_CASE), `message` (human, sentence-case). Optional: `hint`, `retry_after_seconds`.
**No emoji. Ever.** (caveman mode + accessibility).

### Component 5: Spinner
**Shape:** UTF-8 Braille frames `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏` cycling at 80ms. ASCII fallback `\|/-\\` at 100ms.
**Format:** `<frame> <verb-ing>: <object>` to stderr. Color: DIM frame, plain verb. Example: `⠋ submitting: brief.yaml`.
**Suppressed when:** `--quiet`, non-TTY, `NO_COLOR`, `--json`. JSON mode never emits a spinner.
**Used in:** `markos run` (pre-SSE-attach), `markos login` (polling), `markos generate` (LLM call), `markos doctor` (active check). Hand-rolled per CONTEXT D-08; reference helper to add in `bin/lib/cli/spinner.cjs` if not yet present (auditor check).

### Component 6: Progress Bar
**Shape:** `[████████░░░░] 67% (2/3 steps)` — 12-cell bar by default (terminal-width responsive: 12 cells at <100 cols, 24 cells at ≥100 cols). UTF-8 fill `█` / empty `░`; ASCII `#` / `.`.
**Color:** GREEN fill, DIM empty. Whole bar suppressed when `--quiet`, non-TTY, `NO_COLOR`, `--json`.
**Used in:** `markos run --watch` (step counter), `markos generate` (token budget if known). When step count is unknown → spinner (Component 5) instead.

### Component 7: Confirmation Prompt
**Shape:** `<question> [y/N] ` to stderr; reads from stdin via `node:readline`. Default = NO (capital `N`). Accept: `y`, `Y`, `yes`, `YES` → proceed. Anything else → abort with `INVALID_ARGS` exit 1.
**Bypass:** `--yes` or `--force` flag (locked in CONTEXT D-09 + RESEARCH §Anti-Patterns "Prompting during --yes / CI"). Non-TTY without `--yes` → refuse with hint "rerun with --yes for non-interactive use".
**Used in:** `markos keys revoke <key_id>` (always confirm — destructive), `markos env push` (always confirm — overwrites server state), `markos env pull` if target exists (CONTEXT line 108 + Pitfall 4), `markos doctor --fix` per-fix (unless `--yes`).

### Component 8: Code Display Box (`markos login`)
**Shape:** Bordered box framing the OAuth user_code + URL + countdown.
**UTF-8 tier:**
```
  Open in browser: https://app.markos.com/cli/authorize

  ╔═══════════════════════╗
  ║      WDJB-MJHT        ║
  ╚═══════════════════════╝

  Code expires in 14m 30s. Polling...
```
**ASCII tier:**
```
  Open in browser: https://app.markos.com/cli/authorize

  +-----------------------+
  |      WDJB-MJHT        |
  +-----------------------+

  Code expires in 14m 30s. Polling...
```
**Color:** Box border DIM, code BOLD (no color — must be readable on any terminal bg), URL plain, countdown DIM.
**Headless mode** (`!isTTY` or `CI=1`): print the URL + code as plain lines, no box, no auto-poll countdown (per RESEARCH §Pattern 6 + open-browser.cjs:524-528).

### Component 9: Interactive Selection (deferred for v1)
**v1 posture:** Phase 204 ships ONLY confirmation prompts (Component 7). No multi-select / arrow-key navigation. `markos init` uses `bin/install.cjs` interactive flow which is `readline`-based question/answer (already shipped). `markos keys list` is read-only — selection happens via `markos keys revoke <key_id>` argument.
**Rationale:** Avoids `@inquirer/prompts` dep (CONTEXT D-08 zero-dep rule). If needed in v2, `Plant-seed` backlog.

## Command Grammar (Interaction Contract)

Flat command surface (CONTEXT D-01). Pattern: `markos <verb> [<noun>] [args] [flags]`.

| Command | Args | Flags | Output Mode (default) | Mutating | Exit Code Family |
|---------|------|-------|------------------------|----------|------------------|
| `markos init` | — | `--yes`, `--profile`, `--preset` | banner + interactive (delegates to `install.cjs`) | YES | 0 / 1 / 5 |
| `markos generate` | `<brief.yaml>` | `--out`, `--json`, `--quiet` | result blob (200-02 baseline) | NO | 0 / 1 / 2 / 3 |
| `markos plan` | `<brief.yaml>` | `--json`, `--quiet` | table or JSON | NO (dry-run) | 0 / 1 / 3 |
| `markos run` | `<brief.yaml>` | `--watch` (default true), `--timeout`, `--json`, `--quiet` | progress (stderr) + run event tail (stdout) | YES | 0 / 1 / 2 / 3 / 4 |
| `markos run <run_id>` | `<run_id>` | (status sub-pattern) | status line + recent events | NO | 0 / 1 / 3 |
| `markos eval` | `<brief.yaml>` | `--draft=<path>`, `--json` | rubric table + score | NO | 0 / 1 |
| `markos login` | — | `--token=<api-key>`, `--profile` | code-display box + spinner + success | YES (writes keychain) | 0 / 3 |
| `markos keys list` | — | `--json` | table | NO | 0 / 3 |
| `markos keys create` | `[name]` | `--json` | code-display box (one-time secret) + JSON | YES | 0 / 1 / 3 / 4 |
| `markos keys revoke` | `<key_id>` | `--yes` | confirm prompt + status | YES | 0 / 1 / 3 / 4 |
| `markos whoami` | — | `--verbose`, `--json` | status lines | NO | 0 / 3 |
| `markos env list` | — | `--json` | table (values masked) | NO | 0 / 3 |
| `markos env pull` | `[profile]` | `--force`, `--diff`, `--merge` | summary + diff | YES (writes `.markos-local/.env`) | 0 / 1 / 3 |
| `markos env push` | — | `--yes` | confirm + summary | YES | 0 / 1 / 3 / 4 |
| `markos env delete` | `<key>` | `--yes` | confirm + status | YES | 0 / 1 / 3 / 4 |
| `markos status` | — | `--watch`, `--json` | section banners + status lines + table | NO | 0 / 3 |
| `markos doctor` | — | `--check-only`, `--fix`, `--yes`, `--json` | check list + summary | YES (when `--fix`) | 0 / 1 / 5 |

**Global flags** (every command accepts):
- `--profile=<name>` (CONTEXT D-06; defaults to `MARKOS_PROFILE` env or `default`)
- `--json` (CONTEXT D-07; force JSON)
- `--format=table|json|yaml` (CONTEXT D-07; explicit)
- `--quiet` (CONTEXT D-07; suppress INFO stderr)
- `--help` / `-h`
- `--version` / `-v`

**Argument semantics:**
- Positional first, flags second. Both `--key=value` and `--key value` accepted (RESEARCH §Standard Stack: existing parser).
- Boolean flags accept negation: `--watch` true, `--no-watch` false (RESEARCH Pattern 5: SSE).
- Profile selector `--profile=<name>` MUST be respected before reading keychain.

**Mask rules (CONTEXT D-04):**
- `markos env list` shows last 4 chars only: `DATABASE_URL=•••••••@db.foo.com:5432/postgres` (masked at `=` boundary, full chars after last `@` shown for visual confirmation).
- `markos keys list` shows `key_fingerprint = sha256:<8-char-prefix>` only (RESEARCH §OAuth Wire Protocol). Never the raw key.
- `markos keys create` shows the raw key ONCE in a code-display box with hint `hint: store this key now; it will not be shown again.`

## Copywriting Contract (Caveman Mode)

Per `obsidian/brain/Brand Stance.md`: AI-first, quietly confident, evidence-backed. **Caveman mode applies:**
- Drop articles: "tenant" not "the tenant"
- Sentence fragments OK: "Login expired. Run `markos login` again."
- Imperative voice for hints: "rerun with `--yes`"
- No marketing copy ever: NO "Welcome to MarkOS!", NO "🎉", NO "Congrats!"
- Lowercase hints (`hint: ...`); SCREAMING_SNAKE_CASE error codes; sentence-case messages.
- Terse > complete. ≤80 cols per line. ≤2 lines per error message.

| Element | Copy |
|---------|------|
| Primary CTA per command | The verb is the command; no "Click here". Final success is `✓ <action>` (e.g. `✓ logged in as ten_abc123`, `✓ revoked key_xyz`). |
| Empty state (table) | `(no rows)` — period. (`output.cjs:71`) |
| Empty state (collection-specific) | `markos keys list` empty: `(no api keys)` + hint `hint: create one with 'markos keys create'`. `markos env list` empty: `(no env vars)`. `markos status` no recent runs: `(no recent runs)`. |
| Error envelope | `{ error: <CODE>, message: <human>, hint: <fix>, retry_after_seconds?: <n> }` (CONTEXT D-10) |
| Auth failure | code `UNAUTHORIZED`; message `not logged in or token expired`; hint `run 'markos login'`. Exit 3. |
| Quota exceeded | code `QUOTA_EXCEEDED`; message `tenant quota exceeded for plan: <plan>`; hint `upgrade at https://app.markos.com/billing or wait <retry_after_seconds>s`; pricing-sensitive copy MUST use `{{MARKOS_PRICING_ENGINE_PENDING}}` placeholder where plan-tier upgrade copy would otherwise hard-code prices (CLAUDE.md §Placeholder rule). |
| Network error | code `NETWORK_ERROR`; message `failed to reach markos.com`; hint `check connection or retry`. Exit 2. |
| Validation error | code `INVALID_ARGS`; message `<field>: <reason>`; hint `<corrective action>`. Exit 1. |
| Destructive confirm: `markos keys revoke` | prompt: `revoke key <key_id> ('<key_name>')? [y/N] ` |
| Destructive confirm: `markos env push` | prompt: `push <N> env vars to tenant <tenant_id>? [y/N] ` |
| Destructive confirm: `markos env pull` (when target exists) | prompt: `overwrite .markos-local/.env? [y/N] ` (default NO; `--force` bypasses; `--diff` previews; `--merge` preserves local-only) |
| Destructive confirm: `markos doctor --fix` (per check) | prompt: `apply fix: <fix-description>? [y/N] ` |
| `markos login` success | `✓ logged in as <tenant_name> (<tenant_id>)` |
| `markos login` expired | error code `TOKEN_EXPIRED`; message `login expired before approval`; hint `run 'markos login' again`. Exit 3. |
| `markos doctor` summary | `<N> passed · <N> warned · <N> failed` (DIM dot separators, color per status). Exit 0 if all pass-or-warn; exit 1 if any fail. |
| Pricing-sensitive output | NEVER hard-code public prices. Use `{{MARKOS_PRICING_ENGINE_PENDING}}` until approved `PricingRecommendation` exists (CLAUDE.md §Placeholder rule + CONTEXT non-negotiable line 65-66). Affects: `markos status` plan/tier display, `markos doctor` quota warnings, any error referencing upgrades. |

**Anti-patterns forbidden in CLI output (locked):**
- Emoji anywhere in error/warning paths (caveman + a11y)
- Color-only status (always dual-encode with icon/word)
- Marketing copy (`Welcome`, `🎉`, "Get started in seconds!")
- Trailing whitespace on any line (breaks pipe contracts)
- Mixed human/JSON in same stream
- Verbose default output ("Initializing parser... Loading config... Reading keychain...") — all such lines are `--verbose` only
- Hard-coded public prices in any output path (use `{{MARKOS_PRICING_ENGINE_PENDING}}`)
- Audit-row footprints in default output (audit emit happens silently; surface only in `--verbose` per CONTEXT line 110 + RESEARCH §Audit footprint)

## Exit Code Contract (CONTEXT D-10, locked)

Frozen in `bin/lib/cli/output.cjs::EXIT_CODES`. **Public stable API** — documented in `docs/cli/errors.md` (shipped 204-12).

| Code | Name | Meaning | Examples |
|------|------|---------|----------|
| 0 | SUCCESS | command completed successfully | login OK, run completed, all doctor checks pass |
| 1 | USER_ERROR | invalid input, validation failure, prompt declined | bad flags, missing brief.yaml, user said "n" to confirm, doctor fail (check-only) |
| 2 | TRANSIENT | retry-safe (network, 5xx, queue backpressure) | network timeout, server 502, SSE reconnect failed |
| 3 | AUTH_FAILURE | token / session problem | 401, no keychain entry, expired login, device flow expired |
| 4 | QUOTA_PERMISSION | 403 / 429 / quota | tenant quota hit, role gate denied, rate-limited |
| 5 | INTERNAL_BUG | unexpected exception | uncaught throw, malformed JSON from server, internal contract violation |

Mapping table at `errors.cjs::ERROR_TO_EXIT` (line 11-30) is the **canonical source**. CI scripts may rely on these codes.

## Accessibility (CLI a11y)

**Color-blind users (8% of male adult population):** All status uses dual-encoded icon + color. Never color-only. (Locked above.)

**Screen-reader users (terminal screen readers like NVDA + Orca + VoiceOver):** Output is plain text by default in non-TTY mode (auto-JSON or `NO_COLOR`). Box-drawing characters spoken as "vertical bar / horizontal bar" — acceptable. Avoid Unicode emoji (often unspoken or spoken as garbled SGML names). All informational symbols have ASCII fallback (see §Token Layer 3).

**Low-vision / high-contrast users:** Honor `NO_COLOR` (per spec). Bold-only as emphasis fallback when color disabled. No color-on-color combinations (e.g. never red text on yellow bg).

**Cognitive load:** Caveman mode IS the a11y win — short messages, terse hints, predictable structure. Error envelope always 3 lines + JSON form. Predictable exit codes.

**Pipe / scripting users:** Non-TTY auto-switches to JSON. Final newline always present (no missing-trailing-newline trap). stdout reserved for results, stderr for everything else (POSIX convention).
