# v3.6.0 Research Task Framework

**Milestone:** v3.6.0 Deep Research Tailoring Engine
**Purpose:** Define the strategic operating framework for every major research type MarkOS should support, and map each one to the best MCP, API, CLI, and agentic execution surface.
**Status:** Draft refinement
**Updated:** 2026-04-13

---

## 1. Strategic Principle

MarkOS should not treat all research as one generic prompt. It should route research through a typed framework:

1. classify the research task
2. apply the right filter stack
3. choose the right provider mix
4. synthesize into a context pack
5. generate only the artifact deltas that matter
6. preserve citations, freshness, confidence, and audit lineage

This means research becomes a structured operating system function rather than a one-off prompt.

## 2. Canonical Research Types

| Research Type | Strategic Goal | Typical Questions | Core Filters | Main Outputs |
|---|---|---|---|---|
| Market Landscape Research | Understand category structure and opportunity | market size, growth, demand shifts, major trends | industry, geography, company size, maturity | MARKET-TRENDS, INDUSTRY, positioning inputs |

| Company Intelligence | Understand a specific company in context | what they sell, how they position, what changed recently | company, product family, geo, pricing tier | ORG-PROFILE, PRODUCT-RESEARCH, account brief |
| Competitive Intelligence | Compare direct and indirect alternatives | who else wins, what angles are saturated, where are the gaps | industry, niche, audience, price band | COMPETITIVE-INTEL, differentiator set |
| Niche / Vertical Research | Go deep into one micro-market or specialized segment | regulations, norms, vocabulary, channel fit | niche, subindustry, region, compliance, company stage | niche memo, vertical messaging rules |
| Industry Regulation Research | Surface restrictions and risk constraints | claims allowed, privacy, data handling, ad limitations | industry, country, regulation class | compliance notes, safe copy rules |
| Product / Offer Research | Clarify what is being sold and why it matters | value props, objections, use cases, proof | product family, audience, maturity, market | PRODUCT-RESEARCH, offer matrix |
| Messaging Research | Find winning language and positioning angles | what language resonates, what proof persuades | audience, industry, stage, awareness level | messaging framework deltas |
| Channel Research | Match channels to intent and buying motion | where attention lives, what performs by channel | channel, audience, region, objective | MSP channel plans, channel prioritization |
| SEO / Search Intent Research | Map demand to queries and content gaps | what people search, what terms convert, what is underserved | industry, niche, location, funnel stage | keyword clusters, content strategy |

| Trend and Signal Monitoring | Track what changed and why it matters now | emerging narratives, risks, catalysts, seasonality | industry, region, time window | trend snapshot, update recommendations |
| Account-Based / Target Company Research | Prepare tailored strategy for one named target account | priorities, product line, motion, likely pains | company, account tier, region, audience role | ABM brief, outreach adaptation |

## 3. Provider Routing Framework

| Task Pattern | Best First Source | Best External Source | When to Use | Notes |
|---|---|---|---|---|
| Fast company and market lookup | MarkOS vault + MIR/MSP | Tavily | quick synthesis and search-driven enrichment | Already fits existing codebase patterns |
| Long-form cited investigation | MarkOS MCP knowledge + local evidence | OpenAI deep research / web search | when the question needs multiple hops and source comparison | Best for strategic memos and high-value research runs |
| Structured website extraction | company website + current MIR | Firecrawl | when the website needs structured extraction, branding capture, or deep page scrape | Useful for site refresh and company intelligence |
| Internal knowledge grounding | PageIndex / literacy vault | MarkOS read-only MCP | when the model should use company knowledge with no external dependency | safest default mode |
| Competitor discovery | current company brief | Tavily + selective Firecrawl | when named competitors or alternatives must be discovered or profiled | preserve citations and freshness |
| Sensitive or private research | internal vault only | none by default | when external leakage risk is unacceptable | read-only internal mode only |

### Routing rule

MarkOS should choose providers in this order:

1. internal approved evidence
2. internal MCP search/fetch over literacy, MIR, MSP, and approved artifacts
3. fast external search and extraction
4. deep long-running external synthesis only when the task truly requires it

## 4. Command Surface Design

### A. MCP tools

MarkOS should expose a read-only MCP server with a minimal, safe tool set:

- search_context
- fetch_document
- get_filter_taxonomy
- build_context_pack
- preview_artifact_delta
- compare_sources

These tools should work in:
- Claude Code
- VS Code Copilot chat with MCP support
- Cursor
- ChatGPT apps / deep research style integrations

### B. API surface

Recommended future API endpoints:

- POST /api/research/query
- POST /api/research/context-pack
- POST /api/research/delta
- POST /api/research/evaluate
- GET /api/research/providers
- GET /api/research/filter-taxonomy

### C. CLI / agentic commands

Recommended MarkOS commands:

- markos research run --type market
- markos research run --type audience
- markos research run --type niche
- markos research pack build
- markos research delta mir
- markos research delta msp
- markos research compare-sources
- markos research eval

Recommended slash-style agent flows:

- /markos-research market
- /markos-research audience
- /markos-research niche
- /markos-tailor-mir
- /markos-tailor-msp
- /markos-refresh-context

## 5. Framework by Client Surface

| Client Surface | Recommended Mode | Notes |
|---|---|---|
| Claude Code | Remote or local MCP first | best for multi-step research workflows with tool access |
| VS Code Copilot | MCP when available, local CLI fallback | good for repo-aware synthesis and artifact delta previews |
| Cursor | MCP-first with remote connectors | useful for long-running research and coding follow-through |
| Direct OpenAI API | Responses API with web search and deep research models | best for hosted or backend orchestration |
| Internal MarkOS runtime | local retrieval and provider router | safest default for production artifact generation |

---

## 6. Strategic Acceptance Bar

This milestone should only count as successful if MarkOS can reliably do all of the following:

- classify what kind of research is being requested
- choose the right combination of providers and internal knowledge
- return a cited context pack with freshness and confidence
- generate section-level MIR and MSP deltas
- preserve auditability and operator review
- adapt output meaningfully across industries, niches, audiences, and company contexts
