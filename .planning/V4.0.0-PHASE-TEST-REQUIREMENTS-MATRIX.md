# v4.0.0 Phase Test Requirements Matrix

**Date:** 2026-04-23  
**Purpose:** Give execution agents phase-specific and plan-specific testing obligations for Phases 204-228 using `Vitest`, `Playwright`, and `Chromatic`.

## How to use this matrix

- `Vitest` rows define deterministic business-logic obligations.
- `Playwright` rows define browser-proof obligations.
- `Chromatic` rows define visual regression obligations for stable UI states.
- `n/a` means the layer is optional for that plan, not forbidden for the phase.
- Every row is additive to the Phase 200 quality baseline.
