## Roadmap Overview

**Total Phases:** 5 (84-88 pending → v3.4.0 is phases 73-83)  
**Total Plans:** ~18-20 planned tasks across phases

### Phase 84: Vault Foundation (Obsidian Mind + PageIndex Contracts)
**Goal:** Establish hybrid vault structure, Obsidian Mind integration contracts, PageIndex adapter layer, and tenant-safe vault schema with deterministic pathing.  
**Requirements Mapped:** VAULT-01, VAULT-02, VAULT-03  
**Depends on:** v3.4.0 completion (Phase 83)  
**Status:** 📋 Planned  

**Plans:**
- [ ] 84-01-PLAN.md — Wave 1 vault structure contract: hybrid disciplines + semantic indices, Obsidian directory conventions, deterministic path resolution
- [ ] 84-02-PLAN.md — Wave 2 PageIndex adapter: retrieval contracts, audience/discipline query support, caching primitives, agentic orchestration hooks
- [ ] 84-03-PLAN.md — Wave 3 tenant-safe schema: Supabase RLS + vault metadata, isolation tests, and deterministic artifact referencing

**Key Outcomes:**
- Hybrid vault structure specification locked and testable
- PageIndex adapter with audience + discipline filtering ready
- Tenant isolation foundations proven in unit tests

---

### Phase 85: Ingestion Pipeline and Bidirectional Obsidian Sync
**Goal:** Enable operators to edit vault artifacts through Obsidian Mind with automatic backend sync, audit trail recording, PageIndex re-indexing, idempotent conflict resolution, and audience-tagged artifact validation.  
**Requirements Mapped:** LITV-01, LITV-02, LITV-03, LITV-04  
**Depends on:** Phase 84  
**Status:** 📋 Planned  

**Plans:**
- [ ] 85-01-PLAN.md — Wave 1 bidirectional sync contract: Obsidian file watcher, webhook ingestion, deterministic conflict resolution (last-write-wins + audit recovery)
- [ ] 85-02-PLAN.md — Wave 2 backend indexing: audit trail recording, PageIndex re-indexing triggers, metadata extraction
- [ ] 85-03-PLAN.md — Wave 3 idempotency gates: deduplication, conflict recovery path, full audit ledger
- [ ] 85-04-PLAN.md — Wave 4 audience tagging: artifact validation, audience metadata enforcement, role-scoped visibility tests

**Key Outcomes:**
- Obsidian-to-backend sync pipeline live and idempotent
- Audit trail fully recorded
- Audience-tagged artifacts validated and queryable

---

### Phase 86: Agentic Retrieval Modes (Retrieve+Reason, Retrieve+Apply, Retrieve+Iterate)
**Goal:** Implement three distinct retrieval modes for agents aligned with agentic execution patterns: raw artifact for customization, actionable templates, and iterative refinement with outcome verification.  
**Requirements Mapped:** ROLEV-01, ROLEV-02, ROLEV-03  
**Depends on:** Phase 84, Phase 85  
**Status:** 📋 Planned  

**Plans:**
- [ ] 86-01-PLAN.md — Wave 1 Retrieve+Reason mode: raw artifact retrieval for LLM customization, deterministic response contracts
- [ ] 86-02-PLAN.md — Wave 2 Retrieve+Apply and Retrieve+Iterate modes: actionable templates, outcome verification contracts
- [ ] 86-03-PLAN.md — Wave 3 query filtering: discipline-scoped and audience-scoped retrieval, PageIndex query orchestration, caching validation
- [ ] 86-04-PLAN.md — Wave 4 execution handoffs: deterministic payload construction, reasoning context preservation, agent multi-step loop support

**Key Outcomes:**
- Three retrieval modes fully implemented and tested
- Queries support both discipline and audience filtering
- Deterministic execution payloads ready for agentic consumption

---

### Phase 87: Dual Role-Views (Operator Management + Agent Execution)
**Goal:** Expose Obsidian-driven operator surfaces for artifact management and agent role-views using PageIndex retrieval with auditable execution logs; establish clear separation between human editor and agentic consumer contexts.  
**Requirements Mapped:** ROLEV-04  
**Depends on:** Phase 84, Phase 85, Phase 86  
**Status:** 📋 Planned  

**Plans:**
- [ ] 87-01-PLAN.md — Wave 1 operator views: Obsidian integration surfaces, artifact organization, basic publish/visibility controls
- [ ] 87-02-PLAN.md — Wave 2 agent views: PageIndex retrieval UI, execution log display, agentic query builder
- [ ] 87-03-PLAN.md — Wave 3 permissions and access controls: role-scoped visibility, operator-only edits, agent-only retrieval, audit trails
- [ ] 87-04-PLAN.md — Wave 4 UX ergonomics: discovery surfaces, search, filtering, saved queries, execution feedback

**Key Outcomes:**
- Operator can manage vault entirely from Obsidian
- Agents can retrieve, verify, and execute with full audit trail
- Role separation enforced at API and UI layer

---

### Phase 88: Governance, Verification, and Milestone Closure
**Goal:** Enforce tenant isolation (Supabase RLS + PageIndex scoping), implement hardened verification for agent executions without approval gates, collect execution telemetry, validate v3.4.0 non-regression baselines, and close milestone with proof of SLAs, sync stability, and isolation integrity.  
**Requirements Mapped:** GOVV-01, GOVV-02, GOVV-03, GOVV-04, GOVV-05  
**Depends on:** Phase 84, Phase 85, Phase 86, Phase 87  
**Status:** 📋 Planned  

**Plans:**
- [ ] 88-01-PLAN.md — Wave 1 tenant isolation: RLS schema enforcement, PageIndex query scoping, isolation tests, cross-tenant leakage detection
- [ ] 88-02-PLAN.md — Wave 2 execution telemetry: artifact ID tracking, retrieval mode logging, outcome vs. evidence comparison, anomaly flagging
- [ ] 88-03-PLAN.md — Wave 3 hardened verification: reasoning capture, outcome comparison to archived evidence, on-demand agent access gates (no human approval required)
- [ ] 88-04-PLAN.md — Wave 4 v3.4.0 regression gates: branding determinism baseline validation, governance publish/rollback preservation, UAT surface non-regression
- [ ] 88-05-PLAN.md — Wave 5 milestone closure: SLA validation, Obsidian sync stability proof, tenant isolation verification, v3.4.0 regression evidence, final acceptance ledger

**Key Outcomes:**
- Tenant isolation enforced end-to-end
- Agent execution fully telemetered and verifiable without approval gates
- v3.4.0 non-regression guarantee validated
- Milestone closed with proof of all 16 requirements
