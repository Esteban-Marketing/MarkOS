# Shared Business Model Coverage Map

This document defines the Phase 97 coverage contract for universal template families. It keeps the library additive: one reusable base plus deterministic family mapping and model-specific overlays only where the persuasion physics are materially different.

## EVIDENCE BASE
- Universal structure prevents template sprawl when the same strategic problem appears across multiple business models.
- Distinct overlays are only justified when trust posture, buying friction, or proof needs change meaningfully.

## CORE TACTICS
### Canonical family fallback
Use one stable family map so B2B, B2C, SaaS, agencies, services, consulting, info products, and ecommerce all resolve before the runtime falls back to a generic empty response.

### Alias normalization
Preserve existing repo slugs while normalizing DTC and Marketplace to ecommerce, and Agents-aaS to agency.

### Overlay trigger rule
Reserve overlays for SaaS, consulting, ecommerce, and info-products. Those models diverge most from the universal base in proof mechanics, urgency, and trust-building style.

## PERFORMANCE BENCHMARKS
| Check | Pass condition |
|------|----------------|
| Canonical coverage | All required families resolve deterministically |
| Overlay discipline | Only the 4 most distinct families receive first-class overlays |
| Runtime safety | Existing B2B, B2C, and SaaS behavior remains additive and backward-compatible |
