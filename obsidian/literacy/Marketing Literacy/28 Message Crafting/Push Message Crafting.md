# Push Message Crafting

> Channel: **push notification**. Stage 6 of [[Message Crafting Pipeline]]. Every draft in this channel conforms to the rules below + brand voice from [[Brand System Canon]].

## Constraints

- Title ≤ 35 chars; body ≤ 90.
- One CTA; deep-link directly to payoff surface.
- Timezone + activity window aware.
- Frequency cap per user per week.
- iOS MPP + Android personal preference respected.

## Canonical structure

1. Title — action or status
2. Body — specific payoff
3. Deep link — specific screen

## Hook patterns

- status change
- personalized recommendation
- time-bound opportunity

## Proof patterns

- user context ("you saved X"); specific metric

## Anti-patterns (auto-reject)

- generic engagement pings
- dark-pattern urgency
- vague rewards

## Voice defaults

Crisp, warm, user-centric — never begging for attention.

## Audience × pain tailoring

Every draft composes: **this channel's canonical structure × resolved [[Audience Archetype Canon|archetype]] × tagged pain from [[Pain-Point Engine]] × brand voice from [[Brand System Canon]] × optional triggers from [[Neuro Audit Canon]]**.

## Rejection criteria

- Missing archetype or pain tag → reject (Canon commandment I + II).
- Violates constraints above → reject or revise.
- Fails voice classifier or claim check → reject.
